**维护者**: 心跳时刻 - 番茄小说创作和运营
**最后更新**: 2026-03-25
**版本**: v1.0
**状态**: ✅ 可用

---

## 概览

番茄小说重复段落检测器是一个自动化工具，用于检测小说中的重复段落，帮助作者识别和修复重复内容，提高故事质量和读者体验。

**核心目标**：
- 自动检测重复段落（相似度≥80%）
- 识别重复模式（结构性重复、描写性重复、情感表达重复）
- 生成详细的检测报告
- 提供优化建议

---

## 功能概述

### 核心功能

#### 1. 段落分割
- **智能分割**：按空行分割段落
- **过滤短段落**：忽略小于10个字符的段落
- **保留上下文**：保留段落的前后文，便于理解

#### 2. 相似度计算
- **编辑距离算法**：使用动态规划计算段落相似度
- **相似度阈值**：支持自定义相似度阈值（默认80%）
- **双向比较**：段落A与段落B比较，段落B与段落A比较

#### 3. 重复检测
- **高相似度检测**：检测100%完全重复的段落
- **中度相似度检测**：检测80%-99%相似的段落
- **重复模式识别**：识别重复模式（结构性、描写性、情感表达）

#### 4. 报告生成
- **检测报告**：生成详细的检测报告（JSON/Markdown）
- **重复清单**：列出所有重复段落及其位置
- **优化建议**：提供针对性的优化建议

## 工具使用

### 工具1：重复段落检测脚本

**文件路径**: `~/.openclaw/skills/fanqie-duplicate-detector/duplicate-detector.js`

**使用方法**:

```bash
# 检测单个文件
node ~/.openclaw/skills/fanqie-duplicate-detector/duplicate-detector.js /path/to/story.md

# 检测目录下的所有文件
node ~/.openclaw/skills/fanqie-duplicate-detector/duplicate-detector.js /path/to/directory --all

# 自定义相似度阈值
node ~/.openclaw/skills/fanqie-duplicate-detector/duplicate-detector.js /path/to/story.md --threshold 90

# 输出JSON格式
node ~/.openclaw/skills/fanqie-duplicate-detector/duplicate-detector.js /path/to/story.md --format json
```

**输出示例**:

```
📊 重复段落检测报告
文件: 35号故事-穿成不受宠的太子妃-完整内容.md
总段落数: 486
检测阈值: 80%相似度

⚠️  发现 144 处重复段落

1. 段落 32 ≈ 段落 206
   相似度: 100.0%
   段落32: 原来，我一直都被爱着。...
   段落206: 原来，我一直都被爱着。...

2. 段落 41 ≈ 段落 110
   相似度: 84.6%
   段落41: 我回到偏殿，内心充满期待。...
   段落110: 我回到偏殿，内心充满感动。...
```

**参数说明**:
- `--threshold`: 相似度阈值（默认80%）
- `--format`: 输出格式（默认markdown，可选json）
- `--all`: 检测目录下的所有文件

## 最佳实践

### 1. 何时使用

**使用场景**:
- ✅ 创作完成后，检查重复段落
- ✅ 修改内容后，验证是否引入新的重复
- ✅ 批量检查多个故事
- ✅ 心跳时刻定期检查（每周一次）

**不使用场景**:
- ❌ 创作过程中（影响创作流畅度）
- ❌ 单个段落过短（小于10个字符）

### 2. 如何使用

**检测单个故事**:
```bash
# 1. 进入故事目录
cd ~/.openclaw/workspace/番茄短篇故事集

# 2. 检测故事
node ~/.openclaw/skills/fanqie-duplicate-detector/duplicate-detector.js "📤 待发布/35号故事-穿成不受宠的太子妃-完整内容.md"

# 3. 查看报告
# 输出会显示重复段落的数量、位置和相似度
```

**检测多个故事**:
```bash
# 1. 进入故事目录
cd ~/.openclaw/workspace/番茄短篇故事集

# 2. 检测所有待发布的故事
node ~/.openclaw/skills/fanqie-duplicate-detector/duplicate-detector.js "📤 待发布" --all

# 3. 查看每个故事的报告
# 输出会显示每个故事的重复段落数量
```

### 3. 如何解读报告

**重复率**:
- **0-5%**：优秀，无需优化
- **5-10%**：良好，可以优化
- **10-20%**：一般，需要优化
- **20-30%**：较差，必须优化
- **>30%**：很差，严重优化

**重复类型**:
- **100%完全重复**：必须删除或重写
- **80%-99%高度相似**：建议优化或重写
- **特定模式重复**：识别模式，针对性优化

**优化建议**:
- 合并重复章节
- 优化描写模板
- 增加情节推进
- 丰富情感表达

### 4. 如何修复重复

**步骤1：识别重复模式**
- 查看检测报告
- 识别重复段落的位置
- 分析重复类型（结构性、描写性、情感表达）

**步骤2：设计优化方案**
- 合并重复章节（结构性重复）
- 优化描写模板（描写性重复）
- 丰富情感表达（情感表达重复）
- 增加情节推进（情节推进缓慢）

**步骤3：实施优化**
- 删除完全重复的段落
- 重写高度相似的段落
- 合并重复章节
- 增加新情节和新信息

**步骤4：验证效果**
- 重新检测重复段落
- 对比优化前后的重复率
- 确保新内容没有引入新的重复

## 优化建议

### 1. 避免结构性重复

**问题**：重复使用相同的情节结构

**解决方案**:
- 合并重复章节
- 每章必须有新情节和新信息
- 避免重复使用相同的场景

**示例**:
```
❌ 错误：第2-6章都是"主角去花园 → 见到太子 → 简单对话 → 太子心声 → 主角感动 → 主角离开"
✅ 正确：第2章：进花园 → 第3章：送礼物 → 第4章：遇到困难 → 第5章：表白
```

### 2. 避免描写性重复

**问题**：重复使用相同的描写模板

**解决方案**:
- 减少"表面 vs 内心"对比的重复（每章1-2次）
- 丰富心声的多样性（每次心声都有新信息）
- 减少重复的对话模板
- 减少"离开"场景的重复（只在关键节点使用）

**示例**:
```
❌ 错误："表面上看，他的表情依旧淡淡，但是，他的心声却非常丰富。"（重复15+次）
✅ 正确：
  - "他嘴角微微上扬，眼中闪过一丝喜悦。"
  - "他的心跳加速，手心微微出汗。"
  - "他深吸一口气，努力掩饰内心的激动。"
```

### 3. 避免情感表达重复

**问题**：重复使用相同的情感表达

**解决方案**:
- 情感表达要多样化
- 避免重复使用相同的情感表达
- 情感要有层次（从浅到深）
- 情感要符合人物性格

**示例**:
```
❌ 错误："太幸福了！太幸福了！"（重复6+次）
✅ 正确：
  - "他的眼中闪烁着泪光，声音有些颤抖。"
  - "我的心跳得很快，不知道该说什么。"
  - "我们紧紧相拥，感受彼此的体温。"
```

## 限制和注意事项

### 1. 限制

- **段落长度**：只检测大于10个字符的段落
- **相似度阈值**：默认80%，可以自定义
- **检测精度**：基于编辑距离算法，可能误报
- **性能**：大文件（>100KB）检测时间较长

### 2. 注意事项

- **创作过程中不要使用**：影响创作流畅度
- **检测结果仅供参考**：需要人工判断是否真的需要优化
- **优化后重新检测**：确保新内容没有引入新的重复
- **保留原文件**：优化前备份原文件
