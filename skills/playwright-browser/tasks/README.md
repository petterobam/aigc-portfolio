# 任务系统说明 - 浏览器自动化守护者

任务列表记录所有待处理、进行中和已完成的优化任务。每次心跳后应同步更新。

## 任务文件

- `task-list.md` — 完整任务清单

## 优先级说明

| 级别 | 含义 | 响应时限 |
|------|------|---------|
| P0 | 紧急：Cookie 失效、脚本完全不可用 | 立即处理 |
| P1 | 高优：Cookie 即将过期（< 7 天）、脚本频繁报错 | 本次心跳处理 |
| P2 | 中优：功能改进、新检查项 | 本周内处理 |
| P3 | 低优：优化、文档、长期规划 | 本月内处理 |

## 任务状态

| 标记 | 含义 |
|------|------|
| 🔴 P0 | 紧急，阻塞性问题 |
| 🟠 P1 | 高优先级 |
| 🟡 P2 | 中优先级 |
| 🟢 P3 | 低优先级 |
| ✅ | 已完成 |
| ⏸️ | 暂停（等待外部条件） |
| 🚀 | 进行中 |

## 维护规则

- 每次心跳发现新问题，立即在 task-list.md 添加对应任务
- 完成任务后移入"已完成"区块，不要直接删除（保留历史）
- P0/P1 任务同时更新 `state/current-state.md` 的告警状态
