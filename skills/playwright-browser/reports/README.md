# 检查报告归档 - 浏览器自动化守护者

每次心跳执行完整健康检查后，将报告归档至此目录。

---

## 命名规则

```
YYYYMMDD-HHMMSS-check-report.md
```

示例：`20260325-143000-check-report.md`

---

## 报告内容规范

每份报告应包含：

| 字段 | 说明 |
|------|------|
| 执行时间 | 本次检查的时间戳 |
| Cookie 有效性 | sessionid 是否有效、过期时间、剩余天数 |
| 登录状态 | 访问番茄后台的结果 |
| CDP 端口状态 | 9222 端口是否可达 |
| 脚本执行结果 | 各脚本运行是否正常 |
| 发现问题 | 本次检查发现的异常（无则填"无"） |
| 处理动作 | 本次采取的动作（如刷新 Cookie、记录告警等） |
| 下次建议 | 下次心跳需要关注的事项 |

---

## 归档策略

- 保留最近 **30 份**报告，超出部分自动清理（最旧的优先删除）
- 如发现严重问题（Cookie 失效、脚本崩溃），对应报告标记为 `ALERT-` 前缀，永久保留
- 示例：`ALERT-20260325-143000-session-expired.md`

---

## 与其他文件的关系

- `logs/latest.md` — 最新一次执行的简短摘要（实时更新）
- `reports/` — 完整历史报告归档（长期保留）
- `state/current-state.md` — 当前状态快照（聚合最新数据）
