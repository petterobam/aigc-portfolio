# `/novel:specify` — 故事规格书

**存储路径**：`stories/<故事名>/specification.md`

**触发条件**：用户输入 `/novel:specify` 或提供故事概念

---

## 渐进式规格层级

根据用户输入详细程度自动判断输出层级：

| 层级 | 触发条件 | 产出内容 |
|------|---------|---------|
| Level 1 | 输入 <50 字 | 一句话故事（Logline，≤30字） |
| Level 2 | 输入 50-300 字 | 一段话概要（100-200字，含主角/冲突/目标） |
| Level 3 | 输入 300-1000 字 | 一页纸大纲（含三幕结构/约500字） |
| Level 4 | 已有规格需升级 | 完整规格书（九章全内容） |

---

## 完整规格书（Level 4）章节结构

- **第一章**：故事概要（一句话 + 类型 + 目标读者）
- **第二章**：核心价值主张（卖点差异化，≤3个）
- **第三章**：主要角色（主角弧度 + 对立力量 + 核心配角）
- **第四章**：世界观设定（时代/地点/特殊规则）
- **第五章**：线索管理规格（主线、支线、伏笔列表，带 `[需要澄清]` 标记）
- **第六章**：三幕/方法结构概要
- **第七章**：成功标准（字数目标/质量目标）
- **第八章**：约束条件（平台限制/内容禁区）
- **第九章**：风险预警

---

## 特殊标记

| 标记 | 含义 |
|------|------|
| `[需要澄清]` | 留给 `/novel:clarify` 处理的模糊决策点 |
| `[核心需求]` | 不可妥协的需求，优先保证 |
| `[可选特性]` | 锦上添花，不影响主线 |

**下一步**：提示用户执行 `/novel:clarify`
