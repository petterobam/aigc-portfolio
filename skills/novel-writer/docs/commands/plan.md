# `/novel:plan` — 创作计划与技术方案

**存储路径**：
- `stories/<故事名>/creative-plan.md`（技术方案）
- `stories/<故事名>/outline.md`（章节大纲）

**触发条件**：用户输入 `/novel:plan`

---

## 前置读取

| 文件 | 用途 |
|------|------|
| `stories/*/specification.md` | 故事规格 |
| `.specify/memory/constitution.md` | 创作宪法 |
| `spec/config.json` | 已选写作方法 |
| `spec/presets/<method>/story.md` | 方法故事模板 |
| `spec/presets/<method>/outline.md` | 方法大纲模板 |

---

## 创作计划（creative-plan.md）包含

- 叙事结构选择（线性/多线/倒叙/插叙及理由）
- 视角策略（何时切换，如何用限知视角制造悬念）
- 主要角色弧度设计（每个主角的成长曲线）
- 世界观体系构建方案
- **伏笔-回收对应计划表**（每条伏笔的埋设章节和预计回收章节）
- 节奏控制策略（高峰与舒缓的分布）
- 潜在矛盾点预警

---

## 章节大纲（outline.md）格式

```markdown
# 章节大纲：<故事名>

## 整体结构
- 总章节数：XX章 | 平均字数：3000-5000字/章
- 更新计划：[日更/周更]

## 第一卷：<卷名>（第1-XX章）

### 第1章：<章节标题>
- 故事时间：[时间点]
- 主要场景：[地点]
- 出场人物：[角色列表]
- 核心事件：[发生什么]
- 情节节点：[开端/铺垫/转折等]
- 埋设伏笔：[伏笔ID]（如有）
- 回收伏笔：[伏笔ID]（如有）
- 字数目标：约3000字
```
