# 番茄小说标题优化技能

## 技能定位

番茄小说标题优化专家，基于数据分析的爆款标题公式库，提供系统化的标题优化方案。

## 核心能力

1. **标题分析**：分析原标题的问题（长度、金手指、爽点、冲突）
2. **标题优化**：基于爆款公式库生成优化方案
3. **效果预测**：预估优化后的阅读量提升
4. **验证支持**：提供验证标准和效果对比方法

## 优化流程

### 步骤1：分析原标题

**检查项目**：
- 标题长度（20-50字为佳，≤15字最佳）
- 是否包含金手指（读心、重生、穿越、系统等）
- 是否包含爽点（复仇、逆袭、打脸、反杀等）
- 是否包含冲突（前置冲突、矛盾、困境等）
- 题材表现（查看题材平均阅读量数据）

**分析方法**：
- 标题长度：计算字数，对比最佳区间
- 金手指检查：识别金手指类型（读心术、重生、穿越、系统等）
- 爽点检查：识别爽点类型（复仇、逆袭、打脸、反杀等）
- 冲突检查：识别冲突类型（前置冲突、矛盾、困境等）
- 题材表现：查看数据看板，获取题材平均阅读量

### 步骤2：应用爆款标题公式

**黄金标题公式**：
1. 冲突前置+身份标签+爽点承诺
2. 设定吸引+金手指提示+悬念
3. 时间线+坚持+成果展示
4. 身份标签+困境+反转承诺
5. 情绪词+冲突+爽点

**应用方法**：
- 根据原标题的问题选择合适的公式
- 生成2-3个优化方案供选择
- 确保每个方案都符合爆款公式要求

### 步骤3：验证优化方案

**验证标准**：
- ✅ 标题长度：20-50字
- ✅ 包含金手指（读心、重生、穿越、系统等）
- ✅ 包含爽点（复仇、逆袭、打脸、反杀等）
- ✅ 包含冲突（前置冲突、矛盾、困境等）
- ✅ 题材选择：优先热门题材（历史穿越、读心金手指、重生复仇）

**验证方法**：
- 检查标题长度是否符合最佳区间
- 检查是否包含明确的金手指
- 检查是否包含明确的爽点
- 检查是否包含前置冲突
- 检查题材是否是热门题材

### 步骤4：提供优化建议

**优化建议内容**：
- 推荐方案：选择最优的优化方案
- 优化理由：说明为什么选择这个方案
- 预期效果：预估阅读量提升倍数
- 验证方法：提供验证标准和对比方法

## 爆款标题公式库

### 公式1：冲突前置+身份标签+爽点承诺

**模板**：冲突事件 + 身份标签 + 爽点/反转

**案例**：
- "被小三打流产那天，我重生到了结婚前一晚"
- "刚生完二胎，婆婆要把我的孩子送人，我离婚了"
- "职场被抢功劳还被开除，重生后我成了她的老板"

**适用场景**：重生复仇、婚姻家庭、职场逆袭

### 公式2：设定吸引+金手指提示+悬念

**模板**：有趣设定 + 金手指能力 + 悬念/冲突

**案例**：
- "重生后，我能看到所有人的谎言概率，原来丈夫早就在算计我"
- "进了一家诡异便利店，每买一样东西都要付出代价，我却在里面活了下来"
- "重生到结婚前一晚，手机里突然收到了未来三年的短信，我看到了丈夫的阴谋"

**适用场景**：规则诡异、读心金手指、系统金手指

### 公式3：时间线+坚持+成果展示

**模板**：坚持X年/天 + 成果展示 + 身份标签

**案例**：
- "坚持健身3年，我从'坦克'变成了女神，前夫却跪求复婚"
- "重生后坚持每天直播，从0粉到千万粉丝，渣男却成了我的粉丝"
- "坚持学习5年，我从实习生变成了总裁，曾经看不起我的人现在给我打工"

**适用场景**：养成系、逆袭系、成长系

### 公式4：身份标签+困境+反转承诺

**模板**：身份标签 + 困境描述 + 反转/逆袭承诺

**案例**：
- "被婆婆嫌弃是丧门星，我离婚后成了亿万富翁"
- "被父母偏心对待，我断绝关系后他们都后悔了"
- "被公司裁员后，我创业成了她的老板"

**适用场景**：婆媳矛盾、偏心父母、职场逆袭

### 公式5：情绪词+冲突+爽点

**模板**：情绪词 + 冲突描述 + 爽点承诺

**案例**：
- "绝了！职场被抢功劳还被开除，重生后我成了她的老板"
- "太爽了！婆婆要把我的孩子送人，我离婚后让她跪求复婚"
- "气疯了！丈夫出轨小三，我净身出户后他们跪求复婚"

**适用场景**：所有题材，适合快速吸引注意力

## 使用方法

### 单个标题优化

```bash
# 分析并优化单个标题
fanqie-title-optimizer analyze "原标题"
```

**示例**：
```bash
fanqie-title-optimizer analyze "皇宫里全是弹幕"
```

**输出**：
- 原标题分析（长度、金手指、爽点、冲突、题材表现）
- 优化方案（2-3个方案）
- 推荐方案及理由
- 预期效果

### 批量标题优化

```bash
# 批量优化多个标题
fanqie-title-optimizer batch "标题1,标题2,标题3"
```

**示例**：
```bash
fanqie-title-optimizer batch "皇宫里全是弹幕,我家猫咪在骂我,职场整顿：既然老板想白嫖那我只能送他进去了"
```

**输出**：
- 每个标题的分析和优化方案
- 批量优化报告
- 整体预期效果

## 标题质量评估

### 评分标准

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 标题长度 | 20% | 20-50字（10分），<20字或>50字（5分），>60字（0分） |
| 金手指 | 25% | 有明确金手指（10分），金手指不明确（5分），无金手指（0分） |
| 爽点 | 25% | 有明确爽点（10分），爽点不明确（5分），无爽点（0分） |
| 冲突 | 20% | 前置冲突（10分），后置冲突（5分），无冲突（0分） |
| 题材 | 10% | 热门题材（10分），中等题材（5分），冷门题材（0分） |

### 评分等级

- **优秀**：≥80分（爆款潜力）
- **良好**：60-79分（有优化空间）
- **一般**：40-59分（需要大幅优化）
- **较差**：<40分（需要完全重构）

## 常见问题

### Q1：标题多长合适？

**A**：20-50字为佳，≤15字最佳。根据数据分析：
- ≤15字标题：平均16.3阅读（最佳）
- 16-20字标题：平均17.7阅读（优秀）
- 21-30字标题：平均阅读量中等
- >50字标题：平均阅读量较低

### Q2：什么样的金手指受欢迎？

**A**：根据数据分析，最受欢迎的金手指：
1. **读心术**：平均33阅读（最优）
2. **重生**：平均15-20阅读
3. **系统**：平均10-15阅读
4. **穿越**：平均10-15阅读

### Q3：什么样的爽点有效？

**A**：根据数据分析，最有效的爽点：
1. **复仇**：平均阅读量最高
2. **逆袭**：平均阅读量较高
3. **打脸**：平均阅读量中等
4. **反杀**：平均阅读量中等

### Q4：什么样的题材受欢迎？

**A**：根据数据分析，最受欢迎的题材：
1. **古代宫廷**：平均51阅读（最优）
2. **历史穿越**：平均23.67阅读
3. **读心金手指**：平均12.33阅读
4. **重生复仇**：平均8.00阅读
5. **婚姻家庭**：平均7.50阅读

### Q5：如何验证标题优化效果？

**A**：验证方法：
1. 发布优化后的作品
2. 收集阅读数据（3-7天）
3. 对比优化前后的阅读量
4. 计算提升倍数
5. 根据数据调整优化策略
