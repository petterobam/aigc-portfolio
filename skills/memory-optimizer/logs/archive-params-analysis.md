## 评分系统建议

### 问题分析

**问题：未访问记忆评分不准确**

- 3 条学习日志/笔记评分 2.0，但从未被访问（access_count = 0）
- 这说明评分系统可能没有充分考虑到访问频率
- 或者是评分算法的问题

### 评分权重分析

当前评分权重：
```javascript
{
  accessFrequency: 0.25,  // 访问频率权重
  recency: 0.20,          // 时效性权重
  contentQuality: 0.25,   // 内容质量权重
  titleQuality: 0.15,     // 标题质量权重
  keywordDensity: 0.15    // 关键词密度权重
}
```

**分析**：
- 访问频率权重 25% 已经不低了
- 但未访问的记忆（access_count = 0）评分反而高，说明评分算法有问题
- 可能是访问追踪系统的问题（启发式访问统计不准确）

### 建议

**方案 1：增加访问频率权重**

```javascript
{
  accessFrequency: 0.30,  // 从 0.25 调整到 0.30
  recency: 0.15,          // 从 0.20 调整到 0.15
  contentQuality: 0.25,   // 不变
  titleQuality: 0.15,     // 不变
  keywordDensity: 0.15    // 不变
}
```

**理由**：
- 访问频率是记忆价值的重要指标
- 增加权重可以更好地反映记忆的实际价值
- 减少时效性权重，因为时效性不如访问频率重要

**预期效果**：
- 未访问记忆的评分会降低
- 访问频率高的记忆评分会提高
- 评分会更准确地反映记忆的实际价值

**方案 2：改进访问追踪系统**

**问题**：
- 当前访问追踪是启发式的，不是真实访问统计
- 可能导致访问次数不准确

**解决方案**：
- 集成到 OpenClaw 的记忆读取工具中，实现真实访问追踪
- 或者定期运行 access-tracker.js，更新访问统计

**预期效果**：
- 访问次数更准确
- 评分更准确
- 归档逻辑更有效
