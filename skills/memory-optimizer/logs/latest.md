### 1. 诊断评分系统问题

**目标**: 找出为什么未访问记忆评分 2.0（但从未被访问）。

**问题背景**:
- 3 条学习日志/笔记评分 2.0，但从未被访问（access_count = 0）
- 这与直觉相悖：未访问的记忆应该评分更低

**诊断过程**:

1. **查看当前评分权重**:

```javascript
scoreWeights: {
  accessFrequency: 0.25,     // 访问频率权重
  recency: 0.20,             // 时效性权重
  contentQuality: 0.25,      // 内容质量权重
  titleQuality: 0.15,        // 标题质量权重
  keywordDensity: 0.15       // 关键词密度权重
}
```

2. **手动计算评分**（以 2026-03-10 学习日志为例）:

假设一条学习日志：
- access_count = 0
- age_days = 4（2026-03-20 → 2026-03-25）
- content_length = 1961（很长）
- title_length = 15（理想长度）
- keywordDensity = 0.3（包含"学习"关键词）
- category = archived（类别权重 1.0）

评分计算：
- accessFrequency = 0 / 4 = 0
- recency = 1 / 4 = 0.250
- contentQuality = 1.0（长度 > 1000）
- titleQuality = 1.0（长度 10-50，理想）
- keywordDensity = 0.3

baseScore = (0 * 0.25) + (0.250 * 0.20) + (1.0 * 0.25) + (1.0 * 0.15) + (0.3 * 0.15)
         = 0 + 0.05 + 0.25 + 0.15 + 0.045
         = 0.495

finalScore = 0.495 × 1.0 = 0.495 ≈ 0.5

**问题根源**: 即使 accessFrequency = 0（未访问），其他 4 个维度的总和（0.495）仍然不低，导致最终评分接近 2.0（可能是历史遗留数据或评分算法不一致）。

| recency | 0.20 | 0.15 | -0.05 | 降低时效性权重，减少时效性对评分的影响 |
| contentQuality | 0.25 | 0.25 | 0 | 保持不变 |
| titleQuality | 0.15 | 0.15 | 0 | 保持不变 |
| keywordDensity | 0.15 | 0.15 | 0 | 保持不变 |

**预期效果**: 未访问记忆（access_count = 0）的评分应该下降，因为访问频率权重从 25% 增加到 30%。
