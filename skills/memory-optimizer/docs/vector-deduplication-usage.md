# 向量去重器使用指南

> 本文档说明如何配置和使用记忆优化器的向量去重功能（v1.0.0）。

---

## 功能概述

向量去重器使用 OpenAI Embeddings API 生成内容向量，通过余弦相似度识别语义相似的记忆。相比传统的哈希去重，向量去重能够识别内容不同但语义相同的重复记忆。

**适用场景**：
- 识别语义重复的记忆（如"爽点公式库"和"爽点公式总结"）
- 提升去重准确率
- 为后续的智能合并做准备

**工作原理**：
1. 将记忆内容转换为向量（1536 维）
2. 计算记忆之间的余弦相似度
3. 相似度 > 0.95 视为语义重复

## 快速开始

### 1. 安装依赖

向量去重器需要 `openai` npm 包：

```bash
cd ~/.openclaw/workspace/skills/memory-optimizer
npm install openai --save
```

### 2. 配置 OpenAI API Key

有三种方式配置 API Key：

**方式一：环境变量（推荐）**

```bash
export OPENAI_API_KEY="your-api-key-here"
node scripts/optimize.js
```

**方式二：在 .env 文件中配置**

在 `~/.openclaw/workspace/skills/memory-optimizer/.env` 中添加：

```
OPENAI_API_KEY=your-api-key-here
```

然后在脚本中加载：

```javascript
require('dotenv').config();
const openaiApiKey = process.env.OPENAI_API_KEY;
```

**方式三：在脚本中硬编码（不推荐）**

编辑 `scripts/optimize.js`：

```javascript
const CONFIG = {
  // ... 其他配置
  openaiApiKey: 'your-api-key-here'  // 不推荐，会暴露密钥
};
```

### 3. 启用向量去重

默认情况下，向量去重是启用的。如果需要禁用，编辑 `scripts/optimize.js`：

```javascript
const CONFIG = {
  // ... 其他配置
  enableVectorDedup: true  // 设为 false 禁用
};
```

## 配置参数

在 `scripts/optimize.js` 的 `CONFIG` 对象中配置：

```javascript
const CONFIG = {
  // 向量去重配置
  enableVectorDedup: true,             // 是否启用向量去重（默认: true）
  openaiApiKey: '',                     // OpenAI API Key（推荐使用环境变量）
  vectorSimilarityThreshold: 0.95,      // 向量相似度阈值（默认: 0.95）
  vectorSaveToDB: false                // 是否将向量存储到数据库（默认: false）
};
```

### 参数说明

- **enableVectorDedup**: 是否启用向量去重。如果设为 `false`，即使配置了 API Key 也不会运行。

- **openaiApiKey**: OpenAI API Key。推荐使用环境变量 `OPENAI_API_KEY`，如果配置了环境变量，此参数可以留空。

- **vectorSimilarityThreshold**: 相似度阈值，默认 `0.95`。值越大，识别越严格（只识别高度相似的记忆）；值越小，识别越宽松。

- **vectorSaveToDB**: 是否将向量存储到数据库。如果设为 `true`，向量会以 JSON 格式存储在 `content.embedding` 字段中。这样可以避免重复调用 API，但会增加数据库大小。

## 成本估算

### API 调用成本

OpenAI Embeddings API 定价（`text-embedding-3-small`）：
- 价格：$0.00002 / 1K tokens
- 记忆长度：假设平均 500 tokens
- 成本：每条记忆约 $0.00001

**示例计算**：
- 100 条记忆：约 $0.001（0.001 美元）
- 1000 条记忆：约 $0.01（0.01 美元）
- 5000 条记忆：约 $0.05（0.05 美元）

**建议**：设置 `vectorSaveToDB: true` 存储向量，避免重复调用 API。

### 时间成本

- API 调用延迟：每条记忆约 200-500ms
- 批量处理（10 条/批）：100 条记忆约 5-10 秒

## 性能优化

### 1. 启用向量存储

设置 `vectorSaveToDB: true` 存储向量到数据库，避免重复调用 API：

```javascript
const CONFIG = {
  vectorSaveToDB: true  // 首次调用后，后续直接从数据库读取
};
```

### 2. 调整批量大小

根据网络延迟调整 `batchSize`，默认是 10：

```javascript
const CONFIG = {
  batchSize: 20  // 网络快时可以增大批量
};
```

**注意**：OpenAI API 限制每次调用最多 2048 个文本，因此 `batchSize` 不要超过 2048。

### 3. 使用缓存

向量去重器默认启用内存缓存，相同内容不会重复调用 API。

## 故障排查

### 问题 1：未配置 API Key

**错误信息**：
```
⚠️  向量去重已启用但未配置 OPENAI_API_KEY，跳过向量去重
```

**解决方案**：
配置环境变量或在脚本中设置 `openaiApiKey`。

### 问题 2：API 调用失败

**错误信息**：
```
❌ 向量去重失败: Request failed with status code 401
```

**解决方案**：
检查 API Key 是否正确，是否有足够的余额。

### 问题 3：向量生成失败

**错误信息**：
```
生成向量失败 [ID]: Content length exceeds 8191 tokens
```

**解决方案**：
OpenAI API 限制每次调用最大 8191 tokens。如果记忆内容过长，可以：
1. 截断内容（保留前 8000 tokens）
2. 使用更小的模型（如 `text-embedding-ada-002`）

## 技术细节

### 余弦相似度计算

向量去重器使用余弦相似度计算两个向量的相似度：

```javascript
cosineSimilarity(vecA, vecB) = (vecA · vecB) / (||vecA|| * ||vecB||)
```

值域：[-1, 1]
- 1：完全相同
- 0：正交（无关）
- -1：完全相反

实际应用中，我们只关心 [0, 1] 范围，因为向量都是归一化的。

### OpenAI Embeddings 模型

当前使用 `text-embedding-3-small`：
- 维度：1536
- 最大输入：8191 tokens
- 价格：$0.00002 / 1K tokens

未来可以升级到 `text-embedding-3-large`：
- 维度：3072
- 最大输入：8191 tokens
- 价格：$0.00013 / 1K tokens

### 向量存储格式

向量以 JSON 数组格式存储在数据库中：

```json
{
  "embedding": "[0.1234, -0.5678, 0.9012, ...]"
}
```

这种格式虽然效率不高，但易于读取和调试。未来可以迁移到专门的向量数据库（如 LanceDB）。
