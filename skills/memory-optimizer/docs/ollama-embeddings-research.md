**创建时间**: 2026-03-25 11:49
**目标**: 使用本地 Ollama 替代 OpenAI 实现向量去重，解决 API Key 配置问题

## 背景

当前向量去重功能（Phase 1）已完成，但需要配置 `OPENAI_API_KEY` 才能运行。用户尚未配置 API Key，导致向量去重功能被阻塞。

本地发现 Ollama 已安装并运行（v0.18.0），有 24 个模型可用，包括 gemma:2b、mistral、llama2 等。Ollama 支持本地 embeddings API，可以作为 OpenAI 的替代方案。

---

## 可行性测试

### 测试 1：Ollama Embeddings API

**命令**:
```bash
curl -s http://localhost:11434/api/embeddings -d '{
  "model": "gemma:2b",
  "prompt": "Hello, world!"
}'
```

**结果**: ✅ 成功
- 返回了 768 维向量（gemma:2b 的 embedding 维度）
- 向量格式与 OpenAI 兼容（浮点数数组）
- 响应速度快（< 1 秒）

**向量示例**（前 10 维）:
```json
[1.963453292847, 2.084216833114624, -0.329877644777298, -0.4422370493412018, 0.6559513211250305, -0.9409817457199097, -0.22229589521884918, 1.795267939567566, -0.2726794183254242, -0.48220089077949524]
```

## 成本分析

### OpenAI 方案

- **Token 消耗**: 假设每条记忆 500 tokens，每周新增 10 条记忆
- **每周成本**: 500 × 10 × $0.0001 / 1000 = $0.0005
- **年度成本**: $0.0005 × 52 = $0.026
- **API Key 配置**: 需要用户手动配置

### Ollama 方案

- **Token 消耗**: 本地运行，无 API 调用
- **每周成本**: $0
- **年度成本**: $0
- **API Key 配置**: 无需配置
- **本地资源**: 需要约 2GB 内存（gemma:2b 模型）

**结论**: Ollama 方案成本更低（$0 vs $0.026/年），且无需配置，推荐优先实施。
