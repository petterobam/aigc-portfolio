## 核心知识点

### 1. 多项式采样（Multinomial Sampling）

**定义：**

多项式采样（Multinomial Sampling）是生成式模型中一种常见的**随机采样**方法。在生成下一个Token时，以模型预测的概率分布为依据，在概率分布中"按概率大小"随机抽取Token（而非等概率随机抽样）。

**核心思想：**

假设模型输出的概率分布为：
```
Token A: 0.4
Token B: 0.3
Token C: 0.2
Token D: 0.1
```

在多项式采样中，每个Token被选中的概率与其模型预测的概率成正比：
- Token A 有 40% 的概率被选中
- Token B 有 30% 的概率被选中
- Token C 有 20% 的概率被选中
- Token D 有 10% 的概率被选中

**与确定性解码的区别：**

| 方法 | 选择方式 | 特点 |
|------|---------|------|
| 贪婪搜索 | 总是选择概率最高的Token | 确定性、无多样性 |
| 波束搜索 | 维护多个候选路径 | 确定性、中等多样性 |
| 多项式采样 | 按概率分布随机采样 | 随机性、高多样性 |

**数学表示：**

给定词汇表 V 和模型输出的概率分布 P(x|x1, ..., xt-1)，多项式采样按照以下方式选择下一个Token：

```
x_t ~ Multinomial(P(·|x1, ..., xt-1))
```

即，Token x_t 被选中的概率为：
```
P(x_t = w | x1, ..., xt-1)
```

**代码示例：**

```python
import torch
import torch.nn.functional as F

def multinomial_sampling(logits, temperature=1.0):
    """
    多项式采样

    Args:
        logits: 模型输出的logits，形状为 [batch_size, vocab_size]
        temperature: 温度参数，控制随机性

    Returns:
        sampled_tokens: 采样的Token，形状为 [batch_size]
    """
    # 应用温度参数
    scaled_logits = logits / temperature

    # 计算概率分布
    probs = F.softmax(scaled_logits, dim=-1)

    # 多项式采样
    sampled_tokens = torch.multinomial(probs, num_samples=1).squeeze(-1)

    return sampled_tokens

# 使用示例
logits = torch.tensor([[2.0, 1.0, 0.5, -1.0]])  # 4个候选Token
sampled = multinomial_sampling(logits, temperature=1.0)
print(f"采样的Token ID: {sampled.item()}")
```

**温度参数的影响：**

温度参数 `temperature` 控制采样的随机性：

- **温度 < 1.0**：使概率分布更陡峭，采样更倾向于高概率Token
- **温度 = 1.0**：使用原始概率分布
- **温度 > 1.0**：使概率分布更平滑，采样更均匀

```python
import matplotlib.pyplot as plt
import numpy as np

# 示例概率分布
probs = np.array([0.4, 0.3, 0.2, 0.1])
tokens = ['A', 'B', 'C', 'D']

# 不同温度下的采样结果
temperatures = [0.5, 1.0, 2.0]

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

for i, temp in enumerate(temperatures):
    scaled_probs = probs ** (1/temp)
    scaled_probs = scaled_probs / scaled_probs.sum()

    axes[i].bar(tokens, scaled_probs)
    axes[i].set_title(f'Temperature = {temp}')
    axes[i].set_ylim(0, 0.6)

plt.tight_layout()
plt.show()
```

**应用场景：**

- **创意写作**：需要生成多样性的内容
- **对话系统**：避免重复、增加自然性
- **代码生成**：生成不同的代码实现方式

**优缺点：**

✅ **优点：**
- 生成多样性高
- 避免重复
- 更符合人类的表达方式

❌ **缺点：**
- 可能生成低质量的Token
- 需要多次采样才能得到好结果
- 受温度参数影响大

### 2. Top-K 采样（Top-K Sampling）

**定义：**

Top-K采样是一种在生成任务中常用的策略，类似于多项式采样，但其采样候选池经过限制。使用Top-K采样时，每一步生成Token时仅保留模型预测概率最高的**前K个**词，并从中按概率分布进行随机抽样。

**核心思想：**

1. 从模型输出的概率分布中选择概率最高的前K个Token
2. 将这K个Token的概率归一化
3. 从这K个Token中进行多项式采样

**示例：**

假设模型输出的概率分布为：
```
Token A: 0.4
Token B: 0.3
Token C: 0.2
Token D: 0.08
Token E: 0.01
Token F: 0.01
```

当 K=3 时：
1. 保留前3个Token：A (0.4), B (0.3), C (0.2)
2. 归一化概率：
   - Token A: 0.4 / (0.4 + 0.3 + 0.2) = 0.44
   - Token B: 0.3 / 0.9 = 0.33
   - Token C: 0.2 / 0.9 = 0.23
3. 从这3个Token中按归一化后的概率进行采样

**为什么Top-K有效：**

- 过滤掉低概率Token，减少生成无意义内容的可能性
- 保留多样性，因为仍然在前K个Token中随机采样
- 提高生成质量

**K值的权衡：**

| K值 | 特点 | 适用场景 |
|-----|------|---------|
| K=1 | 等同于贪婪搜索 | 需要确定性的任务 |
| K=10-50 | 适中的多样性 | 通用文本生成 |
| K=100-500 | 高多样性 | 创意写作、代码生成 |
| K=全部词汇 | 等同于多项式采样 | 最高多样性 |

**代码示例：**

```python
import torch
import torch.nn.functional as F

def top_k_sampling(logits, k, temperature=1.0):
    """
    Top-K采样

    Args:
        logits: 模型输出的logits，形状为 [batch_size, vocab_size]
        k: 保留的前K个Token
        temperature: 温度参数

    Returns:
        sampled_tokens: 采样的Token，形状为 [batch_size]
    """
    # 应用温度参数
    scaled_logits = logits / temperature

    # 计算概率分布
    probs = F.softmax(scaled_logits, dim=-1)

    # 获取前K个最大的值及其索引
    top_k_probs, top_k_indices = torch.topk(probs, k, dim=-1)

    # 归一化概率
    top_k_probs = top_k_probs / top_k_probs.sum(dim=-1, keepdim=True)

    # 从前K个Token中采样
    sampled_indices = torch.multinomial(top_k_probs, num_samples=1).squeeze(-1)

    # 获取实际的Token ID
    sampled_tokens = torch.gather(top_k_indices, -1, sampled_indices.unsqueeze(-1)).squeeze(-1)

    return sampled_tokens

# 使用示例
logits = torch.randn(1, 10000)  # 假设词汇表大小为10000
sampled = top_k_sampling(logits, k=50, temperature=1.0)
print(f"采样的Token ID: {sampled.item()}")
```

**动态K值：**

一些改进方法使用动态K值：

```python
def adaptive_top_k_sampling(logits, base_k=50, min_k=5, max_k=100):
    """
    自适应Top-K采样

    根据概率分布的陡峭程度动态调整K值
    """
    probs = F.softmax(logits, dim=-1)
    top_k_probs, _ = torch.topk(probs, base_k, dim=-1)

    # 计算熵（衡量分布的陡峭程度）
    entropy = -(top_k_probs * torch.log(top_k_probs + 1e-10)).sum(dim=-1)

    # 归一化熵到 [0, 1]
    max_entropy = torch.log(torch.tensor(base_k, dtype=torch.float))
    normalized_entropy = entropy / max_entropy

    # 根据熵调整K值
    dynamic_k = min_k + (max_k - min_k) * (1 - normalized_entropy)

    # 使用动态K值进行采样
    sampled = top_k_sampling(logits, int(dynamic_k.item()))
    return sampled
```

**实际应用：**

- **GPT-2**: 使用 Top-K 采样
- **开源模型**: 大多数支持 Top-K 采样

### 3. Top-P 采样（Top-P Sampling / Nucleus Sampling）

**定义：**

Top-P采样，又称**核采样**（Nucleus Sampling），该方法通过动态选择一个最小候选集合，使得候选词的概率和达到设定的概率阈值P，然后，在该候选集合中随机采样。与Top-K采样相比，Top-P采样能够根据概率累积**动态调整**候选集的大小。

**核心思想：**

Top-P 采样不是固定候选集的大小（K），而是根据概率累积动态调整：

1. 将概率从高到低排序
2. 从最高概率开始累积概率，直到累积概率达到P
3. 在这个"核"（nucleus）中进行采样

**示例：**

假设模型输出的概率分布为（按降序排列）：
```
Token A: 0.4
Token B: 0.3
Token C: 0.2
Token D: 0.08
Token E: 0.01
Token F: 0.01
```

当 P=0.9 时：
1. 从最高概率开始累积：
   - Token A: 0.4 → 累积: 0.4
   - Token B: 0.3 → 累积: 0.7
   - Token C: 0.2 → 累积: 0.9 ✓ 达到阈值
2. 核包含: {A, B, C}
3. 在核中采样（归一化后）

当 P=0.7 时：
1. 从最高概率开始累积：
   - Token A: 0.4 → 累积: 0.4
   - Token B: 0.3 → 累积: 0.7 ✓ 达到阈值
2. 核包含: {A, B}

**为什么Top-P更灵活？**

Top-P 采样能够根据概率分布的形状动态调整候选集大小：
- 当概率分布很陡峭时（少数Token占主导），核较小
- 当概率分布很平缓时（很多Token概率相近），核较大

**代码示例：**

```python
import torch
import torch.nn.functional as F

def top_p_sampling(logits, p, temperature=1.0):
    """
    Top-P采样（核采样）

    Args:
        logits: 模型输出的logits，形状为 [batch_size, vocab_size]
        p: 概率阈值，通常在0.9-0.95之间
        temperature: 温度参数

    Returns:
        sampled_tokens: 采样的Token，形状为 [batch_size]
    """
    # 应用温度参数
    scaled_logits = logits / temperature

    # 计算概率分布
    probs = F.softmax(scaled_logits, dim=-1)

    # 按概率降序排序
    sorted_probs, sorted_indices = torch.sort(probs, descending=True, dim=-1)

    # 计算累积概率
    cumulative_probs = torch.cumsum(sorted_probs, dim=-1)

    # 创建掩码：只保留累积概率 <= p 的Token
    # 但至少保留第一个Token（避免候选集为空）
    sorted_indices_to_remove = cumulative_probs > p
    sorted_indices_to_remove[..., 0] = False  # 至少保留一个

    # 应用掩码
    indices_to_remove = sorted_indices_to_remove.scatter(
        -1, sorted_indices, sorted_indices_to_remove
    )
    probs[indices_to_remove] = 0.0

    # 重新归一化
    probs = probs / probs.sum(dim=-1, keepdim=True)

    # 多项式采样
    sampled_tokens = torch.multinomial(probs, num_samples=1).squeeze(-1)

    return sampled_tokens

# 使用示例
logits = torch.randn(1, 10000)
sampled = top_p_sampling(logits, p=0.9, temperature=1.0)
print(f"采样的Token ID: {sampled.item()}")
```

**Top-P vs Top-K 对比：**

| 维度 | Top-K | Top-P |
|------|-------|-------|
| 候选集大小 | 固定K | 动态调整 |
| 自适应性 | 低 | 高 |
| 计算复杂度 | 低（只需要排序前K） | 高（需要全排序） |
| 适用场景 | 通用 | 概率分布变化大的场景 |

**Top-P + Top-K 组合：**

很多实际系统结合使用 Top-K 和 Top-P：

```python
def combined_sampling(logits, k=50, p=0.9, temperature=1.0):
    """
    组合采样：先Top-K，再Top-P
    """
    # 先应用Top-K
    top_k_logits, top_k_indices = torch.topk(logits, k, dim=-1)

    # 在Top-K的基础上应用Top-P
    sampled_indices_in_topk = top_p_sampling(top_k_logits, p, temperature)

    # 转换回原始的Token ID
    sampled_tokens = torch.gather(top_k_indices, -1, sampled_indices_in_topk.unsqueeze(-1)).squeeze(-1)

    return sampled_tokens
```

**实际应用：**

- **GPT-2**: 论文中对比了 Top-K 和 Top-P，发现 Top-P 效果更好
- **Hugging Face Transformers**: 大多数生成函数支持 `top_p` 参数
- **OpenAI API**: 使用 `top_p` 参数（默认 1.0）

**推荐的 P 值：**

- **P = 0.9**: 适合大多数任务
- **P = 0.95**: 更高的多样性
- **P = 0.8**: 更保守，质量更高

### 4. 对比搜索（Contrastive Search）

**定义：**

对比搜索（Contrastive Search）是一种解码策略，旨在平衡生成文本的**确定性**和**多样性**。它通过对比不同候选Token的"吸引力"（attractiveness）和"一致性"（coherence）来选择最佳的下一个Token。

**核心思想：**

对比搜索同时考虑两个因素：

1. **确定性**：模型预测的概率（高概率 = 高确定性）
2. **多样性**：新Token与已生成上下文的相似度（低相似度 = 高多样性）

**数学表示：**

给定候选Token w，其得分为：

```
score(w) = (1 - α) * log P(w|context) - α * max_sim(w, context)
```

其中：
- `P(w|context)` 是模型预测的概率
- `max_sim(w, context)` 是Token w与已生成上下文的最大相似度
- `α` 是超参数，控制确定性和多样性的权衡

**为什么对比搜索有效？**

- **避免重复**：通过惩罚与上下文相似的Token，减少重复生成
- **保持质量**：仍然考虑模型预测的概率，确保生成质量
- **无需超参数调整**：相比于Top-K/Top-P，对参数不太敏感

**代码示例：**

```python
import torch
import torch.nn.functional as F

def contrastive_search(
    model,
    input_ids,
    max_length=50,
    alpha=0.6,
    top_k=4,
    decoding_step=5
):
    """
    对比搜索

    Args:
        model: 语言模型
        input_ids: 输入Token IDs，形状为 [batch_size, seq_len]
        max_length: 最大生成长度
        alpha: 权衡参数（0-1）
        top_k: 每步保留的候选数
        decoding_step: 对比步长

    Returns:
        generated_ids: 生成的Token IDs
    """
    batch_size = input_ids.shape[0]
    device = input_ids.device

    # 复制输入
    current_ids = input_ids.clone()

    for step in range(max_length):
        # 获取模型输出
        with torch.no_grad():
            outputs = model(current_ids)
            logits = outputs.logits[:, -1, :]  # 最后一个Token的logits

        # Top-K筛选
        top_k_logits, top_k_indices = torch.topk(logits, top_k, dim=-1)
        top_k_probs = F.softmax(top_k_logits, dim=-1)

        # 计算相似度（与上下文最后一个decoding_step个Token）
        if current_ids.shape[1] >= decoding_step:
            context_ids = current_ids[:, -decoding_step:]
            context_embeddings = model.get_input_embeddings()(context_ids)  # [batch, decoding_step, hidden_dim]

            # 计算候选Token的嵌入
            candidate_embeddings = model.get_input_embeddings()(top_k_indices)  # [batch, top_k, hidden_dim]

            # 计算最大相似度
            similarities = torch.cosine_similarity(
                candidate_embeddings.unsqueeze(2),  # [batch, top_k, 1, hidden_dim]
                context_embeddings.unsqueeze(1),    # [batch, 1, decoding_step, hidden_dim]
                dim=-1
            )  # [batch, top_k, decoding_step]

            max_similarities = similarities.max(dim=-1)[0]  # [batch, top_k]
        else:
            max_similarities = torch.zeros_like(top_k_probs)

        # 计算对比分数
        contrastive_scores = (1 - alpha) * torch.log(top_k_probs + 1e-10) - alpha * max_similarities

        # 选择最高分数的Token
        best_indices = torch.argmax(contrastive_scores, dim=-1, keepdim=True)
        selected_tokens = torch.gather(top_k_indices, -1, best_indices).squeeze(-1)

        # 添加到序列
        current_ids = torch.cat([current_ids, selected_tokens.unsqueeze(-1)], dim=-1)

        # 检查是否到达结束符
        if (selected_tokens == model.config.eos_token_id).all():
            break

    return current_ids

# 使用示例（需要实际的模型）
# generated_ids = contrastive_search(model, input_ids, max_length=50)
```

**对比搜索 vs 其他解码策略：**

| 方法 | 确定性 | 多样性 | 重复率 | 适用场景 |
|------|--------|--------|--------|---------|
| 贪婪搜索 | 高 | 低 | 高 | 确定性任务 |
| Beam Search | 高 | 中 | 中 | 通用任务 |
| Top-K/Top-P | 中 | 高 | 高 | 创意任务 |
| 对比搜索 | 高 | 中 | 低 | 长文本生成 |

**优势：**

1. **低重复率**：显式惩罚重复Token
2. **高质量**：仍然考虑模型概率
3. **无需调参**：相比Top-K/Top-P，对参数不敏感
4. **一致性**：生成的内容更加连贯

**实际应用：**

- **论文**: Contrastive Search for Decoding（ICLR 2022）
- **实现**: Hugging Face Transformers 已支持
- **应用场景**: 长文本生成、故事创作

### 5. 投机解码（Speculative Decoding）

**定义：**

投机解码（Speculative Decoding）是一种加速大模型推理的技术。其核心思想是：使用一个较小的**辅助模型**（draft model）快速生成多个候选Token，然后使用一个较大的**目标模型**（target model）并行验证这些Token，保留验证通过的Token，拒绝不通过的。

**核心思想：**

1. **Draft（草稿）阶段**：小模型快速生成k个候选Token
2. **Verify（验证）阶段**：大模型并行验证这k个Token
3. **Accept/Reject（接受/拒绝）**：验证通过的Token被保留，不通过的Token需要重新生成

**为什么投机解码有效？**

- **并行计算**：大模型一次性验证多个Token，而不是逐个生成
- **小模型快速**：辅助模型虽然质量稍差，但速度快得多
- **接受率高**：如果小模型和大模型的方向一致，接受率会很高

**流程图：**

```
上下文: "The quick brown fox"

Draft阶段（小模型）:
  Token1: jumps    (draft概率: 0.6)
  Token2: over     (draft概率: 0.5)
  Token3: the      (draft概率: 0.7)
  Token4: lazy     (draft概率: 0.4)

Verify阶段（大模型并行验证）:
  Token1: jumps    (目标概率: 0.8)  ✓ 接受 (0.8 > 随机数)
  Token2: over     (目标概率: 0.6)  ✓ 接受 (0.6 > 随机数)
  Token3: the      (目标概率: 0.3)  ✗ 拒绝 (0.3 < 随机数)
  Token4: lazy     (不验证，因为Token3被拒绝)

最终生成: "The quick brown fox jumps over"
```

**数学表示：**

对于第i个草稿Token `x_i`，其接受概率为：

```
P_accept(x_i) = min(1, P_target(x_i) / P_draft(x_i))
```

接受决策：生成一个随机数 `u ~ Uniform(0, 1)`，如果 `u <= P_accept(x_i)`，则接受该Token。

**代码示例：**

```python
import torch
import torch.nn.functional as F

def speculative_decoding(
    draft_model,
    target_model,
    input_ids,
    max_length=50,
    speculation_length=5,
    temperature=1.0
):
    """
    投机解码

    Args:
        draft_model: 辅助模型（小模型）
        target_model: 目标模型（大模型）
        input_ids: 输入Token IDs
        max_length: 最大生成长度
        speculation_length: 每次推测的Token数
        temperature: 温度参数

    Returns:
        generated_ids: 生成的Token IDs
    """
    current_ids = input_ids.clone()
    device = input_ids.device

    for _ in range(max_length):
        # --- Draft阶段 ---
        draft_logits = draft_model(current_ids).logits[:, -1, :]
        draft_probs = F.softmax(draft_logits / temperature, dim=-1)

        # 采样k个草稿Token
        draft_tokens = []
        for _ in range(speculation_length):
            sampled_token = torch.multinomial(draft_probs, num_samples=1)
            draft_tokens.append(sampled_token)
            # 更新draft概率用于下一个Token
            # （这里简化处理，实际应该更新上下文）

        draft_tokens = torch.cat(draft_tokens, dim=-1)  # [batch, k]

        # --- Verify阶段 ---
        # 构建完整序列用于验证
        sequences_to_verify = []
        for i in range(speculation_length + 1):
            if i == 0:
                sequences_to_verify.append(current_ids)
            else:
                sequences_to_verify.append(
                    torch.cat([current_ids, draft_tokens[:, :i]], dim=-1)
                )

        # 批量验证（简化处理）
        accepted_count = 0
        for i in range(speculation_length):
            target_logits = target_model(
                torch.cat([current_ids, draft_tokens[:, :i+1]], dim=-1)
            ).logits[:, -1, :]

            target_probs = F.softmax(target_logits / temperature, dim=-1)

            # 获取目标模型对草稿Token的概率
            draft_token_id = draft_tokens[:, i].unsqueeze(-1)
            target_prob_for_draft = target_probs.gather(-1, draft_token_id).squeeze(-1)

            # 获取草稿模型对该Token的概率
            draft_prob_for_draft = draft_probs.gather(-1, draft_token_id).squeeze(-1)

            # 计算接受概率
            accept_prob = torch.min(
                torch.ones_like(target_prob_for_draft),
                target_prob_for_draft / (draft_prob_for_draft + 1e-10)
            )

            # 随机决定是否接受
            u = torch.rand_like(accept_prob)
            if (u <= accept_prob).all():
                accepted_count += 1
            else:
                break

        # 接受所有通过的Token
        if accepted_count > 0:
            current_ids = torch.cat([
                current_ids,
                draft_tokens[:, :accepted_count]
            ], dim=-1)
        else:
            # 如果全部被拒绝，用目标模型重新生成一个Token
            target_logits = target_model(current_ids).logits[:, -1, :]
            target_probs = F.softmax(target_logits / temperature, dim=-1)
            new_token = torch.multinomial(target_probs, num_samples=1)
            current_ids = torch.cat([current_ids, new_token], dim=-1)

        # 检查是否到达结束符或最大长度
        if (current_ids.shape[1] >= input_ids.shape[1] + max_length or
            (current_ids[:, -1] == target_model.config.eos_token_id).all()):
            break

    return current_ids
```

## 8. 常见问题与解决方案

### Q1: 为什么生成的文本重复？

**原因：**
- 使用贪婪搜索或Beam Search
- 温度过低
- 缺乏多样性约束

**解决方案：**
- 使用Top-P或Top-K采样
- 提高温度参数
- 使用对比搜索
- 添加重复惩罚

```python
def repetition_penalty(logits, tokens_generated, penalty=1.2):
    """
    重复惩罚
    """
    for i in range(len(logits)):
        for token in set(tokens_generated):
            logits[i, token] /= penalty
    return logits
```

### Q2: 生成的文本质量不稳定？

**原因：**
- 温度过高
- Top-K或Top-P参数设置不当
- 模型能力不足

**解决方案：**
- 降低温度（0.5-0.8）
- 调整Top-P（0.85-0.95）
- 使用更大的模型
- 使用Few-shot提示

### Q3: 推理速度太慢？

**原因：**
- 使用Beam Search且beam size过大
- 模型太大
- 缺乏优化

**解决方案：**
- 使用投机解码
- 减小beam size
- 使用量化模型
- 使用KV-cache
- 使用更快的推理引擎（如vLLM、TensorRT-LLM）

### Q4: 如何评估解码策略的效果？

**评估指标：**

1. **自动评估**：

   - Distinct-n（词汇多样性）

2. **人工评估**：
   - 流畅性
   - 连贯性
   - 相关性
   - 多样性
