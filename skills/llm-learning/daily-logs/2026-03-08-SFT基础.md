## 核心知识点

### 1. 微调技术分类

微调技术根据实现方式可以分为四大类：

**（1）基于模型主体的微调**
- **全参数微调**：更新模型的所有参数
- **部分参数微调**：只更新部分层或模块的参数
- 开发成本较低，但参数量大

**（2）并联低秩微调**
- 在原始权重旁添加低秩分解的适配器
- 需要引入额外的新模块
- 实施过程相对复杂

**（3）Adapter Tuning**
- 在Transformer层之间插入适配器模块
- 需要引入额外的新模块
- 实施过程相对复杂

**（4）基于 Prompt 的微调**
- 不修改模型参数
- 从模型的输入着手进行微调
- 通过优化输入提示来引导模型输出

### 2. 指令数据的来源

**指令数据** 是为模型提供的一组引导性输入及期望输出的描述，通常包括：
- 问题（或任务提示）
- 对应的答案

**常见来源：**
1. **公开数据集**
   - FLAN、Super-NaturalInstructions、Alpaca、ShareGPT 等
   - 涵盖多种任务类型和领域

2. **人工标注**
   - 根据特定任务需求定制
   - 质量可控，但成本高

3. **模型生成**
   - 使用更强的模型生成指令数据
   - 如 Self-Instruct、Evol-Instruct 等方法

4. **社区贡献**
   - 开源社区的共享数据
   - 如 OSS-Instruct、OpenOrca 等

### 3. 数据拼接（Packing）

**问题背景：**
- 模型训练时通常使用固定长度的输入
- 当输入的数据长度不一致时，对于短序列，会在其末尾填充以匹配最大序列长度
- 这会导致计算资源的**浪费**

**Packing 的解决方案：**
将多条数据拼接（Packing）在一起，填充到一个固定长度的输入序列中。

**关键技术：**
为了确保计算过程中不同数据之间互不干扰，通常需要：
1. **重设位置编号**（Reset position ID）
   - 为每条数据独立分配位置ID
   - 避免不同数据之间的位置混淆

2. **重设注意力掩码**（Reset attention mask）
   - 为每条数据设置独立的注意力掩码
   - 确保注意力计算只在各自数据内进行
   - 防止不同数据之间的信息泄漏

**优势：**
- 提高训练效率
- 减少填充造成的计算浪费
- 更好地利用 GPU 资源

## 实践建议

1. **选择合适的微调方法**
   - 资源有限时，优先考虑 LoRA 或 Prefix-Tuning
   - 追求极致性能时，可以考虑全参数微调
   - 快速实验时，Prompt 微调是不错的选择

2. **构建高质量指令数据**
   - 混合使用公开数据集和自定义数据
   - 确保数据多样性，覆盖不同任务类型
   - 人工审核关键样本，保证数据质量

3. **优化数据拼接策略**
   - 根据任务特点设计 packing 策略
   - 考虑数据长度分布，合理设置 batch size
   - 实现高效的 attention mask 管理

4. **实验验证**
   - 小规模实验验证微调效果
   - 对比不同微调方法的性能
   - 关注训练效率和资源占用

## 代码示例

### LoRA 微调示例（伪代码）

```python
# LoRA 微调核心思想：ΔW 具有低秩性
# 将权重更新分解为两个低秩矩阵的乘积

import torch
import torch.nn as nn

class LoRALayer(nn.Module):
    def __init__(self, in_features, out_features, rank=8):
        super().__init__()
        self.rank = rank
        self.lora_A = nn.Parameter(torch.randn(in_features, rank))
        # B 使用全0初始化，确保训练初期不对模型造成过大扰动
        self.lora_B = nn.Parameter(torch.zeros(rank, out_features))

    def forward(self, x):
        # 计算 ΔW = A × B
        delta_W = self.lora_A @ self.lora_B
        # 返回低秩适配的输出
        return x @ delta_W

# 应用 LoRA 到线性层
original_layer = nn.Linear(4096, 4096)
lora_adapter = LoRALayer(4096, 4096, rank=8)

# 前向传播时：output = original_layer(x) + lora_adapter(x)
# 参数量对比：
# 原始层：4096 × 4096 = 16,777,216
# LoRA：4096 × 8 + 8 × 4096 = 65,536（仅 0.39%）
```

### 数据拼接示例

```python
def pack_sequences(sequences, max_length, eos_token_id):
    """
    将多个序列打包到一个固定长度的 batch 中
    """
    packed_sequence = []
    attention_mask = []
    position_ids = []

    current_position = 0
    for seq in sequences:
        # 添加序列到打包结果中
        packed_sequence.extend(seq)
        packed_sequence.append(eos_token_id)

        # 为该序列创建 attention mask
        seq_length = len(seq) + 1
        attention_mask.extend([1] * seq_length)

        # 为该序列创建 position ids
        position_ids.extend(list(range(seq_length)))

        # 添加分隔符（可选）
        attention_mask.append(0)  # 不参与注意力计算
        current_position += seq_length + 1

    # 填充到 max_length
    padding_length = max_length - len(packed_sequence)
    if padding_length > 0:
        packed_sequence.extend([0] * padding_length)
        attention_mask.extend([0] * padding_length)
        position_ids.extend([0] * padding_length)

    return {
        'input_ids': torch.tensor(packed_sequence),
        'attention_mask': torch.tensor(attention_mask),
        'position_ids': torch.tensor(position_ids)
    }
```
