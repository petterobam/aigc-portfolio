## 学习主题
LoRA（Low-Rank Adaptation，低秩适配微调）

## 核心知识点

### 1. LoRA 核心思想

LoRA 由微软研究团队于 2021 年发表，是目前应用最广泛的参数高效微调方法之一。

**核心思想**：微调前后模型的参数差异 ∆W 具有低秩性。

这意味着：
- 微调只会改变模型参数的一小部分信息
- 大部分参数保持稳定，只有少量需要调整
- 这种调整可以通过低秩矩阵来表示

**图像参考**：`images_chinese/source_svg/【SFT】LoRA（1 of 2）.svg`

### 2. 低秩分解原理

一个矩阵具有**低秩性**，意味着它包含较多的冗余信息，可以将其分解为多个低维矩阵而不会损失太多有用信息。

**实际案例**：
- 原始权重矩阵：1024 × 1024 = 1,048,576 参数
- LoRA 分解：1024 × 2 矩阵 + 2 × 1024 矩阵
- 新参数量：2,048 + 2,048 = 4,096 参数
- 参数减少比：4,096 / 1,048,576 ≈ **0.4%**

**数学表达**：
```
W_new = W_frozen + A × B
```
- W_frozen：冻结的预训练权重
- A：形状为 d × r 的矩阵（r 是秩，通常很小，如 8、16、32）
- B：形状为 r × d 的矩阵
- 仅训练 A 和 B，W_frozen 保持不变

**图像参考**：`images_chinese/source_svg/【SFT】LoRA（1 of 2）.svg`

### 3. LoRA 初始化策略

LoRA 的两个矩阵 A 和 B 采用不同的初始化方式：

**矩阵 A（下投影）**：
- 使用随机数初始化
- 常用 kaiming 初始化
- 目的：提供初始的适应能力

**矩阵 B（上投影）**：
- 使用全 0 初始化，或极小的随机数初始化
- 目的：确保训练初期，插入的 LoRA 模块不会对模型整体输出造成过大的扰动

**初始化的智慧**：
- 初始时 A×B ≈ 0（因为 B ≈ 0）
- 模型行为与冻结的预训练模型几乎相同
- 训练过程中，A×B 逐渐学习到需要调整的部分
- 这种"平滑启动"方式避免了初始扰动过大

## 代码示例

### PyTorch LoRA 实现（简化版）

```python
import torch
import torch.nn as nn

class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, rank=8, alpha=16):
        super().__init__()
        # 冻结的原始线性层
        self.linear = nn.Linear(in_features, out_features, bias=False)
        self.linear.weight.requires_grad = False  # 冻结参数

        # LoRA 参数
        self.rank = rank
        self.alpha = alpha  # 缩放因子

        # 矩阵 A：随机初始化（下投影）
        self.lora_A = nn.Parameter(torch.randn(in_features, rank) * 0.01)

        # 矩阵 B：零初始化（上投影）
        self.lora_B = nn.Parameter(torch.zeros(rank, out_features))

        self.scaling = alpha / rank

    def forward(self, x):
        # 原始输出
        result = self.linear(x)
        # LoRA 增量
        lora_result = (x @ self.lora_A @ self.lora_B) * self.scaling
        return result + lora_result


# 使用示例
class ModelWithLoRA(nn.Module):
    def __init__(self, model, rank=8):
        super().__init__()
        self.model = model
        self.rank = rank

        # 将某些层替换为 LoRA 版本
        for name, module in model.named_modules():
            if isinstance(module, nn.Linear) and "mlp" in name:
                # 在线性层后添加 LoRA 适配器
                lora = LoRALinear(
                    module.in_features,
                    module.out_features,
                    rank=rank
                )
                setattr(self.model, f"{name}_lora", lora)

    def forward(self, x):
        return self.model(x)


# 训练时只更新 LoRA 参数
model = ModelWithLoRA(base_model, rank=16)
optimizer = torch.optim.Adam([
    p for n, p in model.named_parameters()
    if "lora" in n
], lr=1e-4)
```

### 使用 Hugging Face PEFT（推荐）

```python
from transformers import AutoModelForCausalLM
from peft import LoraConfig, get_peft_model

# 加载基础模型
model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-7B")

# 配置 LoRA
lora_config = LoraConfig(
    r=16,                    # 秩
    lora_alpha=32,           # 缩放因子
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],  # 应用 LoRA 的层
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# 应用 LoRA
model = get_peft_model(model, lora_config)

# 查看可训练参数
model.print_trainable_parameters()
# 输出示例：
# trainable params: 16,777,216 || all params: 7,619,257,344 || trainable%: 0.2202

# 训练（只更新 LoRA 参数）
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)
```

## 实践建议

### 1. LoRA 参数选择

**秩（rank）的选择**：
- **小模型**（< 1B）：rank = 4-8
- **中等模型**（1B-10B）：rank = 8-32
- **大模型**（> 10B）：rank = 16-64
- **经验法则**：rank 越大，容量越高，但计算量也越大

**缩放因子（alpha）的选择**：
- 通常设置为 rank 的 2-4 倍
- `alpha = 2 × rank` 是常见配置
- 可以通过实验调整，影响学习速度

**目标模块选择**：
- **注意力层**：q_proj, k_proj, v_proj, o_proj（最常见）
- **前馈层**：gate_proj, up_proj, down_proj
- **组合使用**：同时微调注意力和前馈层效果更好

### 2. LoRA vs 全参数微调

| 特性 | LoRA 微调 | 全参数微调 |
|------|-----------|------------|
| 显存需求 | 低（约 10-20%） | 高（100%） |
| 训练速度 | 快 | 慢 |
| 模型容量 | 较低（受秩限制） | 完全容量 |
| 部署便捷性 | 高（只需加载 LoRA 权重） | 中（需要完整模型） |
| 适用场景 | 轻量微调、资源受限 | 完全适配、极致效果 |

### 3. LoRA 变体

**QLoRA**（Quantized LoRA）：
- 结合 4-bit 量化 + LoRA
- 进一步降低显存需求
- 可以在 24GB 显存上微调 65B 模型

**AdaLoRA**（Adaptive LoRA）：
- 动态调整每个层的秩
- 为重要的层分配更高的秩

**PiSSA**（Principal Singular values and Singular Vectors Adaptation）：
- 基于奇异值分解的 LoRA
- 理论上更有数学基础

### 4. 训练建议

**学习率**：
- LoRA 学习率通常比全参数微调高
- 范围：1e-4 到 5e-4

**训练轮数**：
- LoRA 收敛通常更快
- 3-10 个 epoch 即可

**Batch Size**：
- 可以使用更大的 batch size（因为参数少）
- 提高训练稳定性

## 关键优势

1. **参数效率**：可训练参数减少到 0.1%-1%
2. **显存友好**：大幅降低 GPU 显存需求
3. **训练快速**：参数少，梯度计算快
4. **部署灵活**：可以为不同任务训练多个 LoRA 模块，按需加载
5. **无损性能**：在大多数任务上接近全参数微调效果

## 常见问题

**Q: LoRA 会不会降低模型性能？**
A: 在大多数情况下不会。LoRA 在 SFT 任务上表现接近全参数微调，有时甚至更好（因为防止过拟合）。

**Q: 什么时候不应该用 LoRA？**
A: 需要完全重构模型知识、或需要极大改动时。例如，预训练、完全领域适配等。

**Q: 如何选择 rank？**
A: 从小（8-16）开始，如果效果不好逐步增加。rank 越大，容量越高，但计算量也越大。
