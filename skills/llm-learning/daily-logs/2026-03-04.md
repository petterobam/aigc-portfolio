# 第1天：LLM 总体架构

## 日期
2026-03-04

## 核心知识点

### 1. LLM 的两种主要架构形式

**Decoder-Only（仅解码器）**
- 当前主流大模型（GPT、LLaMA、Qwen 等）都采用此架构
- 特点：只包含解码器部分，通过自回归方式生成文本
- 优势：生成能力强，训练效率高

**MoE（Mixture of Experts，专家混合模型）**
- 在 FFN（前馈网络）部分引入多个专家网络
- 特点：每个输入只激活部分专家，提高模型容量和效率
- 代表模型：Switch Transformer、GLM-4 系列、Mixtral

**关键差异**：两者整体架构相似，主要区别在于 MoE 在 FFN 层引入了专家路由机制

---

### 2. LLM 的三大组成部分

```
输入层 → 多层 Decoder 堆叠 → 输出层
```

**① 输入层**
- 功能：将文本转换为数值矩阵（Tensor）
- 包含：Embedding 层、位置编码等
- 示例："你好" → [0.12, -0.34, 0.56, ...]

**② 多层 Decoder 堆叠**
- 核心组件：Multi-Head Attention（多头注意力）
- 重复多次：常见 32-96 层（如 GPT-3 有 96 层）
- 每层包含：Self-Attention、FFN、残差连接、LayerNorm

**③ 输出层**
- LM Head（语言模型头）：将隐藏状态转为 Logits
- Logits → Softmax → 概率分布
- 解码：根据概率选择下一个词

---

### 3. LLM 训练的两个阶段

**阶段一：预训练（Pre-Training）**
- 数据：海量文本（万亿级 token）
- 目标：学习语言知识和世界知识
- 方法：自监督学习（预测下一个词）
- 特点：
  - 消耗算力最大
  - 时间最长（数周到数月）
  - 成本最高（百万美元级）

**阶段二：后训练（Post-Training）**
- 目标：对齐人类偏好、增强特定能力
- 方法：
  - SFT（监督微调）
  - RLHF（人类反馈强化学习）
  - DPO（直接偏好优化）
  - 蒸馏（知识转移）
- 特点：
  - 数据量较小
  - 训练时间较短
  - 多轮迭代

**比例估算**：预训练 : 后训练 ≈ 90% : 10%（但后训练决定了模型的最终效果）

---

### 4. 自监督学习：预训练的核心方法

**什么是自监督学习？**
- 不需要人工标注
- 从数据本身生成监督信号
- 最经典任务：预测下一个词

**工作原理**
```
输入："今天天气真__"
模型预测："好"（概率最高）
实际标签："好"（数据中的真实词）
```

**优势**
- 数据获取成本低
- 训练规模可无限扩展
- 学到的知识广泛且通用

**类比**：就像让孩子通过大量阅读自然学会语言，而不需要老师逐词解释
