## 核心知识点

### 1️⃣ 强化学习基础架构

强化学习的运行主要涉及两个核心角色：智能体（Agent）和环境（Environment）。

**核心循环：**
- **智能体（Agent）**：通过感知环境的状态，基于策略选择并执行动作
- **环境（Environment）**：接收动作后更新状态并反馈奖励信号

**示例场景**（AGI 旅游公司）：
- 旅客选择旅游行程 → 自动驾驶汽车全程负责路线规划、驾驶、接送
- 通过多次实际运行，结合累计奖励（旅客评分），基于强化学习训练旅游路线规划模型
- 目标：逐步优化乘客的旅游体验，确定最受欢迎的最佳路线

**关键概念映射：**
- 状态（State, S）：当前环境情况，如车辆位置、交通状况
- 动作（Action, A）：智能体采取的操作，如左转、加速
- 奖励（Reward, R）：环境反馈的评分，如乘客满意度
- 策略（Policy, π）：状态到动作的映射规则

### 2️⃣ 马尔可夫决策过程（MDP）

MDP 是强化学习的理论基础，在马尔可夫链的基础上引入了动作（A）和奖励（R）的概念。

**MDP 五元组：**
1. **状态空间（S）**：所有可能的状态集合
2. **动作空间（A）**：所有可能的动作集合
3. **转移概率（P）**：在状态 s 采取动作 a 转移到状态 s' 的概率
4. **奖励函数（R）**：在状态 s 采取动作 a 获得的即时奖励
5. **折扣因子（γ）**：衡量未来奖励的重要性（0 ≤ γ ≤ 1）

**马尔可夫性质：**
- 当前状态包含了所有历史信息
- 未来只取决于当前状态，与历史无关
- 数学表达：P(S_{t+1}|S_t, A_t, S_{t-1}, A_{t-1}, ...) = P(S_{t+1}|S_t, A_t)

**重要性：**
- MDP 为强化学习提供了数学形式化的框架
- 大多数 RL 算法都基于 MDP 假设
- 即使部分可观测环境也可转换为 POMDP 问题处理

### 3️⃣ 策略（Policy）定义

策略是强化学习的核心概念，定义了智能体的行为规则。

**策略定义：**
- **确定性策略**：π(s) = a
  - 在每个状态下输出一个确定的动作
  - 适用于连续动作空间
  - 示例：π(状态="前方5米有行人") = "刹车"

- **随机性策略**：π(a|s) = P(a|s)
  - 在每个状态下输出动作的概率分布
  - 适用于离散动作空间
  - 示例：π(左转|前方岔路) = 0.3，π(右转|前方岔路) = 0.7

**策略参数化：**
- 表格型策略：有限状态/动作空间，用表格存储
- 函数近似策略：用神经网络拟合策略函数 π_θ(a|s)
- θ 为策略参数，通过梯度下降优化

**策略目标：**
- 最大化期望回报（Expected Return）
- J(π) = E_π[G_t] = E_π[Σ_{k=0}^∞ γ^k R_{t+k+1}]

### 4️⃣ 探索与利用问题（Exploration-Exploitation）

这是强化学习的核心挑战之一。

**探索（Exploration）：**
- 尝试未知动作，收集更多信息
- 目的：发现更好的策略
- 风险：可能获得低奖励

**利用（Exploitation）：**
- 选择当前已知的最优动作
- 目的：最大化即时收益
- 风险：可能陷入局部最优

**平衡策略：**

**ε-贪婪策略（Epsilon-Greedy）：**
```python
def epsilon_greedy_policy(state, q_values, epsilon=0.1):
    """ε-贪婪策略"""
    if random.random() < epsilon:
        # 探索：随机选择动作
        return random.choice(range(len(q_values)))
    else:
        # 利用：选择Q值最大的动作
        return argmax(q_values)
```

**动态 ε 值：**
- 训练初期：ε 较大（如 0.5），更多探索
- 训练后期：ε 逐渐减小（如 0.01），更多利用
- 衰减公式：ε_t = max(ε_min, ε_0 * decay^t)

**软策略：**
- Softmax 策略：按概率分布采样
- Boltzmann 策略：基于 Q 值的指数加权

**实践意义：**
- 智能体需要平衡短期收益和长期学习
- 过度探索 → 学习缓慢，难以收敛
- 过度利用 → 局部最优，无法达到全局最优

### 5️⃣ 三大机器学习范式对比

强化学习与监督学习、无监督学习并列。

**监督学习（Supervised Learning）：**
- 数据：有标签的样本 (x, y)
- 目标：学习映射 f(x) → y
- 反馈：即时、明确的标签
- 示例：图像分类、机器翻译

**无监督学习（Unsupervised Learning）：**
- 数据：无标签的样本 x
- 目标：发现数据内在结构
- 反馈：无明确反馈
- 示例：聚类、降维

**强化学习（Reinforcement Learning）：**
- 数据：状态、动作、奖励序列 (s, a, r, s')
- 目标：最大化累积奖励
- 反馈：延迟、稀疏的奖励信号
- 示例：游戏 AI、机器人控制

**关键区别：**
| 维度 | 监督学习 | 无监督学习 | 强化学习 |
|------|---------|-----------|---------|
| 标签 | 有 | 无 | 稀疏奖励 |
| 反馈 | 即时 | 无 | 延迟 |
| 目标 | 预测 | 结构发现 | 累积奖励 |
| 数据分布 | 独立同分布 | 独立同分布 | 序列相关 |

## 代码示例

### 示例 1：定义简单的 MDP 环境

```python
import numpy as np

class SimpleMDP:
    """简单的网格世界 MDP"""
    
    def __init__(self, size=4):
        self.size = size
        self.state = (0, 0)  # 初始位置
        self.actions = ['上', '下', '左', '右']
        self.goal = (size-1, size-1)  # 目标位置
        
    def reset(self):
        """重置环境"""
        self.state = (0, 0)
        return self.state
    
    def step(self, action):
        """执行动作，返回 (next_state, reward, done)"""
        x, y = self.state
        
        # 执行动作
        if action == '上':
            x = max(0, x - 1)
        elif action == '下':
            x = min(self.size - 1, x + 1)
        elif action == '左':
            y = max(0, y - 1)
        elif action == '右':
            y = min(self.size - 1, y + 1)
        
        # 更新状态
        self.state = (x, y)
        
        # 计算奖励
        if self.state == self.goal:
            reward = 10
            done = True
        else:
            reward = -0.1  # 每步小惩罚，鼓励快速到达
            done = False
        
        return self.state, reward, done
    
    def get_state_space(self):
        """状态空间大小"""
        return self.size * self.size
    
    def get_action_space(self):
        """动作空间"""
        return len(self.actions)
```

### 示例 2：ε-贪婪策略实现

```python
import numpy as np

def epsilon_greedy_policy(q_table, state, epsilon):
    """
    ε-贪婪策略选择动作
    
    参数:
        q_table: Q值表，形状为 [state_count, action_count]
        state: 当前状态索引
        epsilon: 探索概率 (0-1)
    
    返回:
        action: 选择的动作索引
    """
    if np.random.random() < epsilon:
        # 探索：随机选择动作
        action = np.random.randint(len(q_table[state]))
    else:
        # 利用：选择Q值最大的动作
        action = np.argmax(q_table[state])
    
    return action

def dynamic_epsilon_decay(initial_epsilon, min_epsilon, decay_rate, episode):
    """
    动态衰减 ε 值
    
    参数:
        initial_epsilon: 初始 ε 值
        min_epsilon: 最小 ε 值
        decay_rate: 衰减率
        episode: 当前回合数
    
    返回:
        epsilon: 当前回合的 ε 值
    """
    epsilon = max(min_epsilon, initial_epsilon * (decay_rate ** episode))
    return epsilon
```

### 示例 3：策略函数定义

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class PolicyNetwork(nn.Module):
    """随机性策略网络（Actor）"""
    
    def __init__(self, state_dim, action_dim, hidden_dim=64):
        super(PolicyNetwork, self).__init__()
        
        self.fc1 = nn.Linear(state_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, hidden_dim)
        self.fc3 = nn.Linear(hidden_dim, action_dim)
    
    def forward(self, state):
        """前向传播，输出动作概率分布"""
        x = F.relu(self.fc1(state))
        x = F.relu(self.fc2(x))
        logits = self.fc3(x)
        
        # 转换为概率分布
        action_probs = F.softmax(logits, dim=-1)
        return action_probs
    
    def select_action(self, state):
        """根据策略选择动作"""
        with torch.no_grad():
            action_probs = self.forward(state)
            action_dist = torch.distributions.Categorical(action_probs)
            action = action_dist.sample()
            return action.item(), action_dist.log_prob(action)

# 使用示例
state_dim = 4  # 状态维度
action_dim = 2  # 动作维度

policy = PolicyNetwork(state_dim, action_dim)
state = torch.randn(1, state_dim)
action, log_prob = policy.select_action(state)

print(f"选择的动作: {action}")
print(f"对数概率: {log_prob.item():.4f}")
```

### 示例 4：计算回报（Return）

```python
import numpy as np

def compute_return(rewards, gamma=0.99):
    """
    计算回报 G = r_t + γ * r_{t+1} + γ^2 * r_{t+2} + ...
    
    参数:
        rewards: 奖励序列 [r_t, r_{t+1}, ..., r_T]
        gamma: 折扣因子
    
    返回:
        G_t: 从时间步 t 开始的回报
    """
    G = 0
    for i, r in enumerate(rewards):
        G += (gamma ** i) * r
    return G

def compute_returns_batch(rewards_list, gamma=0.99):
    """
    批量计算多个序列的回报（反向迭代）
    
    参数:
        rewards_list: 奖励序列列表 [[r0, r1, ...], [...], ...]
        gamma: 折扣因子
    
    返回:
        returns: 每个时间步的回报 [[G0, G1, ...], [...], ...]
    """
    returns = []
    for rewards in rewards_list:
        G = 0
        episode_returns = []
        # 反向迭代计算回报
        for r in reversed(rewards):
            G = r + gamma * G
            episode_returns.insert(0, G)
        returns.append(episode_returns)
    return returns

# 使用示例
rewards = [1, 2, 3, 4, 5]
G = compute_return(rewards, gamma=0.9)
print(f"回报 G: {G:.4f}")

# 批量计算
rewards_batch = [
    [1, 1, 1, 10],
    [0, 0, 0, 10],
    [1, 2, 3, 4]
]
returns = compute_returns_batch(rewards_batch, gamma=0.9)
print(f"批量回报: {returns}")
```

### ⚠️ 常见误区
1. **混淆奖励与回报**：奖励是即时反馈，回报是累积奖励
2. **忽视马尔可夫性质**：不要假设环境满足马尔可夫性，实际应用中需要验证
3. **过度依赖探索**：探索应该随着训练逐渐减少
4. **策略参数化不当**：小规模问题可以用表格，大规模问题需要神经网络
