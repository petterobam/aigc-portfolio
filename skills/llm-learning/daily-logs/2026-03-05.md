## 核心知识点

### 1. LLM 输入层：文本 → 数值矩阵

**为什么需要输入层？**
- LLM 只能处理数值，不能直接理解文本
- 需要将人类可读的文本转换为机器可计算的数值矩阵（Tensor）

**输入层的处理流程**

```
文本 → Tokenizer → Token IDs → Embeddings → 数值矩阵
```

**① Tokenizer（分词器）**
- 功能：将文本切分为词元（Token）
- 示例：
  ```
  "你好世界" → ["你", "好", "世界"]
  "Hello world" → ["Hello", " world"]
  ```
- 不同模型的 Tokenizer 不同（GPT-4、LLaMA、Qwen 使用不同的词表）

**② Token IDs（词元编号）**
- 每个 Token 对应一个唯一的整数 ID
- 词表大小：通常 5万-15万（如 GPT-3 是 50257）
- 示例：
  ```
  "你" → 12345
  "好" → 67890
  "世界" → 13579
  ```
- 一一对应关系：每个 Token ID 映射到一个具体的词元

**③ Embeddings（词嵌入）**
- 将 Token IDs 转换为固定长度的向量
- 维度：常见 768、1024、4096 等（hidden_size）
- 示例：
  ```
  Token ID 12345 → [0.12, -0.34, 0.56, ..., 0.78] (768维)
  ```
- 语义表示：相似的词在向量空间中距离更近
  - "猫" 和 "狗" 的向量距离 < "猫" 和 "桌子" 的向量距离

**④ 位置编码（Positional Encoding）**
- Transformer 原本不区分位置，需要显式注入位置信息
- 常见方法：
  - **Sinusoidal（正弦位置编码）**：原始 Transformer 使用
  - **RoPE（旋转位置编码）**：现代主流（LLaMA、Qwen、GLM）
  - **ALiBi**：早期尝试
- 作用：让模型知道 "我爱你" 和 "你爱我" 是不同的

**输出矩阵形状**
```
输入："你好世界"
输出矩阵形状：(4, 768)
- 4：序列长度（4个Token）
- 768：隐藏层维度（每个Token用768维向量表示）
```

### 2. LLM 输出层：隐藏状态 → 概率分布

**输出层的处理流程**

```
隐藏状态 → LM Head → Logits → Softmax → 概率分布 → 解码
```

**① 隐藏状态（Hidden States）**
- 来源：Decoder 最后一层的输出
- 形状：(序列长度, hidden_size)
- 包含：输入序列的所有语义信息
- 示例：
  ```
  输入："你好" (2个Token, hidden_size=768)
  隐藏状态：(2, 768) 的数值矩阵
  ```

**② LM Head（语言模型头）**
- 结构：全连接层（Linear Layer）
- 功能：将隐藏状态映射到词表大小
- 输出：Logits（未归一化的得分）
- 形状：(序列长度, vocab_size)
  ```
  示例：(3, 151936)
  - 3：3个位置的Token
  - 151936：词表大小（151936个可能的词）
  ```

**③ Logits（未归一化的得分）**
- 每个词对应一个得分
- 得分越高，成为下一个词的可能性越大
- 示例：
  ```
  第3个位置的 Logits: [2.0, 3.1, -1.7, ..., -1.7]
  - 对应词表中的每个词
  - "我" 的得分 = 2.0
  - "是" 的得分 = 3.1 (最高)
  ```

**④ 提取最后一个位置的 Logits**
- 预测下一个词只需要最后一个位置的 Logits
- 为什么？因为模型通过自回归方式，每次只预测一个新词
- 示例：
  ```
  完整 Logits: (3, 151936)
  最后一个: (151936,) 向量 [2.0, 3.1, -1.7, ...]
  ```

**⑤ Softmax：Logits → 概率分布**
- 功能：将 Logits 转换为概率（0-1之间，总和为1）
- 公式：
  ```
  P(x) = exp(x) / sum(exp(all))
  ```
- 示例：
  ```
  Logits: [2.0, 3.1, -1.7, ...]
  概率: [0.01, 0.03, 0.001, ...]
  总和: 1.0 (100%)
  ```
- 概率解释：
  - "我" 的概率 = 0.34 (34%)
  - "是" 的概率 = 0.01 (1%)
  - 其他词的概率总和 = 0.65 (65%)

**⑥ 解码（Decoding）**
- 根据概率分布选择下一个词
- 常见策略：
  - **贪婪搜索**：选择概率最高的词
  - **波束搜索**：保留多个候选序列
  - **Top-K 采样**：从概率最高的 K 个词中随机选择
  - **Top-P 采样**：从累积概率达到 P 的词中随机选择
- 示例（贪婪搜索）：
  ```
  概率最高: "我" (0.34)
  选择: "我"
  新输入: "你好我"
  继续预测下一个词...
  ```

### 3. TokenID 与词元的映射关系

**一一对应**
- 每个 Token ID 对应一个具体的词元
- 词表是固定的，训练和推理时使用同一个词表
- 示例：
  ```
  Token ID: 12345
  对应词元: "你"
  Token ID: 67890
  对应词元: "好"
  ```

**序列位置**
- 每个位置都有一个 Token ID
- 序列长度等于 Token 数量
- 示例（33个Token的序列）：
  ```
  位置0: Token ID 12345 → "你"
  位置1: Token ID 67890 → "好"
  位置2: Token ID 13579 → "世"
  ...
  位置32: Token ID 98765 → "。"
  ```

**为什么重要？**
- 训练时：通过 Token ID 找到对应的标签
- 推理时：通过概率找到最佳 Token ID，再转换为词元
- 调试时：可以查看模型具体预测了哪个词

### 4. 生成与解码过程

**自回归生成**
- 每次只预测一个词
- 将新词加入输入序列
- 重复直到遇到结束符（<EOS>）

**示例**
```
步骤1:
输入: "你"
输出: 概率分布 → 选择 "好"
新序列: "你好"

步骤2:
输入: "你好"
输出: 概率分布 → 选择 "世"
新序列: "你好世"

步骤3:
输入: "你好世"
输出: 概率分布 → 选择 "界"
新序列: "你好世界"

步骤4:
输入: "你好世界"
输出: 概率分布 → 选择 "<EOS>"
结束
```

**每个Token的计算成本**
- 每个Token生成都需要完整走一遍所有Transformer层
- 成本随序列长度线性增长
- 优化：KV Cache（缓存Key和Value，避免重复计算）
