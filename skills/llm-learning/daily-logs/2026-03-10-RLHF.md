## 核心知识点

### 1. RLHF 两阶段训练流程

**RLHF（Reinforcement Learning from Human Feedback，基于人类反馈的强化学习）** 是让大模型对齐人类价值观的关键技术，被广泛应用于 ChatGPT、Claude 等模型的训练。

RLHF 采用**两阶段**训练架构：

#### 阶段一：奖励模型（Reward Model）训练
- **目的**：训练一个能够评估模型回答质量的奖励模型
- **数据来源**：人类标注的偏好数据对（更喜欢的回答 vs 不太喜欢的回答）
- **输入**：Prompt + 候选回答
- **输出**：奖励分数（标量值）
- **训练方法**：使用对比学习或排序学习，让奖励模型学会区分优质回答和劣质回答

#### 阶段二：基于 PPO 的强化学习训练
- **目的**：使用奖励模型提供的奖励信号，优化语言模型的生成策略
- **算法**：PPO（Proximal Policy Optimization，近端策略优化）
- **核心机制**：
  - **策略模型（Policy Model）**：待训练的模型
  - **参考模型（Reference Model）**：冻结的旧版本模型，用于 KL 约束
  - **奖励模型（Reward Model）**：阶段一训练得到的模型，提供即时奖励
  - **价值模型（Value Model）**：用于评估长期回报（Advantage 计算）

#### 训练流程图

```
阶段一：奖励模型训练
┌─────────────────────────────────────────────────┐
│  Prompt 收集 → 候选回答生成 → 人工标注偏好      │
│           ↓                                    │
│  构建偏好对 → 训练奖励模型 RM                  │
└─────────────────────────────────────────────────┘
                    ↓
阶段二：强化学习训练
┌─────────────────────────────────────────────────┐
│  四个模型协作：                               │
│  - 策略模型：生成回答                         │
│  - 参考模型：计算 KL 约束                     │
│  - 奖励模型：提供奖励分数                     │
│  - 价值模型：估计长期回报                     │
│           ↓                                    │
│  PPO 算法优化策略模型参数                    │
└─────────────────────────────────────────────────┘
```

### 2. 奖励模型（RM）结构

奖励模型通常基于预训练的语言模型，在其基础上添加一个**奖励头（Reward Head）**。

#### 模型结构

```
输入层（Prompt + 回答）
    ↓
N 层 Transformer Decoder（共享底座）
    ↓
奖励头（Reward Head）
    ↓
输出：奖励分数（标量）
```

#### 关键特点

1. **共享底座**
   - 奖励模型通常与语言模型共享 N 层 Transformer Decoder
   - 仅最后一层不同（LM Head → Reward Head）
   - 复用预训练知识，减少训练成本

2. **标量输出**
   - 输出单个数值，表示该回答的质量
   - 范围可以是任意实数（通常归一化到特定区间）

3. **逐 Token 评分**
   - 可以对回答的每个 Token 进行奖励打分
   - 形状：（序列长度，1）
   - 实际使用时通常只取最后一个 Token（对应 `<EOS>`）的奖励分数作为整体奖励

4. **训练目标**
   - 让优质回答获得更高的奖励分数
   - 让劣质回答获得较低的奖励分数
   - 通过对比损失优化模型参数

#### 与语言模型的区别

| 特性 | 语言模型（LLM） | 奖励模型（RM） |
|------|---------------|---------------|
| **输出层** | LM Head（词表大小） | Reward Head（1 维） |
| **输出形状** | （seq_len, vocab_size） | （seq_len, 1） |
| **输出含义** | 每个位置的概率分布 | 每个位置的奖励分数 |
| **训练目标** | 预测下一个 Token | 评估回答质量 |
| **使用场景** | 生成文本 | 评分和排序 |

### 3. 奖励模型训练

#### 数据构建

1. **Prompt 收集**
   - 收集多样化的提示词
   - 覆盖不同主题、风格、难度
   - 确保数据分布均衡

2. **候选回答生成**
   - 使用 SFT 模型对每个 Prompt 生成多个候选回答（通常 2-4 个）
   - 采样参数可以多样化（温度、Top-P 等）
   - 保留生成时的概率信息

3. **人工标注**
   - 标注员对候选回答进行两两比较
   - 选出更喜欢的回答
   - 标注标准：准确性、有用性、安全性、流畅性等
   - 记录标注一致性（inter-annotator agreement）

4. **偏好对构建**
   - 构建三元组：(Prompt, 优质回答 y_w, 劣质回答 y_l)
   - 表示为：人类偏好 y_w 优于 y_l

#### Loss 设计

使用 **Bradley-Terry 模型**或类似的排序损失：

```
Loss = -log(σ(r(x, y_w) - r(x, y_l)))
```

其中：
- `x` 是 Prompt
- `y_w` 是优质回答
- `y_l` 是劣质回答
- `r(·)` 是奖励模型的输出
- `σ` 是 sigmoid 函数

**物理意义**：
- 计算优质回答和劣质回答的奖励分数差
- 通过 sigmoid 转换为概率
- 目标是让这个概率最大化（即优质回答得分更高）

#### 训练流程

```python
def train_reward_model(rm_model, dataloader, optimizer, epochs=3):
    for epoch in range(epochs):
        for batch in dataloader:
            # batch 包含：(prompt, chosen, rejected)
            prompt, chosen, rejected = batch

            # 计算奖励分数
            # shape: (batch_size, seq_len, 1)
            reward_chosen = rm_model(prompt, chosen)
            reward_rejected = rm_model(prompt, rejected)

            # 取最后一个 token 的奖励
            # shape: (batch_size, 1)
            reward_chosen = reward_chosen[:, -1, :]
            reward_rejected = reward_rejected[:, -1, :]

            # 计算损失
            # 优质回答应该比劣质回答得分更高
            loss = -F.logsigmoid(reward_chosen - reward_rejected).mean()

            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            # 打印统计信息
            accuracy = (reward_chosen > reward_rejected).float().mean()
            print(f"Loss: {loss.item():.4f}, Accuracy: {accuracy.item():.4f}")
```

#### 训练技巧

1. **数据增强**
   - 对同一 Prompt 生成多个候选回答
   - 构建多个偏好对
   - 增加数据多样性

2. **学习率调度**
   - 使用较小的学习率（如 1e-5）
   - 使用余弦退火调度
   - 避免过拟合

3. **正则化**
   - Dropout（通常 0.1-0.3）
   - 权重衰减（如 0.01）
   - 梯度裁剪（如 max_norm=1.0）

4. **评估指标**
   - 准确率：奖励模型正确区分偏好对的比例
   - AUC：ROC 曲线下面积
   - 相关系数：奖励分数与人类评分的相关性

## 代码示例

### 奖励模型定义

```python
import torch
import torch.nn as nn

class RewardModel(nn.Module):
    def __init__(self, base_model, hidden_size=768):
        super().__init__()
        # 冻结的底层 Transformer
        self.base_model = base_model
        for param in self.base_model.parameters():
            param.requires_grad = False

        # 替换输出层为奖励头
        # 输出维度从 vocab_size 变为 1
        self.reward_head = nn.Linear(hidden_size, 1)

    def forward(self, input_ids, attention_mask=None):
        """
        Args:
            input_ids: (batch_size, seq_len)
            attention_mask: (batch_size, seq_len)

        Returns:
            rewards: (batch_size, seq_len, 1)
        """
        # 获取隐藏状态
        outputs = self.base_model(
            input_ids=input_ids,
            attention_mask=attention_mask,
            output_hidden_states=True
        )

        # 获取最后一层的隐藏状态
        # shape: (batch_size, seq_len, hidden_size)
        hidden_states = outputs.hidden_states[-1]

        # 计算奖励
        # shape: (batch_size, seq_len, 1)
        rewards = self.reward_head(hidden_states)

        return rewards


# 使用示例
from transformers import AutoModelForCausalLM

# 加载基础模型
base_model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-7B")

# 创建奖励模型
rm_model = RewardModel(base_model, hidden_size=4096)

# 训练（只训练 reward_head）
optimizer = torch.optim.AdamW(
    rm_model.reward_head.parameters(),
    lr=1e-4
)
```

from torch.utils.data import Dataset, DataLoader

class PreferenceDataset(Dataset):
    def __init__(self, data, tokenizer, max_length=512):
        self.data = data
        self.tokenizer = tokenizer
        self.max_length = max_length

    def __len__(self):
        return len(self.data)

    def __getitem__(self, idx):
        item = self.data[idx]
        prompt = item['prompt']
        chosen = item['chosen']
        rejected = item['rejected']

        # Tokenize
        prompt_enc = self.tokenizer(
            prompt,
            max_length=self.max_length,
            truncation=True,
            padding=False,
            return_tensors=None
        )

        chosen_enc = self.tokenizer(
            prompt + chosen,
            max_length=self.max_length,
            truncation=True,
            padding=False,
            return_tensors=None
        )

        rejected_enc = self.tokenizer(
            prompt + rejected,
            max_length=self.max_length,
            truncation=True,
            padding=False,
            return_tensors=None
        )

        return {
            'prompt_input_ids': prompt_enc['input_ids'],
            'prompt_attention_mask': prompt_enc['attention_mask'],
            'chosen_input_ids': chosen_enc['input_ids'],
            'chosen_attention_mask': chosen_enc['attention_mask'],
            'rejected_input_ids': rejected_enc['input_ids'],
            'rejected_attention_mask': rejected_enc['attention_mask'],
        }

## 实践建议

### 1. 数据质量优先

- **标注一致性**：确保标注员对"优质"和"劣质"有一致的标准
- **标注指南**：制定清晰的标注指南和示例
- **质量控制**：定期检查标注质量，计算 inter-annotator agreement
- **数据清洗**：去除低质量、歧义、冲突的标注

### 2. 奖励模型评估

- **Held-out 测试集**：预留部分数据用于测试
- **准确率监控**：奖励模型在测试集上的准确率
- **偏差检测**：检查奖励模型是否对某些偏差敏感（如长度、风格偏好）
- **人类验证**：定期让人类评估奖励模型的评分

### 3. 避免奖励欺骗（Reward Hacking）

- **KL 散度约束**：使用参考模型约束策略模型的偏离
- **定期人工审查**：检查模型的生成结果，发现异常行为
- **奖励模型更新**：定期用新数据微调奖励模型
- **多奖励模型**：使用多个奖励模型组合，降低单一模型的偏见

### 4. 奖励模型漂移

- **分布对齐**：策略模型分布变化时，奖励模型可能需要更新
- **定期微调**：用新数据定期微调奖励模型
- **在线学习**：在 RL 训练过程中持续更新奖励模型
- **分布监控**：监控策略模型和参考模型之间的 KL 距离

## 常见问题

**Q: 为什么奖励模型只用最后一个 Token 的奖励？**
A: 因为最后一个 Token（`<EOS>`）包含了整个序列的语义信息。理论上可以使用所有 Token 奖励的平均或加权，但实践中最后一个 Token 已经足够。

**Q: 奖励模型的训练需要多少数据？**
A: 通常需要数万到数十万对偏好数据。数据量越大，奖励模型越稳健。但质量比数量更重要。

**Q: 奖励模型如何评估回答安全性？**
A: 需要在标注时包含安全性维度（如是否有害、是否歧视）。也可以训练独立的 safety reward model。

**Q: RLHF 和 SFT 的区别是什么？**
A: SFT 是有监督学习，需要标注好的回答；RLHF 是强化学习，只需要偏好标注（哪个回答更好），不需要标注最优回答。
