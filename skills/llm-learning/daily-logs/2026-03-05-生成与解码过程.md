## 核心知识点

### 1. LLM 生成与解码机制

**自回归生成过程**
```
输入: "你"
→ 模型计算: 得到下一个词的概率分布
→ 解码: 选择 "好"
→ 新输入: "你好"
→ 模型计算: 得到下一个词的概率分布
→ 解码: 选择 "世"
→ 新输入: "你好世"
→ 重复直到: 生成 <EOS> (End of Sequence)
```

**关键特性**
- **逐个生成**：每次只预测一个词（也有多Token并行生成的方案）
- **完整前向传播**：每个Token的生成都需要重新走一遍所有Transformer层
- **自回归依赖**：下一个词依赖于之前所有生成的词

**计算成本**
- 时间复杂度：O(序列长度 × 模型层数 × 隐藏层维度)
- 优化技术：KV Cache（缓存Key和Value，避免重复计算）

### 2. 贪婪搜索（Greedy Search）

**核心思想**
- 每次选择**概率最高**的一个Token
- 不考虑全局最优性或多样性

**示例**
```
步骤1:
概率分布: {"好": 0.4, "的": 0.3, "是": 0.2, ...}
选择: "好" (概率最高)

步骤2:
概率分布: {"世": 0.35, "界": 0.3, "上": 0.2, ...}
选择: "世" (概率最高)
```

**优点**
- 简单快速
- 计算成本低

**缺点**
- **缺乏多样性**：每次都生成相同的内容
- **局部最优**：早期选择可能影响后续，导致次优结果

**适用场景**
- 对多样性要求低的任务
- 需要确定性输出的场景
- 快速生成大量文本

---

### 3. 波束搜索（Beam Search）

**核心思想**
- 保留**多个候选序列**（称为Beam）
- 逐步扩展，最终选择得分最高的一条

**参数设置**
- `num_beams`: 波束数量，通常为 1-5
  - num_beams=1: 等同于贪婪搜索
  - num_beams=3: 保留3个候选序列
  - num_beams=5: 保留5个候选序列

**示例（num_beams=3）**
```
步骤1:
输入: "你"
候选1: "你好"   (得分: 0.4)
候选2: "你的"   (得分: 0.3)
候选3: "你是"   (得分: 0.2)

步骤2:
扩展候选1:
- "你好世" (得分: 0.35)
- "你好上" (得分: 0.25)
- "你好是" (得分: 0.15)

扩展候选2:
- "你的世" (得分: 0.28)
- "你的上" (得分: 0.18)
- ...

扩展候选3:
- "你是世" (得分: 0.18)
...

步骤3:
保留得分最高的3个序列:
1. "你好世" (0.35 × 0.4 = 0.14)
2. "你好上" (0.25 × 0.4 = 0.10)
3. "你的世" (0.28 × 0.3 = 0.084)

最终输出: "你好世"
```

**优点**
- 接近全局最优
- 质量优于贪婪搜索

**缺点**
- 计算成本高（与num_beams成正比）
- 可能产生重复内容
- 仍然缺乏多样性

**适用场景**
- 需要高质量生成的任务
- 翻译、摘要等确定性任务
- 不需要多样性的场景

---

### 4. Top-K 采样（Top-K Sampling）

**核心思想**
- 每次只从概率最高的**前K个词**中随机选择
- 避免选择极低概率的词

**参数设置**
- `top_k`: 通常为 50-100
  - top_k=0: 禁用Top-K（即选择全部词表）
  - top_k=50: 只从概率最高的50个词中选择

**示例（top_k=3）**
```
完整概率分布:
{"好": 0.4, "的": 0.3, "是": 0.2, "上": 0.05, "界": 0.03, ...}

Top-3候选:
1. "好" (0.4)
2. "的" (0.3)
3. "是" (0.2)

在这3个词中按概率随机采样:
- 40% 概率选择 "好"
- 30% 概率选择 "的"
- 20% 概率选择 "是"
```

**计算步骤**
1. 对所有词按概率降序排序
2. 保留前K个词
3. 在这K个词中重新归一化概率
4. 按新概率随机采样

**优点**
- **平衡质量和多样性**
- 避免选择极低概率的词
- 计算开销适中

**缺点**
- K值需要调优
- K太大：接近完全随机采样
- K太小：接近贪婪搜索

**适用场景**
- 需要多样性但不过度随机
- 创意写作、对话生成
- 推荐的默认解码策略

---

### 5. Top-P 采样（Top-P Sampling / Nucleus Sampling）

**核心思想**
- 动态选择候选词集合，使累积概率达到阈值P
- **自适应**：根据概率分布自动调整候选集大小

**参数设置**
- `top_p` (P): 通常为 0.9-0.95
  - top_p=0.9: 选择累积概率达到90%的最小词集合
  - top_p=0.95: 选择累积概率达到95%的最小词集合

**示例（top_p=0.9）**
```
概率分布（按概率降序）:
1. "好"   - 概率 0.40 (累积: 0.40)  ✓
2. "的"   - 概率 0.30 (累积: 0.70)  ✓
3. "是"   - 概率 0.15 (累积: 0.85)  ✓
4. "上"   - 概率 0.08 (累积: 0.93)  ✓ (超过0.9，但这是第一个超过的点，所以包含)
5. "界"   - 概率 0.03 (累积: 0.96)  ✗
...

候选集合: ["好", "的", "是", "上"]
在这些词中重新归一化概率并采样。
```

**与 Top-K 的区别**
| 特性 | Top-K | Top-P |
|------|-------|-------|
| 候选集大小 | 固定K | 动态 |
| 适应性 | 差 | 好 |
| 平坦分布 | 包含很多低概率词 | 自动扩展 |
| 陡峭分布 | 可能排除中概率词 | 自动收缩 |

**优点**
- **自适应性强**：根据概率分布自动调整
- 在平坦和陡峭分布下都表现良好
- 不需要精细调参

**缺点**
- 计算稍复杂（需要累积概率）

**适用场景**
- 通用场景的推荐策略
- 不确定概率分布特性的情况
- 与 Top-K 结合使用（同时设置两者）

## 代码示例

### 使用 HuggingFace 实现不同的解码策略

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "gpt2"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)

input_text = "今天天气"
input_ids = tokenizer(input_text, return_tensors="pt").input_ids

# 1. 贪婪搜索
output = model.generate(
    input_ids,
    max_length=20,
    do_sample=False,  # 禁用采样
)
print("贪婪搜索:", tokenizer.decode(output[0], skip_special_tokens=True))

# 2. 波束搜索（num_beams=3）
output = model.generate(
    input_ids,
    max_length=20,
    num_beams=3,
    do_sample=False,
    early_stopping=True,
)
print("波束搜索:", tokenizer.decode(output[0], skip_special_tokens=True))

# 3. Top-K 采样（top_k=50）
output = model.generate(
    input_ids,
    max_length=20,
    do_sample=True,
    top_k=50,
)
print("Top-K 采样:", tokenizer.decode(output[0], skip_special_tokens=True))

# 4. Top-P 采样（top_p=0.9）
output = model.generate(
    input_ids,
    max_length=20,
    do_sample=True,
    top_p=0.9,
)
print("Top-P 采样:", tokenizer.decode(output[0], skip_special_tokens=True))

# 5. 结合 Top-K 和 Top-P
output = model.generate(
    input_ids,
    max_length=20,
    do_sample=True,
    top_k=50,
    top_p=0.9,
)
print("Top-K + Top-P:", tokenizer.decode(output[0], skip_special_tokens=True))
```

### 温度系数（Temperature）

```python
# 温度系数控制概率分布的尖锐程度
# T < 1: 分布更尖锐，更确定性
# T = 1: 原始分布
# T > 1: 分布更平坦，更随机

output = model.generate(
    input_ids,
    max_length=20,
    do_sample=True,
    temperature=0.7,  # 0.7 常用，平衡确定性和多样性
    top_p=0.9,
)
print("温度0.7:", tokenizer.decode(output[0], skip_special_tokens=True))
```

### 解码策略选择指南

| 场景 | 推荐策略 | 参数 |
|------|----------|------|
| 确定性输出（翻译、摘要） | 波束搜索 | num_beams=3-5 |
| 创意写作 | Top-P 采样 | top_p=0.9, temperature=0.8 |
| 对话生成 | Top-K + Top-P | top_k=50, top_p=0.9, temperature=0.7 |
| 快速生成 | 贪婪搜索 | do_sample=False |
| 高质量生成 | 波束搜索 + 长度惩罚 | num_beams=5, length_penalty=1.2 |

### 调参技巧
1. **Top-K**：从 50 开始，根据结果调整
2. **Top-P**：从 0.9 开始，根据多样性需求调整
3. **温度**：从 0.7 开始
   - 太低：输出单调
   - 太高：输出混乱
4. **波束数量**：从 3 开始，超过 5 收益递减
