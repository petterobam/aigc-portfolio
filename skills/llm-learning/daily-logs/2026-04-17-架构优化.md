## 📚 核心知识点

### 1. Pre-norm vs Post-norm - 归一化位置选择

**Pre-norm（前归一化）**：
- 在子层（自注意力或前馈网络）**输入前**先执行 LayerNorm
- 然后经过子层并与原始输入进行残差连接
- **优势**：提升梯度流动、深层模型训练稳定性更好
- 现代大模型（如 GPT-3、LLaMA）普遍采用

**Post-norm（后归一化）**：
- 在子层输出后与原始输入进行残差连接
- **再执行** LayerNorm
- **问题**：在深层网络中可能导致梯度衰减和训练不稳定
- 原始 Transformer 采用的结构

**关键差异**：
```
Pre-norm:  x → LN → SubLayer → +x → Output
Post-norm: x → SubLayer → +x → LN → Output
```

---

### 2. BatchNorm vs LayerNorm - 归一化范围

**BatchNorm（批量归一化）**：
- 对每个小批量的**每个通道**进行标准化
- 操作：减去均值、除以标准差，再加上可学习的缩放和偏移参数
- **适用**：计算机视觉等通道语义较强的任务
- **问题**：在 NLP 中效果不稳定，受 batch size 影响大

**LayerNorm（层归一化）**：
- 对每个**样本**的所有特征维度进行标准化
- 同样使用可学习的缩放和偏移参数
- **适用**：NLP、Transformer 等序列模型
- **优势**：不受 batch size 影响，训练更稳定

**关键差异**：
```
BatchNorm:  对 (Batch, Channel, H, W) 的 Channel 维度归一化
LayerNorm:  对 (Batch, Seq, Hidden) 的 Hidden 维度归一化
```

---

### 3. RMSNorm - 高效归一化

**核心思想**：
- 基于输入特征的均方根值（RMS）进行规范化
- **不计算均值偏移**，仅使用标准差

**计算公式**：
```
RMS(x) = sqrt((1/d) * Σ(x_i²))
x̂ = (x / RMS(x)) × γ + β
```

**优势**：
- 计算更高效（省去均值计算）
- 在多种 Transformer 架构中表现相近或更优
- PaLM、LLaMA 等模型采用

**vs LayerNorm**：
```
LayerNorm:  x̂ = γ × (x - μ) / σ + β
RMSNorm:    x̂ = γ × (x / RMS(x)) + β
```

### 4. 模型剪枝（Pruning）

**核心概念**：
- 通过移除冗余或不重要的权重、神经元或通道
- 减少模型参数量和计算量

**典型流程**：
```
1. 评估重要性：计算每个权重/神经元的重要性指标
2. 标记待剪参数：按重要性排序，选择阈值剪枝
3. 重训练（Fine-tuning）：恢复剪枝后的模型性能
4. 验证性能：确保精度损失可接受
```

**剪枝策略**：
- **非结构化剪枝**：稀疏化权重，推理加速需要专用硬件
- **结构化剪枝**：整行/整列删除，推理加速更直接

**应用场景**：
- 移动端部署
- 边缘计算设备
- 降低推理成本

---

### 5. SwiGLU - 门控线性单元

**基本结构**：
```
SwiGLU(x) = (Wx) ⊗ SiLU(Vx)

其中：
- Wx: 第一条线性映射
- Vx: 第二条线性映射
- SiLU: Swish-1 激活函数
- ⊗: 逐元素相乘
```

**激活优势**：
- 用 SiLU（Swish-1）替换原 GLU 中的 sigmoid
- 门控更平滑、梯度更稳定
- 加速训练收敛
- 提升下游任务表现

**vs ReLU**：
```
ReLU:    max(0, x)           - 硬阈值，梯度在负区间为 0
SiLU:    x / (1 + e^(-x))    - 平滑、非单调、梯度处处非零
```

**应用**：
- LLaMA、PaLM、GLM 等主流模型采用
- 替代传统的 FFN（Feed-Forward Network）

## 💡 实践建议

### 选择归一化策略

**新模型训练**：
- 深度网络（>12 层）：优先选择 Pre-norm + RMSNorm
- 浅层网络：Post-norm + LayerNorm 也可行

**迁移学习**：
- 保持原模型的归一化方式
- 不要随意调整 Pre-norm/Post-norm 位置

### 优化推理性能

**剪枝应用场景**：
- 有明确预算限制时（如 7B → 4B）
- 推理成本敏感的场景
- 硬件资源受限的环境

**注意**：
- 非结构化剪枝需要推理引擎支持稀疏计算
- 结构化剪枝对推理加速更直接
- 剪枝后务必重新训练（Fine-tune）

### 激活函数选择

**推荐优先级**：
1. SwiGLU：当前 SOTA，效果最好
2. GELU：SwiGLU 的轻量版替代
3. ReLU：经典但性能略逊

## 🔍 深入思考

### 为什么现代大模型都采用 Pre-norm？

深层 Transformer 模型中，梯度需要流经多层网络。Post-norm 将 LayerNorm 放在残差连接之后，导致梯度在传播过程中被多次"归一化"，可能出现梯度消失或梯度爆炸。Pre-norm 将 LayerNorm 放在残差连接之前，梯度可以直接通过残差连接传播，更利于深度网络训练。

### RMSNorm 为何能替代 LayerNorm？

LayerNorm 计算均值和标准差两步，而 RMSNorm 只计算标准差（RMS）。在 Transformer 中，均值信息对归一化的贡献相对较小，RMSNorm 在保证性能的同时减少了计算量。此外，RMSNorm 的归一化操作更稳定，数值稳定性更好。

### SwiGLU 的门控机制优势何在？

传统的 FFN 使用单一路径 + ReLU 激活。SwiGLU 通过门控机制（Gated Linear Unit）引入非线性，让模型能够动态地控制信息流。SiLU 的平滑特性使得梯度更稳定，训练更容易收敛。这种设计类似于 LSTM 的门控机制，让模型能够选择性地"记住"或"遗忘"信息。
