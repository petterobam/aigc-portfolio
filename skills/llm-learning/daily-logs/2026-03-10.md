## 核心知识点

### 1. Prefix-Tuning（前缀微调）

**核心思想：**
Prefix-Tuning 由斯坦福大学提出，是一种轻量级的微调方法。它在输入序列的**起始位置插入一段连续的、可训练的向量**（称为前缀 Prefix），Transformer 会将这些向量视为"虚拟 Token"并参与 Attention 计算。

**工作原理：**
- 在每层 Transformer 的前缀位置插入可训练的向量序列
- 这些向量通过调整注意力机制来引导模型输出
- 只训练这些前缀向量，保持模型原有参数冻结

**优势：**
- 参数量极少（通常占总参数的 0.1%-1%）
- 不会破坏预训练模型的知识
- 适合生成式任务，特别是需要上下文控制的场景

### 2. Adapter Tuning（适配器微调）

**核心思想：**
Adapter Tuning 在 Transformer 层之间插入小型神经网络模块（适配器），只训练这些新增模块，而冻结原始模型参数。

**适配器结构：**
典型的适配器由两部分组成：
- **降投影层**：将高维特征降维到低维（如 768 → 64）
- **升投影层**：将低维特征升维回原维度（如 64 → 768）
- 中间通常使用非线性激活函数（如 ReLU）

**参数效率：**
- 每个适配器只有约 4% 的原始层参数量
- 增加的计算开销很小
- 可以针对不同任务训练不同的适配器

**插入位置：**
通常插入在每个 Transformer 层之后、LayerNorm 之前

---

### 3. Prompt Tuning（提示微调）

**核心思想：**
Prompt Tuning 是一种更激进的轻量级方法，它直接优化输入层的虚拟 Token 嵌入，而不修改任何 Transformer 层的参数。

**与 Prefix-Tuning 的区别：**
- **Prompt Tuning**：只在输入层优化，更简洁
- **Prefix-Tuning**：在每一层都插入前缀向量，表达能力更强

**发展历程：**
- **Hard Prompt**：手动设计文本提示（如"请详细回答：..."）
- **Soft Prompt**：学习连续的向量表示，无需人工设计

**适用场景：**
- 生成式任务
- 需要快速适应新任务
- 存储和计算资源受限

---

### 4. 微调方法对比

| 方法 | 参数修改位置 | 参数量 | 性能 | 适用场景 |
|------|------------|--------|------|---------|
| **全参数微调** | 所有层 | 100% | 最佳 | 充分资源，需要最大性能 |
| **LoRA** | 部分层（低秩分解） | 0.1%-1% | 接近全量 | 通用场景，性价比高 |
| **Prefix-Tuning** | 每层前缀 | 0.1%-1% | 生成任务优 | 上下文控制，风格迁移 |
| **Adapter Tuning** | 层间模块 | 3%-5% | 接近全量 | 多任务，参数隔离 |
| **Prompt Tuning** | 仅输入层 | <0.1% | 简单任务够用 | 快速适应，推理时改 |

**选择建议：**
- **通用场景**：首选 LoRA（性价比最高）
- **多任务场景**：Adapter Tuning（参数隔离清晰）
- **风格控制/生成任务**：Prefix-Tuning（对上下文控制强）
- **极低资源**：Prompt Tuning（参数最少）

## 代码示例

### Prefix-Tuning 简化实现

```python
import torch
import torch.nn as nn

class PrefixTuning(nn.Module):
    def __init__(self, hidden_size, prefix_length=10, n_layers=12):
        super().__init__()
        self.prefix_length = prefix_length
        self.n_layers = n_layers

        # 为每层创建前缀向量
        self.prefix_embeddings = nn.Parameter(
            torch.randn(n_layers, prefix_length, hidden_size)
        )

    def forward(self, hidden_states, layer_idx):
        """
        hidden_states: (batch, seq_len, hidden_size)
        layer_idx: 当前层的索引
        """
        # 获取当前层的前缀
        prefix = self.prefix_embeddings[layer_idx]  # (prefix_length, hidden_size)

        # 扩展到 batch 维度
        batch_size = hidden_states.size(0)
        prefix = prefix.unsqueeze(0).expand(batch_size, -1, -1)

        # 将前缀拼接到输入前面
        hidden_states = torch.cat([prefix, hidden_states], dim=1)

        return hidden_states

# 使用示例
prefix_tuning = PrefixTuning(hidden_size=768, prefix_length=10, n_layers=12)
hidden_states = torch.randn(2, 20, 768)  # batch=2, seq_len=20

# 在每层应用前缀
for layer_idx in range(12):
    hidden_states = prefix_tuning(hidden_states, layer_idx)
    # ... 然后传入 Transformer 层
```

### Adapter 模块实现

```python
class Adapter(nn.Module):
    def __init__(self, hidden_size, bottleneck_size=64):
        super().__init__()
        self.down_proj = nn.Linear(hidden_size, bottleneck_size)
        self.up_proj = nn.Linear(bottleneck_size, hidden_size)
        self.activation = nn.ReLU()
        self.layer_norm = nn.LayerNorm(hidden_size)

    def forward(self, x):
        # x: (batch, seq_len, hidden_size)
        residual = x
        x = self.layer_norm(x)
        x = self.down_proj(x)
        x = self.activation(x)
        x = self.up_proj(x)
        return residual + x  # 残差连接

# 插入到 Transformer 层
class TransformerLayerWithAdapter(nn.Module):
    def __init__(self, base_layer, adapter):
        super().__init__()
        self.base_layer = base_layer
        self.adapter = adapter

    def forward(self, x):
        x = self.base_layer(x)
        x = self.adapter(x)  # 适配器
        return x
```

### Prompt Tuning 实现

```python
class PromptTuning(nn.Module):
    def __init__(self, vocab_size, hidden_size, prompt_length=10):
        super().__init__()
        self.prompt_length = prompt_length

        # 直接优化输入层的虚拟 Token
        self.prompt_embeddings = nn.Parameter(
            torch.randn(prompt_length, hidden_size)
        )

    def forward(self, input_ids, embed_layer):
        """
        input_ids: (batch, seq_len)
        embed_layer: 原模型的 embedding 层
        """
        # 原始输入的嵌入
        original_embeds = embed_layer(input_ids)  # (batch, seq_len, hidden_size)

        # 获取前缀嵌入
        batch_size = input_ids.size(0)
        prompt_embeds = self.prompt_embeddings.unsqueeze(0).expand(
            batch_size, -1, -1
        )  # (batch, prompt_length, hidden_size)

        # 拼接
        final_embeds = torch.cat([prompt_embeds, original_embeds], dim=1)

        return final_embeds
```

## 实践建议

### 1. 何时选择 Prefix-Tuning？
- ✅ 需要控制生成风格（如正式/口语化）
- ✅ 需要上下文引导（如摘要长度控制）
- ✅ 模型参数量较大，训练资源有限
- ❌ 需要修改深层语义知识（这种情况建议用 LoRA）

### 2. 何时选择 Adapter Tuning？
- ✅ 需要多任务学习（为每个任务训练独立适配器）
- ✅ 需要参数隔离（防止任务间干扰）
- ✅ 计算资源适中，可以接受 3-5% 的参数增量
- ❌ 推理时需要切换适配器（会增加一点推理开销）

### 3. 何时选择 Prompt Tuning？
- ✅ 极端资源受限（参数量最小）
- ✅ 快速适应新任务（几分钟训练）
- ✅ 任务相对简单（不需要深层语义修改）
- ❌ 复杂推理任务（能力可能不足）

### 4. 训练技巧
- **学习率**：轻量级方法可以使用稍高的学习率（1e-3 ~ 1e-4）
- **训练步数**：通常比全参数微调需要更多步数才能收敛
- **前缀长度**：Prefix-Tuning 中，10-50 个 token 通常足够
- **瓶颈维度**：Adapter 中，64-128 维通常够用

### 5. 性能调优
- **多前缀设计**：可以为不同类型的任务训练不同的前缀
- **层次化适配器**：在不同层使用不同大小的适配器
- **渐进式训练**：先冻结训练，再解冻部分参数微调

## 实际应用案例

### 案例 1：风格迁移
使用 Prefix-Tuning 控制输出风格：
- **正式风格**前缀 → 生成正式、规范的回答
- **口语风格**前缀 → 生成轻松、口语化的回答
- 同一个基础模型，通过切换前缀实现不同风格

### 案例 2：领域适应
使用 Adapter Tuning 为不同领域微调：
- 医疗领域适配器
- 法律领域适配器
- 金融领域适配器
- 推理时根据用户问题选择对应适配器

### 案例 3：快速实验
使用 Prompt Tuning 快速验证新任务：
- 几分钟内训练出可用模型
- 用于 A/B 测试不同提示策略
- 为后续 LoRA 微调提供超参数参考
