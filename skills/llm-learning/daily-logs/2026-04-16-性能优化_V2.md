## 📚 核心知识点

### 1. 位置编码技术

#### ALiBi（Attention with Linear Biases）
- **现状**：已被 RoPE 主流取代，逐渐被淘汰
- **原理**：通过在注意力分数上添加一个与距离相关的线性偏置来编码位置信息
- **缺点**：对长序列的扩展性较差

#### RoPE（Rotary Position Embedding）
- **提出者**：苏剑林
- **核心思想**：对 Query 和 Key 应用旋转变换，将位置信息编码为向量的相位差
- **关键优势**：
  - 天然捕捉相对位置信息
  - 无需额外参数
  - 计算开销低
  - 长序列场景下泛化性好

**维度作用机制：**
- **高频维度**：旋转角度随位置变化快，捕捉短距离局部相对位置
- **低频维度**：旋转角度随位置变化慢，捕捉长距离全局依赖

### 2. 量化（Quantization）

**数值类型对比：**
| 类型 | 总位数 | 特点 | 适用场景 |
|------|--------|------|----------|
| FP32 | 32 | 标准浮点，精度高 | 训练 |
| BF16 | 16 | 8位指数，精度适中 | 训练/推理 |
| FP16 | 16 | 5位指数，范围小 | 推理 |
| INT8 | 8 | 整数，压缩率4x | 推理 |
| INT4 | 4 | 整数，压缩率8x | 极致压缩 |

**量化策略：**
- 训练时：FP32/TF32（保持梯度精度）
- 推理时：FP16/INT8/INT4（减少显存、加速计算）
- 权重量化：显著降低显存占用
- 激活量化：需要谨慎，可能影响精度

### 3. 注意力机制优化

#### MHA（Multi-Head Attention）
- **结构**：每个注意力头都有独立的 Q、K、V 投影矩阵
- **特点**：参数量大，显存占用高
- **优势**：表达能力最强

#### MQA（Multi-Query Attention）
- **结构**：所有头共享 Key 和 Value 投影，仅 Query 独立
- **参数量**：大幅减少（约 1/3）
- **适用**：推理场景

#### GQA（Grouped-Query Attention）
- **结构**：Key 和 Value 按组共享（如每 2-4 个头共享一组）
- **平衡点**：在 MHA 和 MQA 之间
- **应用**：Llama 2、Qwen 等主流模型

#### MLA（Multi-head Latent Attention）
- **结构**：通过低秩分解进一步压缩注意力
- **创新**：DeepSeek V2/V3 的核心技术
- **效果**：极致的参数压缩和推理加速

**对比总结：**
- MHA：参数最大，精度最高，最慢
- MQA：参数最小，精度损失最大，最快
- GQA：折中方案，当前主流选择
- MLA：极端压缩，适合超大模型

### 4. 训练优化技术

#### 梯度累积（Gradient Accumulation）
- **原理**：多个 batch 分别计算梯度，累加后统一更新
- **等效**：实现更大 batch size
- **场景**：显存受限但需要大 batch 训练
- **代价**：训练时间略有增加

#### Gradient Checkpoint（梯度重计算）
- **原理**：只保存关键层激活值，反向时重新计算中间层
- **优势**：大幅减少显存占用（约 30-50%）
- **代价**：计算量增加约 20-30%
- **适用**：训练超大模型时显存不足

#### Full Recomputation（完全重计算）
- **原理**：反向时不保留任何激活值，全部重新计算
- **优势**：显存占用最低
- **代价**：计算量大幅增加
- **适用**：极端显存受限场景

### 5. 推理加速技术

#### KV Cache
- **原理**：缓存历史 token 的 Key 和 Value，避免重复计算
- **收益**：生成每个新 token 的计算复杂度从 O(n²) 降到 O(n)
- **代价**：显存占用随序列长度线性增长
- **优化**：结合量化进一步压缩 KV Cache

#### Continuous Batching（连续批处理）
- **原理**：在推理过程中动态添加/移除请求，充分利用 GPU
- **收益**：提升吞吐量 2-4x
- **适用**：高并发在线推理场景
- **代表框架**：vLLM、TensorRT-LLM

#### FlashAttention
- **原理**：通过算子融合和分块计算减少 HBM 访问
- **收益**：速度提升 2-4x，显存占用减半
- **技术**：IO 感识、分块计算、异步传输
- **版本演进**：FlashAttention-1 → FlashAttention-2 → FlashAttention-3

## 💡 实践建议

### 选择位置编码
- **新模型**：首选 RoPE（主流、稳定、长序列友好）
- **兼容性**：考虑与现有生态的兼容（如 HuggingFace 支持）

### 量化策略
```python
# 推理时的典型配置
load_in_8bit=True  # INT8 量化，平衡精度和速度
# 或
load_in_4bit=True   # INT4 量化，极致压缩
```

### 注意力优化选择
- **通用场景**：GQA（平衡精度和效率）
- **推理优先**：MQA（最快）
- **精度优先**：MHA（最准确）
- **超大模型**：MLA（DeepSeek 风格）

### 训练优化组合
```python
# 显存不足时的推荐配置
gradient_checkpointing=True  # 节省显存
gradient_accumulation_steps=4  # 等效大 batch
bf16=True  # 混合精度训练
```

### 推理加速栈
```
量化 + KV Cache + GQA + FlashAttention + Continuous Batching
= 极致的推理性能
```
