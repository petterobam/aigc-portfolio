```javascript
// 1. 调用数据抓取脚本
execSync('node scripts/fetch-story-list-chrome-v3.js')

// 2. 读取最新数据和昨日数据
const latestData = JSON.parse(fs.readFileSync(latestJsonFile));
const yesterdayData = JSON.parse(fs.readFileSync(yesterdayJsonFile));

// 3. 生成对比报告
const comparisonReport = {
  timestamp: new Date().toISOString(),
  summary: { /* 总体统计 */ },
  dailyChange: { /* 每日变化 */ },
  topStories: [ /* TOP 10 高阅读作品 */ ],
  zeroReadStories: [ /* 零阅读作品列表 */ ]
};

// 4. 保存报告
fs.writeFileSync(reportFile, JSON.stringify(comparisonReport, null, 2));
```

### 数据对比逻辑
- 按预览ID (`previewId`) 匹配昨日数据
- 计算阅读量变化：`readDiff = 最新阅读量 - 昨日阅读量`
- 计算总阅读量变化：`readChange = 最新总阅读量 - 昨日总阅读量`
- 计算新增故事数：`newStories = 最新故事数 - 昨日故事数`

## 经验总结

### 成功经验
1. **模块化设计**：数据抓取和数据监控分离，便于维护
2. **结构化报告**：JSON 格式便于后续处理和分析
3. **自动化流程**：一键执行，自动生成报告
4. **数据对比**：对比昨日数据，发现变化趋势

### 遇到的问题
1. **浏览器等待时间过长**：`fetch-story-list-chrome-v3.js` 等待 5 分钟才关闭浏览器
2. **时间间隔太短**：6 分钟内数据没有更新，无法验证每日变化检测功能

### 解决方案
1. **优化关闭逻辑**：在 `fetch-story-list-chrome-v3.js` 中，抓取完成后立即关闭浏览器
2. **增加执行间隔**：每天执行一次，确保有足够的时间让数据更新
