### 3. 自迭代优化机制是实现"自迭代"的关键

**自迭代优化闭环**:
```
数据采集 → 数据分析 → 优化建议 → 执行优化 → 验证效果 → 更新方法论
   ↓          ↓          ↓          ↓          ↓          ↓
  每天       每天       每天       每天/人工    3天       每周
```

**四层优化机制**:
- Level 1: 标题优化(自动执行,低风险,提升30-50%阅读量)
- Level 2: 标签优化(自动执行,低风险,提升20-30%阅读量)
- Level 3: 内容优化(提醒人工,中风险,提升50-100%阅读量)
- Level 4: 题材转换(提醒人工,高风险,提升100-200%阅读量)

**自迭代优化的核心优势**:
- 基于数据驱动决策,而非主观判断
- 自动识别低表现作品并生成优化建议
- 自动执行低风险优化(如标题优化)
- 提醒人工执行高风险优化(如内容优化)
- 记录优化效果并更新方法论

### 3. 设计7个定时任务

| 任务编号 | 任务名称 | 执行时间 | 执行命令 | 功能 |
|---------|---------|---------|---------|------|
| 任务1 | login-checker | 每天 8:00 | npm run login-checker | 检查登录状态,发送登录提醒 |
| 任务2 | fanqie-data-fetcher | 每天 21:00 | npm run fetch-data | 采集番茄小说已发布作品数据 |
| 任务3 | data-comparator | 每天 21:30 | npm run data-comparator | 对比当前数据和历史数据,识别数据变化 |
| 任务4 | dashboard-updater | 每天 22:00 | npm run dashboard-updater | 读取最新数据,自动更新数据追踪看板 |
| 任务5 | weekly-review | 每周一 8:30 | 触发番茄数据复盘 | 生成周复盘报告 |
| 任务6 | monthly-review | 每月1号 9:00 | 触发番茄数据复盘(月度) | 生成月复盘报告 |
| 任务7 | publish-reminder | 每周三、周日 20:00 | 触发发布提醒 | 提醒今天是发布日,检查发布计划 |

**任务依赖关系**:
- 任务2(数据采集) → 任务3(数据对比) → 任务4(数据看板更新)
- 任务4(数据看板更新) → 任务5(周复盘报告) / 任务6(月复盘报告)

### 4. 设计4层优化机制

| 优化等级 | 优化类型 | 执行方式 | 风险等级 | 预期效果 |
|---------|---------|---------|---------|---------|
| Level 1 | 标题优化 | 自动执行 | 低风险 | 提升 30-50% 阅读量 |
| Level 2 | 标签优化 | 自动执行 | 低风险 | 提升 20-30% 阅读量 |
| Level 3 | 内容优化 | 提醒人工 | 中风险 | 提升 50-100% 阅读量 |
| Level 4 | 题材转换 | 提醒人工 | 高风险 | 提升 100-200% 阅读量 |

**优化触发条件**:
- Level 1: 阅读量 < 5, 标题长度不在10-15字范围,标题没有金手指,标题没有爽点
- Level 2: 阅读量 < 10, 标签数量 < 3个, 标签与内容不符
- Level 3: 阅读量 < 5, 完读率 < 30%, 爽点密度 < 1/1000字
- Level 4: 阅读量 < 2, 题材属于低表现题材, 金手指属于低表现金手指
