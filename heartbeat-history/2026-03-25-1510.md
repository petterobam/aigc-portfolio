## 🎯 技能特性

### 核心功能

1. **日志分类**:
   - LEARNINGS.md - 学习、修正、知识差距、最佳实践
   - ERRORS.md - 命令失败、异常、意外输出
   - FEATURE_REQUESTS.md - 用户请求的功能

2. **自动提升**:
   - 编码代理可以处理这些学习日志为修复
   - 重要学习提升到项目记忆（CLAUDE.md, AGENTS.md, .github/copilot-instructions.md）

3. **重复模式检测**:
   - 检测相似的学习条目
   - 链接相关条目
   - 提升优先级
   - 考虑系统性修复

4. **简化与加固反馈**:
   - 从 `simplify_and_harden` 技能摄入重复模式
   - 将重复模式转化为持久的提示指导
   - 将持久的规则写入代理上下文/系统提示文件

### 多代理支持

**Claude Code**:
- 激活方式：Hooks (UserPromptSubmit, PostToolUse)
- 设置：`.claude/settings.json` 中配置 hooks
- 检测：自动通过 hook 脚本

**Cursor CLI**:
- 激活方式：Hooks（与 Claude Code 相同）
- 设置：`.cursor/settings.json` 中配置 hooks
- 检测：自动通过 hook 脚本

**GitHub Copilot**:
- 激活方式：手动（无 hook 支持）
- 设置：添加到 `.github/copilot-instructions.md`
- 检测：手动审查会话结束

**OpenClaw**:
- 激活方式：工作区注入 + 代理间消息传递
- 设置：工作区自动加载技能
- 检测：会话工具和会话历史

## 💡 使用建议

### 立即使用（安装后）

1. **记录学习**: 当遇到以下情况时，记录到 `.learnings/LEARNINGS.md`
   - 发现非显而易见的解决方案
   - 用户纠正你的错误
   - 发现更好的方法
   - 学习项目约定

2. **记录错误**: 当遇到以下情况时，记录到 `.learnings/ERRORS.md`
   - 命令或操作意外失败
   - 异常或堆栈跟踪
   - 超时或连接失败
   - 意外的输出或行为

3. **记录功能请求**: 当遇到以下情况时，记录到 `.learnings/FEATURE_REQUESTS.md`
   - 用户请求功能："你能不能..."
   - 用户希望有功能："我希望..."
   - 用户问为什么不能做某事："为什么你不能..."

### 定期审查（每周）

1. **审查待处理条目**:
   ```bash
   # 统计待处理条目数
   grep -h "Status\*\*: pending" .learnings/*.md | wc -l
   
   # 列出高优先级待处理条目
   grep -B5 "Priority\*\*: high" .learnings/*.md | grep "^##"
   ```

2. **解决已修复条目**:
   - 更新 `**Status**: pending` → `**Status**: resolved`
   - 提升适用学习到项目记忆

3. **提升适用学习**:
   - 学习适用于多个文件/功能
   - 任何贡献者（人类或 AI）都应该知道
   - 防止重复错误

### 优先级指南

| 优先级 | 何时使用 |
|--------|----------|
| `critical` | 阻塞核心功能、数据丢失风险、安全问题 |
| `high` | 显著影响、影响常见工作流、重复问题 |
| `medium` | 中等影响、存在变通方法 |
| `low` | 轻微不便、边缘情况、锦上添花 |
