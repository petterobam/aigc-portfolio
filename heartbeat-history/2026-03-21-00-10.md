### 1. 调研番茄小说登录和认证机制

**调研方式**：
- 访问番茄小说官网：https://fanqienovel.com
- 访问番茄小说登录页面：https://fanqienovel.com/main/writer/login
- 分析页面结构和登录方式

**调研发现**：

#### 番茄小说官网概览
- 主要板块：作家福利、作家专区、版权专区、最近更新
- 作家专区需要登录才能访问
- 点击"作家专区"会自动跳转到登录页面

#### 登录方式推测（4种可能）

**方式A：手机号+验证码登录**（最可能）
- 流程：输入手机号 → 获取验证码 → 输入验证码 → 登录成功
- 特点：无需密码、安全性高、适合移动端
- 自动化挑战：需要接收短信验证码、验证码有效期短
- 自动化方案：Cookie持久化、短信网关服务

**方式B：账号密码登录**
- 流程：输入手机号/邮箱 → 输入密码 → 登录成功
- 特点：需要预先注册账号、安全性中等、适合PC端
- 自动化挑战：需要预先注册账号、密码可能被加密
- 自动化方案：Cookie持久化、账号密码自动登录

**方式C：第三方登录**
- 流程：点击第三方登录按钮（微信、QQ、抖音等）→ 授权登录 → 登录成功
- 特点：依赖第三方平台、安全性高、适合移动端
- 自动化挑战：需要第三方账号授权、可能需要扫码
- 自动化方案：Cookie持久化、第三方登录API

**方式D：二维码登录**（PC端可能）
- 流程：手机扫码 → 手机确认登录 → PC端登录成功
- 特点：适合PC端、安全性高、需要手机配合
- 自动化挑战：需要手动扫码、无法完全自动化
- 自动化方案：Cookie持久化、用户扫码后保存Cookie

#### 作者认证流程推测（3种可能）

**方案A：简单认证**（最可能）
- 流程：登录账号 → 进入作家专区 → 填写作者信息 → 提交认证申请 → 即时认证成功
- 特点：流程简单、即时认证、无需审核
- 自动化可行性：✅ 高可行性

**方案B：实名认证**
- 流程：登录账号 → 进入作家专区 → 填写作者信息 → 上传身份证照片 → 等待审核 → 认证成功
- 特点：流程中等、需要审核、需要身份证
- 自动化可行性：⚠️ 中等可行性（需要上传身份证照片）

**方案C：作品认证**
- 流程：登录账号 → 进入作家专区 → 创建第一部作品 → 发布作品（需要审核） → 作品审核通过后，自动认证为作者
- 特点：流程复杂、需要审核、依赖作品质量
- 自动化可行性：✅ 高可行性（可以自动发布作品）

#### 登录状态维护机制（3种机制）

**Cookie机制**
- Cookie组成：Session ID、登录时间、用户ID、认证状态
- Cookie有效期：默认7-30天
- 自动化方案：使用Puppeteer的`page.cookies()`保存Cookie、`page.setCookies()`加载Cookie、定期更新Cookie

**Token机制**
- Token类型：Access Token（短期，如2小时）、Refresh Token（长期，如30天）
- Token更新机制：Access Token过期后，使用Refresh Token获取新的Access Token
- 自动化方案：保存Access Token和Refresh Token、定期刷新Access Token、Refresh Token过期后重新登录

**API登录**
- API登录流程：发送登录请求（POST）→ 服务器返回Session ID或Token → 后续请求携带Session ID或Token
- 自动化方案：使用HTTP请求库（axios、fetch）、保存Session ID或Token、定期刷新Token
