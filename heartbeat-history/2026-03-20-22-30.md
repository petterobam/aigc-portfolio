### ✅ 任务 3: 研究 OpenClaw 浏览器配置系统

**核心发现**:

1. **OpenClaw 支持三种浏览器 profile driver**:
   - `openclaw`: OpenClaw 管理的专用浏览器（默认）
   - `existing-session`: 现有的 Chrome 会话（使用 Chrome DevTools MCP）
   - `chrome-relay`: 扩展中继（需要安装扩展）

2. **内置 profile**:
   - `openclaw`: stopped（默认）
   - `user`: stopped（existing-session, transport: chrome-mcp）
   - `chrome-relay`: running（extension）

3. **Chrome DevTools MCP 集成**:
   - OpenClaw 使用官方的 Chrome DevTools MCP 服务器连接到正在运行的 Chrome
   - 这重用了已经在该 Chrome 配置文件中打开的标签页和登录状态
   - 不需要设置 CDP URL，因为 Chrome DevTools MCP 使用 `--autoConnect` 流程

4. **启用 Chrome 远程调试的步骤**:
   - 打开 `chrome://inspect/#remote-debugging`
   - 启用远程调试
   - 保持 Chrome 运行并在 OpenClaw 连接时批准连接提示

### ✅ 任务 4: 发现架构差异

**Playwright MCP vs OpenClaw 浏览器系统**:

| 特性 | Playwright MCP | OpenClaw 浏览器系统 |
|------|----------------|---------------------|
| 配置方式 | MCP 配置文件 | OpenClaw 配置文件 |
| 连接方式 | 直接 CDP 或 CDP endpoint | Chrome DevTools MCP (user profile) |
| 长连接支持 | --extension 模式不支持 | chrome-mcp transport 应该支持 |
| 浏览器控制 | 专用浏览器实例 | 复用现有 Chrome 会话 |

**关键洞察**:
- **问题根源**: Skill 文档中使用的是 Playwright MCP (`@playwright/mcp@latest --extension`)
- **解决方案**: OpenClaw 自带的浏览器系统使用 Chrome DevTools MCP (`chrome-mcp` transport)
- **无需修改 MCP 配置**: 直接使用 OpenClaw 的浏览器配置即可
