### 2. 定时任务配置说明

**编辑 crontab**:
```bash
crontab -e
```

**添加定时任务**（每6小时检查一次）:
```cron
# 每6小时检查一次登录状态
0 */6 * * * cd ~/.openclaw/workspace/番茄短篇故事集 && cd "🛠️ 自动化系统" && node scripts/login-checker.mjs >> logs/cron.log 2>&1
```

**定时任务说明**:
- `0 */6 * * *`: 每6小时的第0分钟执行（00:00, 06:00, 12:00, 18:00, 00:00）
- `cd ~/.openclaw/workspace/番茄短篇故事集 && cd "🛠️ 自动化系统"`: 切换到自动化系统目录
- `node scripts/login-checker.mjs`: 执行登录检查脚本
- `>> logs/cron.log 2>&1`: 将标准输出和标准错误重定向到日志文件

**查看 crontab**:
```bash
crontab -l
```

**删除定时任务**:
```bash

# 删除对应的行，保存退出
```

## 🔧 技术细节

### 登录管理器技术细节

**核心功能**:
- 检查 Cookies 文件是否存在
- 计算 Cookies 文件年龄（天数）
- 启动 Playwright 浏览器（headless 模式）
- 加载 Cookies 到浏览器上下文
- 导航到目标页面（短故事管理页面）
- 检查页面是否包含用户名（验证登录状态）
- 判断是否需要登录
- 判断是否需要登录提醒

**错误处理**:
- Cookies 文件不存在 → 返回 isLoggedIn: false
- Playwright 启动失败 → 返回 isLoggedIn: false
- 页面加载失败 → 返回 isLoggedIn: false
- 其他错误 → 记录错误日志，返回 isLoggedIn: false

---

### 定时登录检查器技术细节

**核心功能**:
- 执行登录管理器检查
- 解析登录管理器输出（stdout）
- 判断登录状态（通过 "已登录" / "未登录" 关键词）
- 判断是否需要登录（通过 "需要登录" 关键词）
- 记录登录检查历史（timestamp、isLoggedIn、needsLogin、output）
- 生成登录提醒（如果需要登录）
- 返回退出码（0 或 1）

**登录历史记录管理**:
- 只保留最近 100 条记录（避免日志文件过大）
- 使用 JSON 格式存储
- 支持使用 jq 命令进行数据分析
