## 技术分析

### Cookie 持久化原理

**为什么有效？**
1. Chrome 浏览器使用 Cookie 存储登录状态
2. Cookie 包含 session token、auth token 等关键信息
3. Playwright 可以加载和保存 Cookie
4. 加载 Cookie 后，浏览器会认为是已登录状态

**技术实现**：
```javascript
// 保存 Cookie
const cookies = await context.cookies();
fs.writeFileSync('cookies.json', JSON.stringify(cookies, null, 2));

// 加载 Cookie
const cookies = JSON.parse(fs.readFileSync('cookies.json'));
await context.addCookies(cookies);
```

**登录状态检测**：
```javascript
// 检查是否显示登录页面（二维码）
const hasQrCode = await page.locator('.slogin-qrcode-scan-page').count() > 0;

// 检查是否已登录（查找用户头像或用户名）
const hasUserInfo = await page.locator('.muye-header-user').count() > 0;

// 如果没有显示登录页面且有用户信息，说明已登录
return !hasQrCode && hasUserInfo;
```

### Cookie 文件管理

**文件命名**：
- 格式：`fanqie-cookies-YYYY-MM-DDTHH-mm-ss.json`
- 示例：`fanqie-cookies-2026-03-19T13-00-00.json`

**文件内容**：
```json
[
  {
    "name": "session_token",
    "value": "xxx",
    "domain": ".fanqienovel.com",
    "path": "/",
    "expires": 1710888000,
    "httpOnly": true,
    "secure": true,
    "sameSite": "Lax"
  },
  ...
]
```

**Cookie 过滤**：
- 只保留关键 Cookie（fanqienovel.com 相关的）
- 包括 session、token、auth 等关键字
- 减少文件大小，提高加载速度

## 关键发现

### ✅ 成功发现

1. **Cookie 持久化是最简单的方案**
   - 无需破解 API
   - 无需解码二维码
   - 无需自动填写账号密码
   - 只需要手动扫码登录一次

2. **Cookie 持久化是最合规的方案**
   - 使用官方登录方式
   - 不违反服务条款
   - 不会导致账号封禁

3. **Playwright 完美支持 Cookie 管理**
   - `context.cookies()` - 获取所有 Cookie
   - `context.addCookies()` - 添加 Cookie
   - `context.clearCookies()` - 清除 Cookie
   - `storageState` - 保存和加载存储状态

4. **Cookie 文件可以长期保存**
   - JSON 格式，易于编辑
   - 可以手动修改 Cookie
   - 可以备份和恢复
   - 可以加密存储（敏感信息）

### ⚠️ 需要注意的问题

1. **Cookie 会过期**
   - 番茄小说的 Cookie 有效期未知（需要实际测试）
   - Cookie 过期后需要重新登录
   - 建议每周刷新一次 Cookie

2. **Cookie 泄露风险**
   - Cookie 包含敏感信息（session token、auth token）
   - 建议加密存储
   - 建议不要分享 Cookie 文件
   - 建议定期更换 Cookie

3. **跨浏览器问题**
   - Cookie 与浏览器绑定
   - 不同浏览器的 Cookie 不兼容
   - 需要使用相同的浏览器加载 Cookie

### 🎯 技术洞察

1. **Cookie 持久化是实现自动化的关键**
   - 解决登录过期问题
   - 提高自动化程度
   - 减少人工干预

2. **Cookie 管理模块是自动化的基础设施**
   - 提供统一的 Cookie 管理接口
   - 简化其他脚本的开发
   - 提高代码复用性

3. **Cookie 有效期是自动化的瓶颈**
   - 如果 Cookie 有效期 > 30 天，几乎无需人工干预
   - 如果 Cookie 有效期 < 7 天，需要更频繁刷新
   - 需要实际测试确定
