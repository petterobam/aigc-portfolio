#### 2. 番茄小说自动化技术栈设计

**整体架构**：
```
自动化运营平台
    ├── AI辅助层（.skill文件）
    ├── 浏览器自动化层（Puppeteer）
    ├── 数据分析层（Python/R）
    └── SOP流程层（标准化操作）
```

**技术栈选型**：

**Layer 1: AI辅助层（对标小红书.skill）**
- LLM API调用（GLM-4.7、GPT-4）
- 提示词工程（Chain-of-Thought、Few-shot Learning）
- 结构化输出

**自动化能力清单**：
| Skill | 功能 | 触发词 |
|-------|------|--------|
| generate_story_outline | 生成故事大纲 | 「生成故事大纲」 |
| generate_chapter | 生成章节内容 | 「生成章节」 |
| generate_title_for_fanqie | 生成爆款标题 | 「生成番茄标题」 |
| optimize_title | 优化标题 | 「优化标题」 |
| generate_tags | 生成题材标签 | 「生成标签」 |
| analyze_story_data | 分析故事数据 | 「分析数据」 |
| review_fanqie_data | 数据复盘 | 「番茄数据复盘」 |

**Layer 2: 浏览器自动化层（核心创新）**

**技术选择**：Puppeteer + Anti-Detection

**核心功能**：
1. **自动登录**
   - Cookie持久化
   - 二维码登录识别
   - 登录状态维护

2. **自动发布**
   - 填充标题
   - 填充封面图（自动上传）
   - 填充正文（支持富文本）
   - 选择题材标签
   - 设置发布参数
   - 提交发布

3. **数据采集**
   - 采集阅读量、点赞、收藏、评论
   - 定时采集（24h/48h/72h）
   - 数据存储到本地/数据库

4. **互动自动化**
   - 自动回复评论（基于话术库）
   - 自动感谢读者
   - 自动引导关注

**技术实现示例**：
- `fanqie-publisher.js` - 发布模块
- `fanqie-collector.js` - 采集模块
- `fanqie-responder.js` - 回复模块
- `cookie-manager.js` - Cookie管理

**Layer 3: 数据分析层**

**技术选择**：
- Python - Pandas、NumPy（数据分析）
- 可视化 - Matplotlib、Plotly（数据可视化）
- 存储 - SQLite（轻量）、PostgreSQL（生产）

**核心功能**：
1. **数据存储** - 故事元数据、发布记录、阅读数据、互动数据
2. **数据分析** - 题材表现分析、标题效果分析、发布时间优化
3. **数据可视化** - 数据看板、趋势图表、对比分析

**数据结构示例**：
- stories表 - 故事元数据
- publish_records表 - 发布记录
