## 💡 解决方案

### 方案 1：使用 MCP 方式操控浏览器（推荐）

**核心思路**: 使用 MCP (Model Context Protocol) 方式操控浏览器，而不是启动新的浏览器窗口。

**优势**:
- 用户可以在真实的浏览器中手动登录
- MCP 可以操控已登录的浏览器窗口
- 不需要启动新的浏览器窗口
- 可以完全自动化后续操作

**技术方案**:
- 使用 Chrome DevTools MCP
- 连接到已登录的 Chrome 浏览器（远程调试端口 9222）
- 使用 MCP 提供的浏览器自动化功能

---

### 方案 2：手动登录 + Cookie 持久化

**核心思路**: 用户手动登录一次，提取 Cookies，然后自动化脚本使用 Cookies 执行后续操作。

**优势**:
- 简单直接，无需复杂的技术方案
- 用户可以在真实浏览器中登录
- Cookies 可以保存 7-30 天

**技术方案**:
- 使用 `extract-cookies.mjs` 脚本提取 Cookies
- 使用 `check-login.mjs` 脚本检查登录状态
- 自动化脚本加载 Cookies，绕过登录流程

---

### 方案 3：混合方案（方案 1 + 方案 2）

**核心思路**: 结合方案 1 和方案 2 的优势，提供更灵活的登录方案。

**优势**:
- 灵活性高，适应不同场景
- 可以自动切换登录方案
- 提供 fallback 机制
