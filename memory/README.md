## 💡 记忆管理原则

### 1. 及时记录

**原则**:
- 重要的事情立即记录,不要依赖"记忆"
- 每天结束时更新当日记录
- 每次心跳后更新心跳记录

**价值**:
- 避免遗忘重要信息
- 形成系统的记忆库
- 便于回顾和总结

### 2. 结构化记录

**原则**:
- 使用统一的格式和模板
- 分类清晰,层次分明
- 标题明确,易于检索

**价值**:
- 提高查找效率
- 便于理解和回顾
- 降低认知负担

### 3. 定期整理

**原则**:
- 每周整理一次记忆文件
- 归档不再频繁使用的旧文件
- 提取重要信息到长期记忆（MEMORY.md）

**价值**:
- 保持记忆文件整洁
- 避免记忆文件过大
- 形成清晰的记忆层级

### 4. 长期积累

**原则**:
- 定期回顾记忆文件
- 提取重要经验到长期记忆（MEMORY.md）
- 持续优化记忆管理方法

**价值**:
- 形成可复用的知识库
- 避免重复犯错
- 持续提升能力

## 🚀 使用方法

### 日常使用

1. **每天开始工作时**:
   ```bash
   # 读取前一天的记录
   cat memory/2026-03-20.md
   ```

2. **每天结束时**:
   ```bash
   # 更新当天的记录
   echo "## $(date +'%H:%M')" >> memory/2026-03-21.md
   echo "今天完成了..." >> memory/2026-03-21.md
   ```

3. **每次心跳后**:
   ```bash
   # 更新心跳记录
   echo "### $(date +'%H:%M') 心跳" >> memory/2026-03-21.md
   echo "**核心任务**: ..." >> memory/2026-03-21.md
   ```

### 定期整理

1. **每周整理**:
   - 归档旧的日常记录（>7天）
   - 整理心跳记录,合并到日常记录
   - 提取重要信息到 MEMORY.md

2. **每月整理**:
   - 创建月度总结报告
   - 归档旧的总结报告
   - 更新 MEMORY.md

### 长期积累

1. **定期回顾**:
   - 回顾重要的记忆文件
   - 提取成功经验
   - 总结失败教训

2. **提取知识**:
   - 将重要经验提取到 MEMORY.md
   - 将重要方法固化到方法论
   - 将重要发现更新到知识库
