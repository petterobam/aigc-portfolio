# 记忆文件 - 番茄小说移动端网页版分析

**记录时间**: 2026-03-25 19:50 (Asia/Shanghai)
**记录人**: 心跳时刻 - 番茄小说创作和运营
**本次心跳任务**: 分析番茄小说移动端网页版是否支持发布功能

---

## 核心发现

### 1. 番茄小说不支持网页端发布 ⭐⭐⭐⭐⭐

**测试结果**:
- PC 端作家后台：不包含发布功能 ❌
- 移动端首页：不包含发布功能 ❌
- 移动端作家后台(novelfm.com)：实际上是番茄音乐 ❌
- 移动端发布页：不包含发布功能 ❌

**结论**:
- 番茄小说不支持网页端发布（PC 和移动端都不支持）
- 发布功能只在 App 中可用
- 验证了上一次心跳(19:20)的发现

---

### 2. 三种自动化发布方案 ⭐⭐⭐⭐⭐

**方案 1: Appium 自动化（推荐）**
- 原理: 使用 Appium 自动化番茄小说 App
- 优点: 可以直接操作真实的番茄小说 App,功能完整
- 缺点: 需要配置模拟器,配置相对复杂
- 预计开发时间: 2-3 天

**方案 2: 网络请求拦截（探索性）**
- 原理: 抓取番茄小说 App 的网络请求,直接调用 API
- 优点: 不需要 UI 自动化,速度快,稳定性高
- 缺点: 需要分析 API 接口,API 可能有签名/加密
- 预计开发时间: 3-5 天

**方案 3: 混合方案（推荐用于短期）**
- 原理: 结合 Playwright 和手动操作
- 优点: 快速实现,可靠性高,开发成本低
- 缺点: 需要人工辅助,自动化程度低
- 预计开发时间: 半天

---

### 3. 分阶段实施方案 ⭐⭐⭐⭐⭐

**短期（本周内）**: 混合方案
- 35号故事今天 20:30 就要发布,时间紧迫
- 混合方案可以快速实现,不耽误发布
- 作为临时方案,验证可行性

**中期（本月内）**: Appium 自动化
- 长期来看,需要完全自动化的发布流程
- Appium 自动化可以覆盖完整的发布流程
- 可以集成到心跳系统中,实现定时发布

**长期（3-6 个月）**: API 自动化
- API 自动化是最稳定、最快速的方案
- 不依赖 UI,不易受界面变化影响
- 易于维护和扩展

## 经验总结

### 1. 问题导向的分析方法很有效 ⭐⭐⭐⭐⭐

**经验**:
- 从问题出发,明确要解决什么问题（番茄小说不支持网页端发布）
- 设计测试方案,验证问题（测试 4 个 URL）
- 分析问题,设计解决方案（3 种技术方案）
- 实施解决方案,验证效果

**优势**:
- 目标明确,不会迷失方向
- 测试方案清晰,易于执行
- 解决方案可行,可以分阶段实施

---

### 2. 技术方案设计要考虑时间成本 ⭐⭐⭐⭐⭐

**经验**:
- 35号故事今天 20:30 就要发布,时间紧迫
- 不能追求完美的长期方案,要先实现可用的短期方案
- 分阶段实施: 短期方案（快速实现）→ 中期方案（完全自动化）→ 长期方案（最优方案）

**优势**:
- 不耽误发布时间
- 逐步优化,逐步完善
- 降低风险,每个阶段都可以验证可行性

---

### 3. 测试要覆盖所有可能的场景 ⭐⭐⭐⭐

**经验**:
- 不仅测试 PC 端,也测试移动端
- 不仅测试作家后台,也测试发布页
- 不仅测试一个 URL,测试 4 个 URL

**发现问题**:
- 移动端作家后台(novelfm.com)实际上是番茄音乐,不是番茄小说
- 如果只测试一个 URL,可能会遗漏这个重要发现

**优势**:
- 全面的测试可以发现所有可能的问题
- 避免遗漏关键信息
- 提供更准确的结论
