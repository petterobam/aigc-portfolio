### 页面状态检测逻辑问题

**当前逻辑**：
```javascript
async function detectPageState(page) {
  const state = await page.evaluate(() => {
    // 检查是否有发布按钮
    const hasPublishButton = ...;

    // 检查是否有"下一步"按钮
    const hasNextButton = ...;

    // 检查是否有标题输入框
    const hasTitleInput = ...;

    // 检查是否有正文输入框
    const hasContentTextarea = ...;

    return {
      hasPublishButton,
      hasNextButton,
      hasTitleInput,
      hasContentTextarea
    };
  });

  // 判断页面状态
  if (state.hasPublishButton) {
    return 'final';
  } else if (state.hasNextButton && !state.hasTitleInput && !state.hasContentTextarea) {
    return 'preview';
  } else if (state.hasNextButton && (state.hasTitleInput || state.hasContentTextarea)) {
    return 'initial';
  } else {
    return 'unknown';
  }
}
```

**问题**：
1. 过于依赖DOM结构
2. 不能准确区分不同的页面
3. 可能所有页面都有相同的DOM结构
4. 需要根据页面内容、URL、标题等综合判断

**改进方向**：
1. 使用URL变化作为判断依据
2. 使用页面标题作为判断依据
3. 使用页面内容作为判断依据
4. 增加更多的页面特征（如按钮文本、提示信息等）

### 填充标题和正文失败

**填充标题失败**：
- 错误信息：未找到标题输入框
- 可能原因：
  1. 页面状态判断错误，不在初始页面
  2. 输入框选择器不准确
  3. 输入框是动态生成的，需要等待

**填充正文成功**：
- 成功填充27,699字
- 说明正文选择器是准确的
- 但为什么标题选择器不准确？

**可能的原因**：
1. 标题输入框在不同的页面状态下有不同的选择器
2. 标题输入框可能被隐藏或禁用
3. 标题输入框可能需要特定的操作才能显示
