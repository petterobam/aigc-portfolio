### 2. 学习七步方法论（基于规格驱动开发 SDD）

**核心流程**：
```
【价值观层】 /constitution → 建立不可妥协的创作原则
    ↓
【需求层】   /specify      → 明确要创造什么样的故事
    ↓
【决策层】   /clarify      → 澄清关键的创作决策
    ↓
【设计层】   /plan         → 设计实现需求的方案
    ↓
【执行层】   /tasks        → 分解为可操作的步骤
    ↓
【实施层】   /write        → 执行具体的创作
    ↓
【验证层】   /analyze      → 确保一致性和质量
```

**核心理念**：
1. 规格驱动创作：先定义"要创造什么"，再考虑"如何创造"
2. 渐进式明确：从模糊的灵感到清晰的规格
3. 可验证的质量：创作不是玄学，质量可以被验证

### 3. 学习质量控制最佳实践

**黄金开篇法则**（前三章决定 80% 读者留存）：
1. 动态场景切入（从冲突/动作/对话直接切入）
2. 核心冲突前置（第一章内必须抛出主角核心冲突）
3. 避免信息轰炸（滴灌式信息透露）
4. 限制出场人数（第一章有姓名角色不超过 3 人）
5. 快速展现金手指（第二或第三章内展现"金手指"作用）

**对标作品学习**（v0.17+ 新增）：
- 使用 book-analysis 插件学习成功作品的写作技巧
- 自动生成三个关键文件：
  * `memory/style-reference.md`（AI 写作时自动应用对标作品风格）
  * `memory/constitution-draft.md`（为创作宪法提供参考草稿）
  * `spec/presets/rhythm-config.json`（为创作计划提供节奏参数）

**双保险机制**（v0.17+ 新增）：
- 写作前：规格质量检查（验证文档）
- 写作后：内容验证检查（扫描章节）

- 伏笔和回收必须清晰（所有伏笔都必须有明确的回收）

- 核心冲突：主角必须逃离午夜电梯的循环死亡，找出"全死光了"背后的真相
- 核心主题：面对未知和恐惧时的勇气，追求真相和正义的决心

**任务分解**（15个任务）：
- 🔴 高优先级（3个）：补全第4-6章、编写第7-8章、设计高潮和结局
- 🟡 中优先级（3个）：完善主角设定、设计其他乘客角色、设计午夜电梯规则
- 🟢 低优先级（3个）：撰写恐怖场景细节、检查逻辑自洽性、检查伏笔回收
- 🟵 备选任务（6个）：增加副线、增加情感元素、增加哲学思考等

## 📊 对比分析：Novel Writer vs 番茄短篇故事集

### 相似点
- 数据驱动：都有追踪系统和数据分析
- 质量保障：都有质量检查清单和验证机制
- 最佳实践：都有完善的文档和方法论

### 差异点
- 创作方法：Novel Writer（七步方法论 SDD） vs 番茄短篇故事集（爆款公式 + 数据驱动）
- 创作规模：Novel Writer（长篇连载 50万字+） vs 番茄短篇故事集（短篇故事 8-15千字）
- 工具形式：Novel Writer（斜杠命令 AI助手） vs 番茄短篇故事集（自动化脚本 + 手动操作）
- 验证重点：Novel Writer（一致性、逻辑性） vs 番茄短篇故事集（阅读量、爆款潜力）

## 🎯 应用建议

### 1. 引入规格驱动开发（SDD）理念
**当前状态**：
- 番茄短篇故事集有爆款公式（基于30个故事数据）
- 有发布包结构（包含元数据）
- 缺少规格驱动开发的系统化流程

**改进建议**：
- 为每个故事创建规格文档（类似 Novel Writer 的 specification.md）
- 包含：核心概念、目标读者、金手指、字数、章节数、成功标准
- 在创作前明确"要创造什么"，再考虑"如何创造"

### 2. 引入质量检查机制（双保险）
**当前状态**：
- 有质量检查清单（发布前检查）
- 缺少写作后的内容验证

**改进建议**：
- 建立双保险机制：
  * 写作前：规格质量检查（爆款公式符合度、字数合理性、标题质量等）
  * 写作后：内容验证检查（重复段落检查、节奏检查、爽点检查等）

### 3. 引入对标作品学习机制
**当前状态**：
- 有爆款公式（基于30个故事数据）
- 有高表现作品分析（TOP1、TOP2等）
- 缺少自动化的风格内化机制

**改进建议**：
- 提炼高表现作品的特征（金手指类型、开篇套路、节奏模式等）
- 为新故事自动应用这些特征

### 4. 引入定期检查流程
**当前状态**：
- 有发布后数据收集计划（24h、48h、72h）
- 缺少创作过程中的定期检查

**改进建议**：
- 建立"写作进度"检查：每完成一章后检查
- 建立"情节一致性"检查：每 3-5 章定期检查
- 建立"爽点节奏"检查：每完成一个故事后检查
