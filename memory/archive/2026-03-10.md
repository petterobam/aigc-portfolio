### 今日主题：阶段1 - 第2天：LLM 输入与输出

**学习时间**：2026-03-10 10:30

**学习内容**：
- LLM 输入层（文本→数值矩阵）
- LLM 输出层（隐藏状态→概率分布）
- TokenID 与词元映射

**核心要点**：

1. **LLM 输入层：文本转数值**
   - LLM的输入层将输入**文本**转换为多维**数值矩阵**（Tensor，张量），以便送往模型主体结构进行计算
   - 主要步骤：
     - Tokenization：将文本分割成词元（tokens）
     - TokenID映射：每个词元转换为唯一的整数ID
     - Embedding lookup：将TokenID转换为向量表示（如896维）
     - 位置编码：添加位置信息（如RoPE）
   - 数据流动：文本 → Token序列 → TokenID序列 → Embedding矩阵 → 添加位置编码 → 模型输入

2. **LLM 输出层：隐藏状态转概率**
   - 输入隐藏状态（Hidden States）：Decoder最后一层的输出，包含前缀序列的语义信息
     - 示例：维度 3×896 的数值矩阵（序列长度=3，隐藏维度=896）
   - 语言模型头（LM Head）：全连接层，将隐藏状态转换为Logits
     - 输出形状：序列长度×词表大小（如 3×151936）
   - 提取最后位置Logits：预测下一个token只依赖最后一个位置的Logits
     - 提取得到 151936 维向量，表示词表中每个词的得分
   - Softmax：将Logits转换为概率分布
     - 概率向量总和为1，表示每个词元作为下一个词的可能性
   - 解码（Decoding）：根据概率分布，应用解码策略确定下一个词元
     - 例如：贪婪搜索选择概率最高的"我"

3. **TokenID 与词元映射关系**
   - 每个Token ID与相应的词元**一一对应**
   - 以SFT数据为例（ChatML格式）：
     - 原始文本经过Tokenizer处理后，被转换为33个Token
     - 对应33个序列位置，每个位置有唯一的TokenID
   - Tokenization的重要性：
     - 不同的Tokenizer会产生不同的词元分割
     - 影响模型性能和推理效率
     - 常见Tokenizer：BPE（Byte Pair Encoding）、SentencePiece等

4. **输入输出数据流总结**

**输入流程**：
```
原始文本 "你好世界"
    ↓ [Tokenizer]
词元序列 ["你", "好", "世", "界"]
    ↓ [TokenID映射]
ID序列 [234, 567, 890, 123]
    ↓ [Embedding Lookup]
向量矩阵 [[0.12, -0.34, ...], ...]  # shape: 4×896
    ↓ [位置编码 + 多层Transformer]
隐藏状态 [[0.56, 0.78, ...], ...]  # shape: 4×896
```

**输出流程**：
```
隐藏状态 (最后位置) [0.56, 0.78, ...]  # shape: 896
    ↓ [LM Head]
Logits [2.0, 3.1, -1.7, ...]  # shape: 151936
    ↓ [Softmax]
概率分布 [0.01, 0.03, 0.001, ...]  # shape: 151936, sum=1
    ↓ [解码策略]
预测词元 "我" (概率=0.34)
```
