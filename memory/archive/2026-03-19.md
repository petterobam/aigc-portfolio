### 关键发现

#### 登录相关问题
1. **登录方式**：番茄小说使用扫码登录（用户体验好但自动化难度高）
2. **登录状态判断**：可以通过页面元素判断登录状态
   - 未登录：显示二维码容器 `.slogin-qrcode-scan-page`
   - 已登录：显示用户信息区 `.muye-header-user`
3. **登录过期**：Chrome 用户数据目录中的登录状态会过期（具体有效期待研究）
4. **自动化等待**：可以自动等待用户扫码登录，但需要人工干预

#### 技术问题
1. **登录状态维护**：如何保持长期有效的登录状态？
   - 可能的解决方案：定期刷新登录状态（每周一次）、cookie 持久化、API 登录
2. **自动化登录**：能否实现自动登录（不依赖扫码）？
   - 需要研究番茄小说是否支持用户名密码登录
   - 需要研究是否有官方 API 可用
3. **登录频率限制**：频繁登录是否会触发安全限制？
   - 需要测试和观察
