### 步骤 2：理解 OpenClaw Agent 工作原理（15分钟）

**Agent 核心概念**（基于官方文档）：

#### 2.1 Agent 架构

```
用户消息 → Agent（大脑） → 工具调用 → 执行结果 → Agent → 回复用户
           ↓
        记忆系统
           ↓
        会话历史
```

**关键组件**：
- **Agent**：AI 大脑，理解意图、决策、生成回复
- **Tools**：工具集（文件操作、网络请求、数据处理等）
- **Session**：会话上下文，保持对话连续性
- **Memory**：长期记忆，跨会话存储重要信息

---

**基础配置**（`~/.openclaw/openclaw.json`）：

#### 4.2 配置自定义工具

**创建工具脚本**（`~/.openclaw/workspace/tools/`）：

**文件**：`quality_checker.py`

```python
#!/usr/bin/env python3
"""
内容质量检查工具
检查重复段落、逻辑问题、格式规范
"""

import sys
import json
from difflib import SequenceMatcher

def check_duplicates(content, threshold=0.95):
    """检查重复段落"""
    paragraphs = content.split('\n\n')
    duplicates = []
    
    for i in range(len(paragraphs)):
        for j in range(i+1, len(paragraphs)):
            similarity = SequenceMatcher(
                None, 
                paragraphs[i], 
                paragraphs[j]
            ).ratio()
            
            if similarity > threshold:
                duplicates.append({
                    'para1': i+1,
                    'para2': j+1,
                    'similarity': similarity,
                    'content': paragraphs[i][:100]
                })
    
    return {
        'has_duplicates': len(duplicates) > 0,
        'duplicates': duplicates,
        'score': 1.0 - (len(duplicates) * 0.1)
    }

def main():
    if len(sys.argv) < 2:
        print("Usage: quality_checker.py <content_file>")
        sys.exit(1)
    
    with open(sys.argv[1], 'r', encoding='utf-8') as f:
        content = f.read()
    
    result = check_duplicates(content)
    print(json.dumps(result, indent=2, ensure_ascii=False))

if __name__ == '__main__':
    main()
```

**注册工具**（在 OpenClaw 配置中）：

```json5
{
  plugins: {
    entries: {
      quality_checker: {
        enabled: true,
        type: "stdio",
        config: {
          command: "python3",
          args: ["~/.openclaw/workspace/tools/quality_checker.py"],
        },
      },
    },
  },
}
```

## 💡 关键要点

### 提示词工程核心技巧

1. **结构化思维**：任务 → 背景 → 要求 → 格式 → 示例
2. **Few-Shot Learning**：提供 2-3 个高质量示例
3. **Chain-of-Thought**：要求 AI 分步骤思考
4. **迭代优化**：版本管理，持续改进
5. **角色扮演**：定义清晰的 AI 角色和工作风格

### Agent 协作核心机制

1. **工具调用**：Agent 自动识别需要调用的工具
2. **工作流编排**：Agent 规划多步骤任务
3. **条件分支**：根据结果决定下一步行动
4. **错误处理**：自动重试或降级方案
5. **结果汇总**：整合多个工具的输出

### 最佳实践

1. **提示词模板化**：将常用提示词保存为模板
2. **工具模块化**：每个工具只做一件事，做好一件事
3. **工作流可视化**：让用户了解 Agent 的执行过程
4. **质量自动化**：集成质量检查工具，形成闭环
5. **版本管理**：保存每个版本的提示词和结果

## 🔧 常见问题

### 问题 1：Agent 调用工具失败

**原因**：工具未正确配置或参数错误

**解决方案**：
```bash
# 检查工具状态
openclaw tools list

# 测试工具
openclaw tools test quality_checker

# 查看日志
openclaw logs --tool quality_checker
```

---

### 问题 2：提示词效果不稳定

**原因**：提示词不够结构化或缺少示例

**解决方案**：
1. 添加 2-3 个高质量示例
2. 明确输出格式要求
3. 使用分步骤指导
4. 添加质量检查标准

---

### 问题 3：Agent 陷入循环

**原因**：工作流设计不合理，缺少终止条件

**解决方案**：

```json5
{
  agents: {
    defaults: {
      tools: {
        maxIterations: 10,  // 最大迭代次数
        timeout: 300000,    // 超时时间（毫秒）
      },
    },
  },
}
```
