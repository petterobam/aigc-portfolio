#### 3.3 创建自定义工具

**创建 Python 工具**（`~/.openclaw/workspace/tools/`）：

**文件**：`content_analyzer.py`

```python
#!/usr/bin/env python3
"""
内容分析工具
分析文本的情感、可读性、关键词等
"""

import sys
import json
from textstat import flesch_reading_ease
from collections import Counter
import re

def analyze_content(text):
    """分析内容质量"""
    
    # 情感分析（简化版）
    positive_words = ['优秀', '出色', '精彩', '感人', '震撼']
    negative_words = ['糟糕', '失败', '无聊', '平淡', '混乱']
    
    positive_count = sum(1 for word in positive_words if word in text)
    negative_count = sum(1 for word in negative_words if word in text)
    
    # 可读性评分
    readability = flesch_reading_ease(text)
    
    # 关键词提取
    words = re.findall(r'\b\w+\b', text.lower())
    keywords = Counter(words).most_common(10)
    
    # 统计信息
    stats = {
        'total_chars': len(text),
        'total_words': len(words),
        'total_paragraphs': len(text.split('\n\n')),
        'avg_word_length': sum(len(w) for w in words) / len(words) if words else 0,
    }
    
    return {
        'sentiment': {
            'positive': positive_count,
            'negative': negative_count,
            'score': (positive_count - negative_count) / max(positive_count + negative_count, 1)
        },
        'readability': readability,
        'keywords': keywords,
        'stats': stats,
        'quality_score': min(100, max(0, 
            50 + 
            (positive_count * 10) - 
            (negative_count * 10) + 
            (readability / 2)
        ))
    }

def main():
    if len(sys.argv) < 2:
        print("Usage: content_analyzer.py <text>")
        sys.exit(1)
    
    text = sys.argv[1]
    result = analyze_content(text)
    print(json.dumps(result, indent=2, ensure_ascii=False))

if __name__ == '__main__':
    main()
```

## 💡 关键要点

### 工具链设计原则

1. **单一职责**：每个工具只做一件事，做好一件事
2. **可组合性**：工具之间可以自由组合，形成复杂流程
3. **可观测性**：每个步骤都有清晰的输入输出和日志
4. **容错性**：设计重试机制和降级方案
5. **可复用性**：将常用流程抽象为模板

### 工作流最佳实践

1. **明确目标**：每个工作流有清晰的输入和预期输出
2. **步骤拆分**：复杂任务拆分为多个简单步骤
3. **条件分支**：根据中间结果决定后续步骤
4. **并行执行**：独立步骤可以并行执行，提高效率
5. **错误处理**：每步都有失败处理机制

### 性能优化技巧

1. **缓存结果**：避免重复计算和请求
2. **批量处理**：合并相似操作，减少调用次数
3. **异步执行**：耗时操作使用后台任务
4. **资源限制**：设置超时和内存限制
5. **监控指标**：记录每步耗时，识别瓶颈

## 🔧 常见问题

### 问题 1：工具调用失败

**原因**：工具未安装或配置错误

**解决方案**：
```bash
# 检查工具状态
openclaw tools list

# 测试工具
openclaw tools test content_analyzer

# 查看日志
openclaw logs --tool content_analyzer
```

---

### 问题 2：工作流执行缓慢

**原因**：步骤串行执行，缺少优化

**解决方案**：
1. 识别可并行的步骤
2. 使用异步工具调用
3. 缓存中间结果
4. 优化提示词长度

---

**原因**：缺少文档和版本管理

**解决方案**：
1. 为每个工具添加详细注释
2. 使用 Git 管理工具和工作流
3. 编写测试用例
4. 定期重构和优化
