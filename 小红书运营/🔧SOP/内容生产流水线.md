# 内容生产流水线 V2.0

> **版本**: 2.0
> **更新日期**: 2026-03-18
> **优化目标**: 减少重复工作，提高生产效率，统一质量标准

---

## 一、现状问题诊断

### 发现的问题

| 问题 | 影响 | 解决方案 |
|-----|------|---------|
| 编号混乱 | 新旧编号并存，容易混淆 | 统一使用V2.0编号规范 |
| 发布时间不规律 | 凌晨发布影响曝光 | 严格控制在20:00-21:00 |
| 数据记录分散 | 多份数据文档信息重复 | 统一使用数据追踪看板 |
| 内容储备不足 | 临时赶工影响质量 | 建立内容储备池（至少3篇） |
| 复盘不及时 | 数据洞察滞后 | 每周固定时间复盘 |

### 数据验证

**Week 1-2 关键发现**:
- ✅ 趋势+教程组合最有效（帕斯蒂尔风259浏览，14收藏）
- ✅ 情绪共鸣是点赞关键（第8篇16点赞突破）
- ⚠️ 凌晨发布严重影响曝光（3篇凌晨发布数据最差）
- ⚠️ 评论互动偏低（平均<2条，CTA需要加强）

## 二、优化后的生产流程

### 阶段1: 选题策划（周一）

**时间**: 每周一 09:00-11:00
**产出**: 本周3篇选题 + 下周2篇备选

**检查清单**:
- [ ] 查看数据看板，分析上周表现
- [ ] 浏览小红书热榜，识别趋势热点
- [ ] 检查竞品监控文档
- [ ] 确定本周3篇选题（按优先级排序）

**选题标准**:
```
有效选题 = 趋势热点 + 实用价值 + 情绪共鸣 + 人设契合
```

**选题模板**:
| 字段 | 内容 |
|-----|------|
| 选题名称 | |
| 蹭的热点 | |
| 目标受众 | |
| 实用价值 | |
| 情绪钩子 | |
| 预期数据 | 浏览: / 点赞: / 收藏: |

---

### 阶段2: 文案创作（周二-周三）

**时间**: 周二 14:00-17:00, 周三 14:00-17:00
**产出**: 3篇文案终稿

**文案结构模板**:
```
【开头】情绪钩子（3行内）
    ↓
【冲突】真实场景/痛点（3-5行）
    ↓
【转折】发现/改变（2-3行）
    ↓
【干货】教程/步骤（核心内容）
    ↓
【收尾】互动CTA（必须带选择题）
    ↓
【标签】8-10个相关标签
```

**文案检查清单**:
- [ ] 标题≤30字，含数字或情绪词
- [ ] 前3行有钩子（好奇/共鸣/冲突）
- [ ] 有真实场景描述（人设强化）
- [ ] 干货内容可操作（步骤清晰）
- [ ] 带选择题CTA（A/B/C/D）
- [ ] 标签8-10个，含热点标签

---

### 阶段3: 视觉制作（周四）

**时间**: 周四 14:00-17:00
**产出**: 3张封面图 + 配图素材

**封面图标准**:
- 尺寸: 3:4 竖版
- 风格: 对镜自拍，真实感
- 妆容: 与文案主题一致
- 表情: 自然，有情绪

**封面图命名规范**:
```
第X篇-关键词-封面.png
例: 第17篇-帕斯蒂尔风春日妆-封面.png
```

**视觉检查清单**:
- [ ] 封面图与标题主题一致
- [ ] 图片清晰，光线充足
- [ ] 妆容效果可见
- [ ] 背景不杂乱

---

### 阶段4: 发布包组装（周五）

**时间**: 周五 14:00-16:00
**产出**: 3个完整发布包

**发布包结构**:
```
📤待发布/
└── 第X篇-标题-发布包.md
    ├── 发布日期/时间
    ├── 封面图路径
    ├── 标题（可直接复制）
    ├── 正文（可直接复制）
    ├── 置顶评论内容
    ├── 发布后动作清单
    └── 数据预期
```

**发布包检查清单**:
- [ ] 标题已优化，含热点关键词
- [ ] 正文完整，格式正确
- [ ] 封面图已生成并测试显示效果
- [ ] 置顶评论已准备
- [ ] 发布时间设定为20:00-21:00
- [ ] 数据预期已填写

---

### 阶段5: 发布执行（每日20:00-21:00）

**发布前检查**:
- [ ] 当前时间在20:00-21:00之间
- [ ] 封面图已上传到手机相册
- [ ] 文案已复制到剪贴板

**发布步骤**:
1. 打开小红书APP → 点击"+" → 选择"图文"
2. 选择封面图
3. 粘贴标题
4. 粘贴正文
5. 确认标签已包含
6. 点击发布
7. 立即粘贴置顶评论

**发布后5分钟内**:
- [ ] 置顶评论已发布
- [ ] 截图记录初始数据
- [ ] 更新数据追踪看板

---

### 阶段6: 数据追踪（每日）

**追踪指标**:
| 指标 | 记录时间 | 工具 |
|-----|---------|------|
| 小眼睛 | 发布后2h/24h | 小红书APP |
| 点赞 | 实时 | 小红书APP |
| 收藏 | 实时 | 小红书APP |
| 评论 | 实时 | 小红书APP |
| 互动率 | 计算 | 数据追踪看板 |

**数据记录模板**:
```
| 新编号 | 发布日期 | 标题 | 小眼睛 | 点赞 | 收藏 | 评论 | 互动率 |
```

---

### 阶段7: 周复盘（周日）

**时间**: 每周日 10:00-12:00
**产出**: 周复盘报告

**复盘内容**:
1. **数据总览**: 本周发布X篇，总浏览X，总点赞X，总收藏X
2. **表现排名**: Top 3和Bottom 3笔记
3. **成功因素**: 什么做对了
4. **失败教训**: 什么可以改进
5. **下周策略**: 基于数据的调整

**复盘模板位置**: `📊数据看板/周复盘/WeekX-复盘报告.md`

## 四、质量标准

### 文案质量标准

| 维度 | 标准 | 检查方式 |
|-----|------|---------|
| 标题吸引力 | 让人想点击 | 自问：我会点吗？ |
| 开头钩子 | 3行内抓住注意力 | 快速浏览测试 |
| 干货价值 | 可操作、有收获 | 步骤是否清晰 |
| 情绪共鸣 | 有温度、有人味 | 读出来感受 |
| 互动设计 | 有明确CTA | 是否带选择题 |

### 视觉质量标准

| 维度 | 标准 | 检查方式 |
|-----|------|---------|
| 主题一致性 | 与标题匹配 | 对比检查 |
| 清晰度 | 无模糊、光线足 | 放大检查 |
| 吸引力 | 让人想点击 | 快速浏览测试 |
| 人设强化 | 符合养成系定位 | 风格统一 |
