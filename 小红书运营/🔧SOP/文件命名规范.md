# 小红书运营文件命名规范 V3.0

> **生效日期**: 2026-03-18
> **适用范围**: 所有新创建文件

---

## 一、命名原则

1. **清晰性**: 一眼能看出文件内容
2. **一致性**: 同类文件使用统一格式
3. **可排序**: 按文件名能正确排序
4. **可追溯**: 包含时间或编号信息

---

## 二、已发布内容命名

### 格式
```
第{序号}篇-{标题}-{类型}.md
```

### 示例
```
第18篇-Week2数据复盘-发布记录.md
第19篇-油皮夏季定妆-发布记录.md
第20篇-单眼皮眼妆教程-发布记录.md
```

### 说明
- **序号**: 按实际发布顺序连续编号，从第18篇开始
- **标题**: 简洁版标题，不超过15字
- **类型**: 发布记录 / 数据复盘 / 封面素材

---

## 三、待发布内容命名

### 格式
```
第{序号}篇-{标题}-发布包.md
```

### 示例
```
第18篇-Week2数据复盘-发布包.md
第19篇-油皮夏季定妆-发布包.md
```

---

## 四、内容生产文件命名

### 4.1 文案稿
```
第{X}篇-{标题}-文案{版本}.md
```
示例: `第18篇-Week2数据复盘-文案终稿.md`

### 4.2 封面图
```
第{X}篇-{描述}-{尺寸}.png
```
示例: `第18篇-对镜自拍-3x4.png`

### 4.3 素材图
```
{日期}-{场景}-{描述}.png
```
示例: `2026-03-18-办公室-妆前对比.png`

---

## 五、数据文件命名

### 5.1 周报
```
Week{X}-数据复盘-{日期}.md
```
示例: `Week2-数据复盘-2026-03-23.md`

### 5.2 月报
```
{月份}-月度复盘.md
```
示例: `2026-03-月度复盘.md`

---

## 六、历史遗留问题处理

### 问题说明
- 第1-17篇文件编号混乱（存在跳号、重复）
- 部分文件使用"发布包"，部分使用"发布记录"

### 解决方案
1. **不修改历史文件名** - 避免链接失效
2. **使用对照表** - 通过`发布记录总表.md`定位文件
3. **从第18篇开始** - 严格执行新规范

---

## 七、快速检查清单

创建文件前检查:
- [ ] 序号是否正确（查看最新发布记录）
- [ ] 标题是否简洁（≤15字）
- [ ] 类型标识是否正确
- [ ] 文件位置是否在正确目录
