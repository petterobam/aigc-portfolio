# 问题处理工作流程

## 标准流程

每次用户提问时，必须遵循以下步骤：

### 第一步：搜索记忆和知识库
```bash
# 搜索 memory/ 文件夹（日记、工作记录）
memory_search "关键词"

# 搜索 knowledge/ 文件夹（技术文档、经验教训）
# 通过 file_path 参数限定搜索范围
```

### 第二步：结合资料回答
- 如果找到相关资料：
  - ✅ 整合信息回答
  - ✅ 标注信息来源：「记忆」或「知识库」
  - ✅ 提供具体文件路径引用
- 如果未找到相关资料：
  - ❓ 说明未找到历史资料
  - ❓ 基于现有知识回答
  - ❓ 建议补充到知识库

### 第三步：更新知识库（可选）
- 如果回答过程中发现重要信息：
  - 记录到 knowledge/ 相关目录
  - 更新或创建新文档
  - 确保下次搜索能找到

## 搜索技巧

### 搜索 memory/
- 搜索格式：`memory_search "关键词"`
- 常用场景：
  - 查找历史决策
  - 回顾工作记录
  - 追踪项目进展
  - 查找踩坑经验

### 搜索 knowledge/
- 常用场景：
  - 查找技术文档
  - 查找工作流程
  - 查找人物信息
  - 查找经验教训

## 回答格式示例

### 找到资料时：
> 根据「知识库」knowledge/tech/xxx.md 的记录，XXX 的用法是...
> 另外，「记忆」memory/2026-03-30.md 中提到上次遇到类似问题时...

### 未找到资料时：
> 搜索了我的记忆和知识库，未找到相关历史记录。基于现有信息，我的理解是...

## 重要原则
1. **先搜索，再回答** - 这是强制性步骤
2. **标注来源** - 让用户知道信息从哪里来
3. **持续积累** - 每次处理新问题时，考虑是否需要记录到知识库
