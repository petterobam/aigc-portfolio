## 💡 关键洞察

### 1. 中文路径是 Next.js/Turbopack 的已知问题

**发现**：
- Next.js 16.2.1 使用 Turbopack 作为构建工具
- Turbopack 无法正确处理文件路径中的中文字符
- 这是一个已知的 Bug，但在 2026 年 3 月仍未修复

**错误信息**：
```
start byte index 33 is not a char boundary; it is inside '之' (bytes 32..35) of `_openclaw_workspace_财富自由之路_aigc-portfolio_...`
```

**解决方案**：
- 将项目复制到不含中文的路径
- 或者创建符号链接（symlink）

**数据支持**：
- 原路径：`~/.openclaw/workspace/aigc-portfolio`（构建失败）
- 新路径：`~/.openclaw/workspace/aigc-portfolio`（构建成功）
- 修复时间：5 分钟

**下一步**：
- 在项目文档中明确说明：项目路径不能包含中文字符
- 在 GitHub README 中添加注意事项
