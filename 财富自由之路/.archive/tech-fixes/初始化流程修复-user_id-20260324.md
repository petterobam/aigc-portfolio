# 初始化流程修复 - user_id 问题 - 2026-03-24 19:54

## 🐛 问题描述

在初始化时点击"开始使用"报错：
```
NOT NULL constraint failed: accounts.user_id
```

## 🔍 根本原因

所有 IPC create 处理函数都缺少 `user_id` 字段：

**修复前的函数**：
- handleAccountCreate - 缺少 user_id
- handleDebtCreate - 缺少 user_id
- handleGoalCreate - 缺少 user_id
- handleTransactionCreate - 缺少 user_id
- handleDreamCreate - 缺少 user_id

## ✅ 解决方案

### 修复了所有 create 函数

#### 1. handleAccountCreate
```typescript
async function handleAccountCreate(_event: any, data: any) {
  const id = `account-${Date.now()}`;
  const stmt = db.prepare(`
    INSERT INTO accounts (id, user_id, name, type, balance, currency, institution, notes, created_at, updated_at)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?, datetime('now'), datetime('now'))
  `);
  stmt.run(id, data.userId || data.user_id, data.name, data.type, data.balance, data.currency || 'CNY', data.institution, data.notes);
  return { id, ...data };
}
```
