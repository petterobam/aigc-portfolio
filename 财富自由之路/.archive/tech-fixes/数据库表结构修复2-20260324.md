# 数据库表结构修复 #2 - 2026-03-24 14:21

## 🐛 问题描述

点击"财务看板"时报错：
```
Error invoking remote method 'goal:getAll': SqliteError: no such column: stage
```

## 🔍 根本原因

`goals` 表定义与 `seed.ts` 中插入的数据不匹配：

**database.ts 中的表定义**：
- 字段：id, user_id, **type**, target_amount, current_amount, monthly_expense, target_date, status, ...

**seed.ts 中插入的字段**：
- 字段：**stage**, target_amount, current_amount, target_date, notes, ...

**问题**：
- ❌ 表定义中有 `type`，但 seed.ts 使用 `stage`
- ❌ 表定义中有 `monthly_expense`，但 seed.ts 使用 `notes`
- ❌ 字段名不匹配导致查询失败

## ✅ 解决方案

### 1. 修复表定义

修改 `apps/desktop/src/main/database.ts`：

**修改前**：
```sql
CREATE TABLE IF NOT EXISTS goals (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  type TEXT NOT NULL,                    -- ❌ 错误字段
  target_amount REAL NOT NULL,
  current_amount REAL DEFAULT 0,
  monthly_expense REAL,                  -- ❌ 错误字段
  target_date DATE,
  status TEXT DEFAULT 'not_started',
  ...
)
```

**修改后**：
```sql
CREATE TABLE IF NOT EXISTS goals (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  stage TEXT NOT NULL,                   -- ✅ 正确字段
  target_amount REAL NOT NULL,
  current_amount REAL DEFAULT 0,
  target_date DATE,
  notes TEXT,                            -- ✅ 正确字段
  status TEXT DEFAULT 'in_progress',
  ...
)
```

### 2. 删除旧数据库

```bash
rm -rf ~/Library/Application\ Support/@wealth-freedom/desktop/
```

## 📊 字段映射表
