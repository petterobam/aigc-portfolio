# 数据库表结构修复 #3 - accounts 表 - 2026-03-24 19:23

## 🐛 问题描述

启动应用时报错：
```
SqliteError: table accounts has no column named institution
```

## 🔍 根本原因

`accounts` 表定义与 `seed.ts` 中插入的数据不匹配：

**database.ts 中的表定义（修复前）**：
- 字段：id, user_id, name, type, icon, color, balance, currency, is_reserved, ...
- ❌ 缺少：`institution`, `notes`

**seed.ts 中插入的字段**：
- 字段：name, type, balance, currency, **institution**, **notes**, ...
- ✅ 使用了：`institution`, `notes`

## ✅ 解决方案

### 1. 修复表定义

修改 `apps/desktop/src/main/database.ts`：

**添加的字段**：
```sql
institution TEXT,  -- 机构名称
notes TEXT,        -- 备注
```

**完整的 accounts 表结构**：
```sql
CREATE TABLE IF NOT EXISTS accounts (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  name TEXT NOT NULL,
  type TEXT NOT NULL,
  icon TEXT,
  color TEXT,
  balance REAL DEFAULT 0,
  currency TEXT DEFAULT 'CNY',
  institution TEXT,        -- ✅ 新增
  notes TEXT,              -- ✅ 新增
  is_reserved INTEGER DEFAULT 0,
  include_in_net_worth INTEGER DEFAULT 1,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
)
```

### 2. 删除旧数据库

```bash
rm -rf ~/Library/Application\ Support/@wealth-freedom/desktop/
```
