# 数据库修复 #4 - 所有表的 user_id 字段 - 2026-03-24 19:25

## 🐛 问题描述

启动应用时报错：
```
SqliteError: NOT NULL constraint failed: accounts.user_id
```

## 🔍 根本原因

所有表（accounts, debts, goals, dreams）的 `user_id` 字段都是 NOT NULL，但 `seed.ts` 在插入测试数据时没有提供 `user_id` 值。

## ✅ 解决方案

### 修改了 seed.ts 中的所有插入语句

#### 1. accounts 表
```sql
-- 修改前
INSERT INTO accounts (name, type, balance, currency, institution, notes, ...)

-- 修改后
INSERT INTO accounts (user_id, name, type, balance, currency, institution, notes, ...)
VALUES ('user-1', ?, ?, ?, ?, ?, ?, ...)
```

#### 2. debts 表
```sql
-- 修改前
INSERT INTO debts (name, total_amount, paid_amount, ...)

-- 修改后
INSERT INTO debts (user_id, name, total_amount, paid_amount, ...)
VALUES ('user-1', ?, ?, ?, ...)
```

#### 3. goals 表
```sql
-- 修改前
INSERT INTO goals (stage, target_amount, current_amount, ...)

-- 修改后
INSERT INTO goals (user_id, stage, target_amount, current_amount, ...)
VALUES ('user-1', ?, ?, ?, ...)
```

#### 4. dreams 表
```sql
-- 修改前
INSERT INTO dreams (title, description, image_url, ..., achieved, ...)

-- 修改后
INSERT INTO dreams (user_id, title, description, image_url, ..., is_achieved, ...)
VALUES ('user-1', ?, ?, ?, ?, ?, ...)
```

**注意**：dreams 表还修复了字段名：
- `achieved` → `is_achieved`（与表定义一致）

### 删除旧数据库

```bash
rm -rf ~/Library/Application\ Support/@wealth-freedom/desktop/
```
