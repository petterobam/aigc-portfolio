# 📦 归档目录（.archive）

> 本目录存放历史记录、过期文档和已完成任务的归档文件。  
> 归档内容**不影响当前工作流**，仅用于回溯和审计。

---

## 目录结构

| 子目录 | 内容说明 | 文件数量级 |
|--------|----------|-----------|
| `cron-reports/` | Cron 定时任务执行总结（按日期命名） | ~5 |
| `heartbeat-reports/` | 历史心跳记录（#220-#256 及更早） | ~40+ |
| `action-lists/` | 自动生成的「立即行动清单」（内容高度重复，已归档） | ~10 |
| `weekly-reminders/` | 过期的周提醒和每周行动建议 | ~2 |
| `project-snapshots/` | 历史项目状态总结、交付清单、里程碑记录 | ~5 |
| `tech-fixes/` | 技术修复记录（数据库修复、API 迁移、图标修复等） | ~15 |
| `onboarding-drafts/` | 早期入门文档草稿（多版本重复，已合并保留 `快速开始指南.md`） | ~8 |
| `backups/` | 历史数据备份快照 | ~2 |

## 归档来源

### cron-reports/
Cron 任务（`wealth-freedom-cron`）自动生成的执行报告，记录每次定时任务的完成情况和系统状态。

### heartbeat-reports/
AI 心跳机制产生的执行记录。每次「心跳」会自主思考并推进项目，这些是历史心跳的详细日志。合并自原 `归档记录/心跳记录/` 和根目录散落的心跳报告。

### action-lists/
Cron 和心跳系统自动生成的行动清单。由于模板化生成，内容高度雷同（主要重复催促 GitHub 部署 + 文章发布 + 财务优化三件事），已统一归档。

### weekly-reminders/
按周生成的晨间提醒和行动建议，过期后归档。

### project-snapshots/
项目不同阶段的状态快照，包括：
- 项目总览、项目里程碑
- 项目交付清单、产品交付清单
- 各时间点的项目状态总结
- 财务自由加速器整合框架早期版本

### tech-fixes/
产品开发过程中的 Bug 修复记录，包括数据库表结构修复、API 迁移、UI 组件修复等。合并自原 `归档记录/技术修复记录/`。

### onboarding-drafts/
项目早期产生的多个版本入门文档（1 分钟开始、今天就开始、立即开始、为什么现在开始等），内容相似度高。已精选 `快速开始指南.md` 保留在项目根目录，其余归档于此。

### backups/
系统状态备份数据。

## 使用指南

- **查找历史心跳记录**：在 `heartbeat-reports/` 中按编号或日期搜索
- **回溯项目进展**：查看 `project-snapshots/` 中的阶段性总结
- **排查历史 Bug**：在 `tech-fixes/` 中搜索相关关键词
- **恢复归档文件**：直接将文件移回对应的活跃目录即可

> ⚠️ 归档文件只移动不删除。如需恢复，将文件从此目录移回项目对应位置。
