### 第三步：代码分析

深入分析 `export.ts` 中的导出逻辑：

```typescript
// ❌ 错误的实现
await new Promise<void>((resolve, reject) => {
  link.addEventListener('click', () => {
    setTimeout(() => {
      document.body.removeChild(link)
      console.log(`图表导出成功: ${filename}`)
      resolve()  // ← 永远不会执行
    }, 100)
  })
})

link.click()
```

**问题分析**：
1. `addEventListener('click', ...)` 事件监听器永远不会被触发
2. Promise 永远不会 resolve，导致 `await` 无限等待
3. 后续图表的导出被阻塞

**为什么 `addEventListener('click')` 不会触发？**
- 这种通过 JavaScript 创建 `<a>` 标签并触发 `click()` 的方式，在某些浏览器中：
  - `click()` 事件可能会被浏览器优化掉
  - 事件监听器可能没有被正确注册
  - 下载是同步的，但事件监听是异步的

### 第四步：修复方案

简化导出逻辑，移除不可靠的事件监听：

```typescript
// ✅ 正确的实现
// 创建下载链接
const link = document.createElement('a')
link.href = dataURL
link.download = `${filename}.${format}`

// 添加到 DOM
document.body.appendChild(link)

// 触发下载（不等待下载完成，直接触发即可）
link.click()

// 等待一小段时间确保下载开始
await new Promise(resolve => setTimeout(resolve, 100))

// 清理 DOM
try {
  document.body.removeChild(link)
} catch (e) {
  // 元素可能已经被移除，忽略
}

console.log(`图表导出成功: ${filename}`)
```

**修复要点**：
1. 移除 `addEventListener('click', ...)` 的逻辑
2. 直接调用 `link.click()` 触发下载
3. 等待 100ms 确保下载开始
4. 清理 DOM 元素
5. **不再等待下载完成**，因为浏览器下载是异步且无法可靠监听的
