# 增长曲线导出问题修复报告

## 问题分析

### 原始问题
- 资产配置可视化工具中，饼图和雷达图可以正常导出为图片
- 增长曲线图表导出失败，数据 URL 为空或无效

### 根本原因

经过详细调试，发现以下三个关键问题：

1. **图表容器高度问题**（主要问题）
   - 增长曲线的 DOM 结构中，图表容器所在的 `.finance-card` 缺少 `.chart-card` 类
   - 导致没有应用 `height: 400px;` 样式
   - Canvas 元素高度为 0（908x0），导致图表无法正确渲染
   - `getDataURL()` 方法返回空数据 URL

2. **图表渲染时机问题**
   - 调用 `getDataURL()` 时，图表可能还没有完全完成渲染
   - 需要强制刷新图表并等待渲染完成

3. **代码问题**
   - `exportChartToImage` 函数中使用了 `await`，但函数声明为普通函数（而非 async）
   - 导致编译错误，阻止了应用正常运行

### 修复 2：强制图表重新渲染并等待

**文件**: `apps/desktop/src/renderer/src/utils/export.ts`

**修改内容**:
```typescript
// 强制图表重新渲染（解决 getDataURL 返回空的问题）
try {
  chartInstance.resize()
} catch (e) {
  // resize 可能失败，忽略
}

// 等待图表渲染完成
await new Promise(resolve => setTimeout(resolve, 300))
```

**效果**: 在调用 `getDataURL()` 之前强制刷新图表并等待 300ms，确保图表完全渲染

### 修复 3：修正函数声明

**文件**: `apps/desktop/src/renderer/src/utils/export.ts`

**修改前**:
```typescript
export function exportChartToImage(
  chartInstance: any,
  filename: string,
  format: 'png' | 'jpeg' = 'png'
): Promise<void> {
  return new Promise((resolve, reject) => {
    // ...
    await new Promise(resolve => setTimeout(resolve, 300)) // ❌ 错误：不能在非 async 函数中使用 await
  })
}
```

**修改后**:
```typescript
export async function exportChartToImage(
  chartInstance: any,
  filename: string,
  format: 'png' | 'jpeg' = 'png'
): Promise<void> {
  try {
    // ...
    await new Promise(resolve => setTimeout(resolve, 300)) // ✅ 正确：在 async 函数中使用 await
  } catch (error) {
    throw error
  }
}
```

**效果**: 修复了编译错误，代码可以正常运行

### 修复 4：调整事件监听器添加顺序

**文件**: `apps/desktop/src/renderer/src/utils/export.ts`

**修改前**:
```typescript
// 监听下载完成事件
await new Promise<void>((resolve, reject) => {
  link.addEventListener('click', () => {
    // ...
  })
})

// 添加到 DOM 并触发下载
document.body.appendChild(link)
link.click()
```

**修改后**:
```typescript

// 监听下载完成事件
await new Promise<void>((resolve, reject) => {
  link.addEventListener('click', () => {
    // ...
  })
})

// 触发下载
link.click()
```

**效果**: 确保事件监听器在元素添加到 DOM 后、触发点击事件前正确设置

## 技术细节

### 为什么 `resize()` 能解决 `getDataURL()` 返回空的问题？

ECharts 的 `getDataURL()` 方法会调用 Canvas 的 `toDataURL()` API。如果 Canvas 的尺寸为 0，或者图表还没有完成渲染，`toDataURL()` 可能返回空数据或抛出错误。

调用 `resize()` 会强制 ECharts 重新计算并渲染图表，确保 Canvas 有正确的尺寸和内容。配合 `setTimeout` 等待，可以确保图表完全渲染完成后再调用 `getDataURL()`。

### 为什么需要等待 300ms？

现代浏览器的渲染是异步的。虽然 `resize()` 方法会立即触发 ECharts 的重新渲染逻辑，但实际的 DOM 更新和 Canvas 绘制可能需要几毫秒的时间。

等待 300ms 是一个保守的估计，足以确保图表完全渲染完成。如果测试发现这个时间可以减少，可以优化为更短的等待时间。
