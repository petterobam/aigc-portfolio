# 产品启动故障排除指南

> 如果产品无法启动，请按照此指南操作

---

## 问题 1：端口被占用

### 症状
```
Error: Port 5173 is already in use
```

### 原因
之前启动的 Electron 进程未正常关闭，仍在后台占用端口。

### 解决方案

#### 方法 1：关闭遗留进程（推荐）

打开终端，运行：

```bash
# 关闭所有 Electron 相关进程
pkill -f Electron
pkill -f electron-vite

# 等待 2 秒
sleep 2

# 重新启动产品
cd ~/.openclaw/workspace/财富自由之路/产品研发/code/wealth-freedom
./start.sh
```

#### 方法 2：重启电脑

如果方法 1 无效，重启电脑后再运行 `./start.sh`。

## 问题 2：模块格式错误

### 症状
```
SyntaxError: Unexpected token 'export'
```

### 原因
shared 包构建格式不兼容 Electron 主进程。

### 状态
✅ 已修复（2026-03-22 21:01）

如果遇到此问题，请确保：
1. 拉取了最新代码
2. 重新构建 shared 包：
   ```bash
   cd ~/.openclaw/workspace/财富自由之路/产品研发/code/wealth-freedom/packages/shared
   pnpm build
   ```

## 问题 3：依赖安装失败

### 症状
```
pnpm: command not found
```

### 解决方案

安装 pnpm：
```bash
npm install -g pnpm
```

然后重新运行 `./start.sh`。

---

## 问题 4：窗口闪退

### 症状
应用启动后立即关闭，没有错误信息。

### 可能原因
1. 数据库初始化失败
2. 渲染进程加载失败

### 调试方法

1. 查看控制台日志：
   ```bash
   cd ~/.openclaw/workspace/财富自由之路/产品研发/code/wealth-freedom/apps/desktop
   pnpm dev
   ```

2. 查看应用数据目录：
   ```bash
   ls ~/Library/Application\ Support/@wealth-freedom/desktop/
   ```

3. 删除应用数据重新启动：
   ```bash
   rm -rf ~/Library/Application\ Support/@wealth-freedom/desktop/
   ```
