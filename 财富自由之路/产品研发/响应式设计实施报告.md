**文件**：`apps/desktop/src/renderer/src/utils/responsive.ts`

**功能**：
- 定义断点系统（xs、sm、md、lg、xl、xxl）
- 提供断点判断函数
- 提供响应式布局工具函数

  - `getCardGridColumns()` - 获取卡片网格列数

- xs: 0px - 超小屏幕（手机竖屏）
- sm: 576px - 小屏幕（手机横屏）
- md: 768px - 中等屏幕（平板）
- lg: 992px - 大屏幕（桌面）
- xl: 1200px - 超大屏幕（大桌面）
- xxl: 1400px - 超超大屏幕（超大桌面）

### 3. 更新 App.vue（侧边栏响应式）

**文件**：`apps/desktop/src/renderer/src/App.vue`

**修改内容**：

1. **添加移动端菜单按钮**
   - 固定在左上角
   - 仅在移动端（< 576px）显示
   - 点击切换侧边栏显示/隐藏

2. **侧边栏响应式处理**
   - 移动端：隐藏侧边栏，点击菜单按钮显示
   - 小屏：180px 宽度
   - 平板：200px 宽度
   - 桌面：220px 宽度
   - 大屏/超大屏：240px 宽度
   - 支持折叠模式（图标仅显示）

3. **菜单折叠支持**
   - 使用 Element Plus Menu 的 `collapse` 属性
   - 侧边栏宽度动态调整（0px / 宽度）
   - 过渡动画（transition: width 0.3s ease）

4. **主内容区响应式**
   - 移动端：16px padding
   - 小屏：16px padding
   - 平板及以上：20px 或 24px padding
   - 动态 padding（基于响应式工具函数）

**新增脚本逻辑**：
```typescript
const sidebarVisible = ref(true)
const sidebarWidth = ref(220)
const mainPadding = ref('20px')

const updateResponsiveLayout = () => {
  const breakpoint = getCurrentBreakpoint()

  // 移动端：隐藏侧边栏
  if (breakpoint === 'xs' || breakpoint === 'sm') {
    sidebarVisible.value = false
    sidebarWidth.value = 0
  } else {
    sidebarVisible.value = true
    sidebarWidth.value = getSidebarWidth()
  }

  // 更新主内容区 padding
  mainPadding.value = getMainPadding()
}

const toggleSidebar = () => {
  sidebarVisible.value = !sidebarVisible.value
  sidebarWidth.value = sidebarVisible.value ? getSidebarWidth() : 0
}

onMounted(() => {
  updateResponsiveLayout()
  const cleanup = onWindowResize(updateResponsiveLayout)
  onUnmounted(cleanup)
})
```

## 响应式设计原则

### 断点策略

| 断点 | 屏幕宽度 | 设备类型 | 网格列数 | 侧边栏 |
|------|---------|---------|---------|--------|
| xs | 0-575px | 手机竖屏 | 1 列 | 隐藏 |
| sm | 576-767px | 手机横屏 | 2 列 | 180px |
| md | 768-991px | 平板 | 3 列 | 200px |
| lg | 992-1199px | 桌面 | 全部列数 | 220px |
| xl | 1200-1399px | 大桌面 | 全部列数 | 240px |
| xxl | ≥1400px | 超大桌面 | 全部列数 | 240px |

### 布局策略

**移动优先**（Mobile First）：
- 基础样式：移动端（单列布局）
- 使用 `min-width` 媒体查询逐步增强
- 优先优化小屏幕体验

**渐进增强**（Progressive Enhancement）：
- 基础功能在所有设备可用
- 在更大屏幕上逐步增加特性
- 平滑过渡，不跳跃

**内容优先**（Content First）：
- 关键内容在小屏幕完整显示
- 次要内容可折叠或隐藏
- 避免横向滚动

## 性能考虑

1. **CSS 媒体查询**：
   - 使用 `min-width` 优于 `max-width`（移动优先）
   - 避免复杂的 CSS 选择器
   - 使用 CSS Grid 和 Flexbox 代替 float

2. **JavaScript 监听**：
   - 使用防抖（debounce）减少 resize 事件触发
   - 及时清理事件监听器（`onUnmounted`）
   - 避免频繁 DOM 操作

3. **图表优化**：
   - 按屏幕尺寸调整数据精度
   - 减少动画复杂度
   - 懒加载非关键图表
