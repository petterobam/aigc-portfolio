## 测试方法

### 测试脚本

使用 Playwright 在真实浏览器环境中执行测试：

```javascript
// 1. 启动浏览器（非 headless）
const browser = await chromium.launch({
  headless: false,
  slowMo: 500,

await page.waitForTimeout(3000);

// 3. 检查图表容器和 Canvas 尺寸

// 4. 模拟点击导出按钮

await exportButton.click();
await page.waitForTimeout(3000);

// 5. 检查下载的文件
const files = fs.readdirSync(downloadDir);
const imageFiles = files.filter(file => /* ... */);
```

### 关键发现

1. **Playwright 的 `waitForEvent('download')` 无法捕获通过 JavaScript 创建 `<a>` 标签触发的下载**
   - 解决方法：直接检查下载目录中的文件
   - 原因：Playwright 的下载事件监听机制与 JavaScript 创建的下载链接不兼容

2. **增长曲线图表容器高度修复成功**
   - 修复前：Canvas 高度为 0（908x0）
   - 修复后：Canvas 高度为 400px（908x400）
   - 修复方法：为增长曲线容器添加 `.chart-card` 类
