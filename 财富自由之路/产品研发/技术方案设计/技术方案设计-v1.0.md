### 1.2 分层架构说明

#### 渲染进程（Renderer）

**View 层**：
- 负责页面布局和交互
- 职责：UI 渲染、事件处理、表单验证
- 不包含业务逻辑，只负责展示和用户交互

**Component 层**：
- 通用组件库
- 职责：可复用的 UI 组件（卡片、按钮、表单等）
- 通过 props 和 events 与父组件通信

**Utils 层**：
- 工具函数
- 职责：日期格式化、金额格式化、数据转换等纯函数

**State 层**：
- Pinia 状态管理
- 职责：跨组件状态共享、异步数据管理、缓存策略
- 通过 IPC 与主进程通信获取数据

#### 主进程（Main）

**IPC 层**：
- 进程间通信接口
- 职责：定义通信协议、参数验证、错误处理
- 使用 `ipcMain.handle` 接收渲染进程请求

**Service 层**：
- 业务逻辑层
- 职责：核心业务规则、数据聚合、复杂计算
- 调用 Repository 层获取数据

**Utils 层**：
- 工具函数
- 职责：加密解密、文件操作、日期计算等

**Data 层**：
- 数据访问层（Repository 模式）
- 职责：封装数据库操作、数据验证、事务管理
- 每个 Repository 负责一个实体的 CRUD

1. **清晰的应用边界**：桌面应用和移动应用完全隔离

4. **可维护性提升**：每个模块职责单一，易于测试和维护

## 2. 数据库设计

### 2.1 数据库表结构（完整版）

#### 2.1.1 用户表（users）

```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT UNIQUE,
  avatar TEXT,
  currency TEXT DEFAULT 'CNY',

  -- 财务参数
  guarantee_months INTEGER DEFAULT 6,           -- 财务保障月数
  expected_return_rate REAL DEFAULT 8.0,        -- 预期年化收益率
  monthly_basic_living REAL,                     -- 财务安全月支出
  monthly_dream_living REAL,                     -- 财务自由月支出

  -- 其他设置
  settings TEXT,                                -- JSON 格式存储其他设置
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,

  -- 索引
  INDEX idx_users_email (email)
);
```

**字段说明**：
- `id`: 用户唯一标识（UUID）
- `guarantee_months`: 财务保障目标月数（6-12 个月）
- `expected_return_rate`: 预期年化收益率，用于计算财务安全/自由目标
- `monthly_basic_living`: 财务安全阶段的基本生活月支出
- `monthly_dream_living`: 财务自由阶段的梦想生活月支出
- `settings`: JSON 格式存储其他设置（主题、语言、通知等）

#### 2.1.2 账户表（accounts）

```sql
CREATE TABLE accounts (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  name TEXT NOT NULL,
  type TEXT NOT NULL,                            -- 现金/存款/投资/固定资产/其他
  icon TEXT,
  color TEXT,

  -- 余额信息
  balance REAL DEFAULT 0,
  currency TEXT DEFAULT 'CNY',

  -- 其他信息
  institution TEXT,                             -- 机构名称（银行、券商等）
  account_number TEXT,                          -- 账号（加密存储）
  notes TEXT,

  -- 标记
  is_reserved INTEGER DEFAULT 0,                -- 是否为储备金账户
  include_in_net_worth INTEGER DEFAULT 1,        -- 是否计入净资产

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,

  -- 索引
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  INDEX idx_accounts_user (user_id),
  INDEX idx_accounts_type (type)
);
```

**字段说明**：
- `type`: 账户类型（现金、存款、投资、固定资产、其他）
- `is_reserved`: 是否为储备金账户（用于财务保障）
- `include_in_net_worth`: 是否计入净资产（有些资产可能不想计入）
- `account_number`: 账号（加密存储，保证隐私）

```sql

  interest_rate REAL,                            -- 年化利率

  -- 优先级和状态
```

**字段说明**：

#### 2.1.4 交易记录表（transactions）

```sql
CREATE TABLE transactions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  account_id TEXT NOT NULL,

  -- 交易信息
  type TEXT NOT NULL,                            -- 收入/支出/转账
  category TEXT NOT NULL,                         -- 分类
  subcategory TEXT,                              -- 子分类
  amount REAL NOT NULL,                           -- 金额
  date DATE NOT NULL,                            -- 交易日期

  -- 描述和标签
  note TEXT,
  description TEXT,
  tags TEXT,                                     -- JSON 数组格式

  -- 附加信息
  related_transaction_id TEXT,                   -- 关联交易（转账等）
  is_recurring INTEGER DEFAULT 0,                -- 是否为周期性交易
  recurring_rule TEXT,                           -- 周期规则（JSON）

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,

  -- 索引
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (account_id) REFERENCES accounts(id) ON DELETE CASCADE,
  INDEX idx_transactions_user_date (user_id, date),
  INDEX idx_transactions_account (account_id),
  INDEX idx_transactions_type (type),
  INDEX idx_transactions_category (category)
);
```

**字段说明**：
- `type`: 交易类型（收入、支出、转账）
- `category`: 分类（工资、餐饮、交通、购物等）
- `tags`: 标签（JSON 数组格式，方便多标签查询）
- `related_transaction_id`: 关联交易（如转账时会有两笔交易关联）
- `is_recurring`: 是否为周期性交易（如每月的固定支出）

**聚合查询优化**：
```sql
-- 使用物化视图或预聚合表
CREATE TABLE monthly_summary (
  user_id TEXT NOT NULL,
  year INTEGER NOT NULL,
  month INTEGER NOT NULL,
  income REAL DEFAULT 0,
  expense REAL DEFAULT 0,
  balance REAL DEFAULT 0,
  PRIMARY KEY (user_id, year, month)
);

-- 触发器自动更新月度汇总
CREATE TRIGGER update_monthly_summary
AFTER INSERT ON transactions
BEGIN
  INSERT OR REPLACE INTO monthly_summary (user_id, year, month, income, expense, balance)
  VALUES (
    NEW.user_id,
    strftime('%Y', NEW.date),
    strftime('%m', NEW.date),
    CASE WHEN NEW.type = 'income' THEN NEW.amount ELSE 0 END,
    CASE WHEN NEW.type = 'expense' THEN NEW.amount ELSE 0 END,
    (SELECT COALESCE(SUM(CASE WHEN type = 'income' THEN amount ELSE -amount END), 0)
     FROM transactions
     WHERE user_id = NEW.user_id
       AND strftime('%Y', date) = strftime('%Y', NEW.date)
       AND strftime('%m', date) = strftime('%m', NEW.date))
  );
END;
```

### 2.3 数据迁移策略

**版本管理**：
```typescript
// 数据库版本号存储在 metadata 表
CREATE TABLE metadata (
  key TEXT PRIMARY KEY,
  value TEXT
);

INSERT INTO metadata (key, value) VALUES ('db_version', '1.0');
```

**迁移脚本**：
```typescript
const migrations = [
  {
    version: '1.1',
    sql: `
      ALTER TABLE accounts ADD COLUMN account_number TEXT;
      ALTER TABLE transactions ADD COLUMN subcategory TEXT;
    `
  },
  {
    version: '1.2',
    sql: `
      CREATE TABLE budgets (...);
      CREATE TABLE reminders (...);
    `
  }
];

export async function migrate(db: Database.Database) {
  const currentVersion = db.prepare('SELECT value FROM metadata WHERE key = ?').get('db_version') || '1.0';

  for (const migration of migrations) {
    if (compareVersions(currentVersion, migration.version) < 0) {
      db.exec(migration.sql);
      db.prepare('UPDATE metadata SET value = ? WHERE key = ?').run(migration.version, 'db_version');
    }
  }
}
```

## 3. API 设计

### 3.1 IPC 通信协议

#### 3.1.1 通信规范

**命名规则**：
- 获取数据：`get{Entity}`
- 创建数据：`create{Entity}`

## 6. 安全设计

### 6.1 数据加密

#### 6.1.1 敏感数据加密

**适用场景**：账号、密码、备注

```typescript
import crypto from 'crypto';

const algorithm = 'aes-256-gcm';
const key = crypto.scryptSync('user-password', 'salt', 32); // 从用户密码派生密钥

// 加密
function encrypt(text: string): { encrypted: string; iv: string; authTag: string } {
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv(algorithm, key, iv);
  let encrypted = cipher.update(text, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  const authTag = cipher.getAuthTag();

  return {
    encrypted,
    iv: iv.toString('hex'),
    authTag: authTag.toString('hex')
  };
}

// 解密
function decrypt(encrypted: string, iv: string, authTag: string): string {
  const decipher = crypto.createDecipheriv(algorithm, key, Buffer.from(iv, 'hex'));
  decipher.setAuthTag(Buffer.from(authTag, 'hex'));
  let decrypted = decipher.update(encrypted, 'hex', 'utf8');
  decrypted += decipher.final('utf8');
  return decrypted;
}
```

```typescript
class PluginManager {
  private plugins: Map<string, Plugin> = new Map();

  async install(plugin: Plugin) {
    if (this.plugins.has(plugin.name)) {
      throw new Error(`Plugin ${plugin.name} already installed`);
    }

    await plugin.install(app);
    this.plugins.set(plugin.name, plugin);

    if (plugin.onLoad) {
      await plugin.onLoad();
    }
  }

  async uninstall(pluginName: string) {
    const plugin = this.plugins.get(pluginName);

    if (!plugin) {
      throw new Error(`Plugin ${pluginName} not found`);
    }

    if (plugin.onUnload) {
      await plugin.onUnload();
    }

    plugin.uninstall(app);
    this.plugins.delete(pluginName);
  }

  getPlugin(pluginName: string): Plugin | undefined {
    return this.plugins.get(pluginName);
  }

  listPlugins(): Plugin[] {
    return Array.from(this.plugins.values());
  }
}
```

```typescript
class ReportEngine {
  async executeReport(template: ReportTemplate) {
    let data: any;

    // 执行查询
    if (template.query.type === 'sql') {
      data = await this.executeSql(template.query.sql!, template.query.params);
    } else if (template.query.type === 'api') {
      data = await this.callApi(template.query.api!, template.query.params);
    }

    // 返回报表数据
    return {
      data,
      visualization: template.visualization
    };
  }

  private async executeSql(sql: string, params?: Record<string, any>) {
    const stmt = db.prepare(sql);
    return stmt.all(params);
  }

  private async callApi(api: string, params?: Record<string, any>) {
    const response = await ipcRenderer.invoke(api, params);
    return response.data;
  }
}
```

```typescript
async function sync() {
  // 1. 获取本地未同步的数据
  const localChanges = await getLocalChanges();
```

### 9.3 数据加密

**端到端加密**：
```typescript
import crypto from 'crypto';

// 生成加密密钥
const encryptionKey = crypto.randomBytes(32);

// 加密数据
function encryptData(data: string, key: Buffer): { encrypted: string; iv: string } {
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);
  let encrypted = cipher.update(data, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  return {
    encrypted,
    iv: iv.toString('hex')
  };
}

// 解密数据
function decryptData(encrypted: string, iv: string, key: Buffer): string {
  const decipher = crypto.createDecipheriv('aes-256-gcm', key, Buffer.from(iv, 'hex'));
  let decrypted = decipher.update(encrypted, 'hex', 'utf8');
  decrypted += decipher.final('utf8');
  return decrypted;
}
```
