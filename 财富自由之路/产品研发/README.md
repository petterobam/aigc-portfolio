# 产品研发

> **产品定位**: 帮助普通人走向财务自由的个人财务管理软件（PC + 移动端）
> **核心价值**: 让财务自由不只是概念，而是每天可量化、可追踪、可行动的目标

---

## 产品愿景

大多数人不是没有能力实现财务自由，而是缺少一个系统帮他们看清现状、拆解目标、持续行动。

这款软件要做的，就是成为用户走向财务自由路上最可靠的工具。

---

## 三大核心问题

1. **我现在在哪里？** — 财务现状看板（净资产、收支、负债）
2. **我要去哪里？** — 财务自由目标计算器（三阶段目标金额）
3. **我怎么到达？** — 行动追踪 + AI 建议 + 进度可视化

---

## 核心功能模块

### 模块一：财务现状看板
- 净资产实时计算（资产 - 负债）
- 月收入 / 月支出 / 月结余追踪
- 负债清单管理（类型、金额、利率、还款进度）
- 资产分类（现金、投资、固定资产等）

### 模块二：财务自由目标计算器
- 财务保障目标：月支出 × 保障月数（6~12 个月）
- 财务安全目标：月所需金额 × 150（8% 年化）
- 财务自由目标：梦想月成本 × 150（8% 年化）
- 距离目标的进度条与预计达成时间

### 模块三：每日记账
- 快速记账（3 秒内完成一笔）
- 支出分类（必要支出 vs 消费欲望支出）
- 消费债预警（识别不必要的提前消费）
- 月度支出报告

### 模块四：投资记录与收益追踪
- 投资组合管理（40:40:20 原则可视化）
- 收益率追踪（年化、累计）
- 被动收入 vs 主动收入占比图
- 距离"利息覆盖支出"的进度

### 模块五：AI 财务顾问
- 基于个人财务数据给出行动建议
- 识别高风险消费行为并预警
- 个性化制定达到财务自由的路径规划

---

## 平台规划

### PC 端
- 深度数据录入与分析
- 财务报表生成与导出
- 投资组合管理
- 目标规划与路径计算

### 移动端
- 日常快速记账
- 财务自由进度查看
- 消费提醒与预警推送
- 数据与 PC 端实时同步

## MVP 范围（第一版）

最小可用产品，优先验证核心价值。

**必须有**：
- [ ] 财务现状录入（净资产 = 资产 - 负债）
- [ ] 三阶段目标金额计算器
- [ ] 每日记账（收入 / 支出分类）
- [ ] 财务自由进度条
- [ ] 月度收支报告

**暂不做**：
- AI 顾问（MVP 后迭代）
- 跨平台数据同步（先做单平台）
- 投资组合深度分析（先做基础版）

## 技术选型方向（待定）

| 端 | 候选方案 | 备注 |
|----|---------|------|
| PC 端 | Electron + React / Tauri + React | Tauri 更轻量，Electron 生态更成熟 |
| 移动端 | React Native / Flutter | 跨平台首选 |
| 数据库 | SQLite（本地）+ 云同步 | 隐私优先，本地存储 |
| 后端（可选）| Node.js / Python FastAPI | 如需云同步或 AI 功能 |

## 变现路径

| 模式 | 描述 | 优先级 |
|------|------|-------|
| 一次性买断 | PC 端买断制，低门槛获客 | 优先 |
| 订阅制 | 移动端月订阅，含 AI 顾问功能 | 中期 |
| 增值服务 | 财务咨询、课程联动 | 后期 |

## 竞品参考

| 竞品 | 优点 | 不足 | 我们的差异化 |
|------|------|------|------------|
| 随手记 | 记账功能完善 | 缺少财务自由目标体系 | 三阶段目标 + 路径规划 |
| YNAB | 预算理念强 | 学习成本高，价格贵 | 更简单，以财务自由为核心 |
| 挖财 | 用户基数大 | 广告多，体验差 | 纯净工具，数据本地优先 |
| MoneyWatcher | 净资产追踪好 | 功能单一 | 更完整的财务自由路径 |
