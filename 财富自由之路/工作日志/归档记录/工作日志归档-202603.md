## 归档记录 #057

**日期**: 2026-03-22 21:01
**主要内容**: 修复产品启动配置问题

1. **修复 shared 包构建配置**
   - 问题：TypeScript 编译输出 ES Modules 格式，与 Electron 主进程不兼容
   - 解决：修改 `tsconfig.json`，将 `module` 从 `ESNext` 改为 `CommonJS`
   - 结果：shared 包成功构建为 CommonJS 格式

2. **修复 electron-vite 端口配置**
   - 问题：端口 5173 被占用时，Vite 自动切换到其他端口，但主进程代码硬编码了 5173
   - 解决：在 `electron.vite.config.ts` 中添加 `server.port: 5173` 和 `strictPort: true`
   - 结果：端口被占用时会报错，而不是静默切换

3. **发现遗留进程问题**
   - 发现多个 Electron 进程在后台运行，占用端口 5173
   - 这些进程是之前启动应用时遗留的
   - 建议：用户需要手动关闭这些进程或重启电脑

**技术细节**:
- shared/tsconfig.json: module: "CommonJS", moduleResolution: "node", declaration: true
- electron.vite.config.ts: server.port: 5173, strictPort: true

**产品状态**: 产品代码 100%，方法论 100%，知识库 100%，启动测试 90%（受阻）

**下次方向**: 等待用户清理遗留进程 → 重新启动产品 → 引导录入财务数据
