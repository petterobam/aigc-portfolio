## 关键洞察

### 1. favicon 设计考虑

**设计理念**：
- 金币：象征财富，符合产品定位
- 上升箭头：象征增长和进步
- ¥ 符号：明确货币属性，面向中文用户

**技术选择**：
- SVG 作为源文件：可缩放，易于修改
- ICO 作为生产文件：浏览器广泛支持
- 32x32 像素：标准 favicon 尺寸，兼容性好

**品牌一致性**：
- 颜色方案：金色（#FFD700）+ 深棕色（#8B4513）
- 与产品主题"财富自由"高度契合
- 简洁明了，易于识别

### 2. 构建错误修复经验

**常见错误类型**：
- HTML 属性值中的中文引号问题
- Vue 文件中的重复标签问题
- 图标库 API 变更导致的导入错误

**修复策略**：
- 仔细阅读错误信息，定位具体问题
- 使用 grep 查找所有相关文件
- 系统性修复所有同类问题
- 逐个验证修复效果

**预防措施**：
- 代码审查时注意语法规范
- 使用 ESLint 等工具进行静态检查
- 定期运行构建测试
- 关注依赖库的版本更新

### 3. Vite public 目录机制

**Vite 构建流程**：
- 源文件：`src/renderer/public/` 目录
- 构建过程：自动复制 public 目录下的文件到输出目录
- 输出位置：`dist/renderer/` 目录

**优势**：
- 静态资源管理简单
- 无需额外配置
- 自动复制，无需手动处理

**最佳实践**：
- 将不需要编译的静态文件放在 public 目录
- 包括 favicon、robots.txt、manifest.json 等
- 避免在 public 目录中放置需要编译的文件
