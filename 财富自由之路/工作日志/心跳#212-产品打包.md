## 关键洞察

### 1. 打包配置的细节很重要

**问题**：electron-builder 打包失败，提示找不到入口文件

**原因**：
- electron-builder 需要打包所有必要的文件（main、preload、renderer）
- `electron-builder.yml` 中的 `files` 字段定义了要打包的文件
- 原配置只包含 `dist/**/*`，缺少 `out/**/*`

**修复**：
```yaml
files:
  - dist/**/*      # renderer 构建输出
  - out/**/*       # main 和 preload 构建输出
  - package.json
```

**教训**：打包配置需要仔细检查，确保所有必要的文件都被包含。

### 2. 图标文件需要准备多种格式

**为什么需要多种图标格式？**
- 不同操作系统有不同的图标格式要求
- macOS：`.icns`（包含多种尺寸）
- Windows：`.ico`（支持多尺寸）
- Linux：`.png`（通常 512x512）

**图标生成策略**：
- 从 SVG 源文件生成，保持矢量清晰度
- 使用自动化脚本批量生成，提高效率
- 生成不同尺寸的图标，适配不同场景

**图标尺寸要求**：
- macOS：16x16 到 1024x1024（视网膜显示器需要 2x、3x）
- Windows：256x256（通常足够）
- Linux：512x512（推荐）

### 3. 首次打包需要下载 Electron 运行时

**现象**：打包时需要下载 Electron 文件（x64: 100MB, arm64: 95MB）

**原因**：
- electron-builder 需要下载特定版本的 Electron 运行时
- 下载到全局缓存目录，后续打包不需要重新下载
- 不同架构（x64、arm64）需要下载不同的版本

**影响**：
- 首次打包耗时较长（15-20分钟）
- 后续打包会更快（使用缓存的运行时）

**优化建议**：
- 提前下载好所需的 Electron 版本
- 使用国内镜像加速下载（如淘宝 npm 镜像）
- 并行打包多个架构版本

## 技术细节

### 1. electron-builder 配置

```yaml
appId: com.wealth-freedom.app
productName: 财富自由之路

directories:
  buildResources: build
  output: release

files:
  - dist/**/*
  - out/**/*       # 修复：添加 main 和 preload 构建输出
  - package.json

mac:
  category: public.app-category.finance
  target:
    - target: dmg
      arch:
        - x64
        - arm64
  icon: build/icon.icns
```

### 2. 图标生成脚本

```bash
#!/bin/bash

# 从 icon.svg 生成各平台图标

# 生成 icon.png (Linux)
magick icon.svg -resize 512x512 icon.png

# 生成 icon.ico (Windows)
magick icon.svg -resize 256x256 icon.ico

# 生成 icon.icns (macOS)
# 先创建 iconset
mkdir -p icon.iconset
magick icon.svg -resize 16x16 icon.iconset/icon_16x16.png
magick icon.svg -resize 32x32 icon.iconset/icon_32x32.png
# ... 其他尺寸

# 使用 iconutil 生成 icns
iconutil -c icns icon.iconset -o icon.icns
```

### 3. 打包命令

```bash
# 打包所有平台
pnpm build

# 打包特定平台
pnpm build --mac
pnpm build --win
pnpm build --linux
```

## 遇到的问题

### 1. 打包失败：找不到入口文件

**错误信息**：
```
Application entry file "out/main/index.js" does not exist. Seems like a wrong configuration.
```

**解决方案**：
在 `electron-builder.yml` 中添加 `out/**/*` 到 `files` 字段

### 2. better-sqlite3 预编译二进制下载超时

**现象**：打包时 better-sqlite3 的预编译二进制文件下载超时

**解决方案**：
- electron-builder 会自动从源代码编译
- 首次编译会耗时较长（5-10分钟）
- 后续打包会使用缓存的二进制文件

### 3. 缺少图标文件

**现象**：打包时找不到 icon.icns、icon.ico、icon.png

**解决方案**：
- 创建图标生成脚本 `generate-icons.sh`
- 从 icon.svg 生成各平台图标文件
