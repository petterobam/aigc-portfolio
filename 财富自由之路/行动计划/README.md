# 行动计划

> 目标落地的执行层，把知识和方法论转化为每天、每周、每月、每季度的具体行动。

---

## 目录结构

```
行动计划/
├── README.md          本文件（行动计划索引与使用说明）
├── 季度计划/           季度 OKR 与关键里程碑
├── 月度计划/           每月具体行动项与目标拆解
└── 周计划/             每周执行清单与复盘
```

---

## 使用原则

1. 计划要和目标对齐：每个行动项都应能回答「这对财务自由有什么帮助」
2. 从季度到周逐层拆解，不要直接跳到日常琐事
3. 每周末复盘，更新下周计划
4. 每月末复盘，更新下月计划，检验季度目标进度
5. 计划是活的，随时可以调整，但调整要记录原因

---

## 三阶段目标参考

在制定计划前，先明确自己当前所处阶段：

| 阶段 | 达成标准 | 当前状态 |
|------|---------|---------|
| 财务保障 | 拥有 6~12 个月生活储备金 | 待评估 |
| 财务安全 | 投资收益覆盖月支出（月支出 × 150 = 所需本金） | 待评估 |
| 财务自由 | 投资收益支撑梦想生活，本金永不动用 | 待评估 |

## 季度计划

路径：`季度计划/`

每季度制定 1 份 OKR 文档，包含：
- 本季度核心目标（不超过 3 个）
- 每个目标的关键结果（可量化）
- 产品研发里程碑
- 财务指标目标

文件命名规范：`YYYYQN_季度计划.md`（如 `2026Q2_季度计划.md`）

---

## 月度计划

路径：`月度计划/`

每月制定 1 份月度计划，包含：
- 本月财务目标（储蓄金额、还债金额、投资金额）
- 产品研发任务清单
- 知识学习目标（读几篇资料、提炼几条方法论）
- 月底复盘总结

文件命名规范：`YYYYMM_月度计划.md`（如 `202604_月度计划.md`）

---

## 周计划

路径：`周计划/`

每周制定 1 份执行清单，包含：
- 本周 3 件最重要的事
- 产品研发具体任务
- 财务操作（记账、转账、复核）
- 周末复盘（完成了什么、卡在哪里、下周调整什么）

文件命名规范：`YYYYWNN_周计划.md`（如 `2026W15_周计划.md`）

## 行动项优先级判断

面对一堆任务不知道从哪里开始，用这个问题筛选：

> 如果这周只能做一件事，哪件事对实现财务自由影响最大？

优先级排序原则：
1. 直接产生收入或减少支出的事
2. 推进产品研发里程碑的事
3. 提升认知和能力的事
4. 整理和归档类的事
