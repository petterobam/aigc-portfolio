# 财富自由之路 v0.1.0：个人财务软件开发实录

> 从财务自由理念到可落地的软件产品，我用 2 个月时间开发了一款个人财务软件。本文记录了从需求分析到产品发布的完整过程，分享了技术选型、开发经验、遇到的坑和解决方案。

---

## 写在前面

2026 年初，我给自己定了一个目标：在 5 年内实现财务自由。为了系统化管理个人财务，追踪进度，我决定自己开发一款个人财务软件。

这不是一个简单的记账 App，而是一个完整的**财务管理系统**，包括：

- **财务数据追踪**：收入、支出、资产、负债
- **财务目标设定**：财务保障、财务安全、财务自由
- **资产配置可视化**：根据不同财务阶段推荐资产配置
- **增长曲线预测**：基于复利计算预测何时达到财务目标

经过 2 个月的努力，我完成了 v0.1.0 版本的开发和发布。本文将记录整个过程，分享我的经验和技术方案。

### 2.2 财务目标设定

财务目标设定基于《财富自由之路》的财务三阶段理论：

1. **财务保障**：拥有 6~12 个月的生活储备金，不被意外击倒
2. **财务安全**：靠投资利息覆盖日常支出，不再被迫工作
3. **财务自由**：靠利息实现梦想生活，本金永不动用

**实现逻辑**：

```typescript
function calculateFinancialGoals(
  monthlyExpense: number,
  currentAssets: number
): FinancialGoals {
  // 财务保障：12 个月生活储备金
  const financialSecurity = monthlyExpense * 12;

  // 财务安全：基于 3.8% 年化收益，需要投资资产 = 月支出 × 316
  const financialSafety = monthlyExpense * 316;

  // 财务自由：财务安全的 2-3 倍（因人而异）
  const financialFreedom = financialSafety * 2;

  return {
    security: financialSecurity,
    safety: financialSafety,
    freedom: financialFreedom,
    progress: {
      security: (currentAssets / financialSecurity) * 100,
      safety: (currentAssets / financialSafety) * 100,
      freedom: (currentAssets / financialFreedom) * 100,
    },
  };
}
```

#### 2.3.3 增长曲线预测

基于复利计算预测何时达到财务目标：

**复利计算公式**：

```typescript
function calculateGrowth(
  currentAssets: number,
  monthlyContribution: number,
  annualReturn: number,
  target: number
): GrowthCurve {
  const monthlyReturn = annualReturn / 12 / 100;
  const months = [];

  for (let i = 0; i < 12 * 30; i++) {
    const month = i + 1;
    const assets = currentAssets * Math.pow(1 + monthlyReturn, month) +
                   monthlyContribution * ((Math.pow(1 + monthlyReturn, month) - 1) / monthlyReturn);

    months.push({
      month,
      year: Math.floor((month - 1) / 12) + 1,
      assets: Math.round(assets),
    });

    if (assets >= target) {
      break;
    }
  }

  return months;
}
```

### 2.4 数据导出

#### 2.4.1 PDF 导出

使用 `jspdf` 和 `html2canvas` 实现：

```typescript
import jsPDF from 'jspdf';
import html2canvas from 'html2canvas';

async function exportToPDF(elementId: string) {
  const element = document.getElementById(elementId);
  if (!element) return;

  const canvas = await html2canvas(element, {
    scale: 2,
    useCORS: true,
  });

  const imgData = canvas.toDataURL('image/png');
  const pdf = new jsPDF('p', 'mm', 'a4');
  const imgWidth = 210;
  const imgHeight = (canvas.height * imgWidth) / canvas.width;

  pdf.addImage(imgData, 'PNG', 0, 0, imgWidth, imgHeight);
  pdf.save('财务报告.pdf');
}
```

## 三、遇到的坑和解决方案

### 3.1 坑 1：Element Plus 废弃属性警告

**问题描述**：
使用 Element Plus 时，控制台出现大量废弃属性警告：
```
[Element Plus Warn]: 'label' is deprecated, use 'value' instead.
```

**原因**：
Element Plus 最新版本将 `el-radio` 和 `el-radio-button` 的 `label` 属性改为了 `value` 属性。

**解决方案**：
全局搜索替换 `label` → `value`，共修复 7 个文件中的 28 处废弃属性。

**修复示例**：

```vue
<!-- 修复前 -->
<el-radio-group v-model="selectedStage">
  <el-radio label="security">财务保障</el-radio>
  <el-radio label="safety">财务安全</el-radio>
  <el-radio label="freedom">财务自由</el-radio>
</el-radio-group>

<!-- 修复后 -->
<el-radio-group v-model="selectedStage">
  <el-radio value="security">财务保障</el-radio>
  <el-radio value="safety">财务安全</el-radio>
  <el-radio value="freedom">财务自由</el-radio>
</el-radio-group>
```

### 3.2 坑 2：增长曲线导出失败

**问题描述**：
导出增长曲线时，PDF 中增长曲线区域是空白。

**原因**：
ECharts 图表容器高度为 0px，导致 `html2canvas` 无法正确渲染。

**解决方案**：

1. **强制图表容器有高度**：
```css
#growth-chart-container {
  height: 400px;
  width: 100%;
}
```

2. **在导出前强制图表重新渲染**：
```typescript
async function exportChartToImage(chartId: string) {
  const chart = echarts.getInstanceByDom(document.getElementById(chartId));
  if (chart) {
    chart.resize();
    await new Promise(resolve => setTimeout(resolve, 100));
  }

  const canvas = await html2canvas(document.getElementById(chartId));
  const imgData = canvas.toDataURL('image/png');
  return imgData;
}
```

### 3.3 坑 3：导出逻辑 Promise 永不 resolve

**问题描述**：
导出 PDF 时，Promise 永远不会 resolve，导致无法下载。

**原因**：
代码中使用了 `addEventListener('click', ...)` 监听点击事件，但事件监听器永远不会触发。

**解决方案**：
移除 `addEventListener`，直接调用 `link.click()`：

```typescript
// 修复前
const link = document.createElement('a');
link.download = '财务报告.pdf';
link.addEventListener('click', () => {
  console.log('Download started');
});

// 修复后
const link = document.createElement('a');
link.href = imgData;
link.download = '财务报告.pdf';
link.click();
```

### 3.4 坑 4：滑块交互测试失败

**问题描述**：
使用 Playwright 测试滑块交互时，测试失败，无法拖拽滑块。

**原因**：
Element Plus 使用 `.el-slider` 类，而不是 `input[type="range"]`。

**解决方案**：
修改测试选择器：

```typescript
// 修复前
await page.fill('input[type="range"]', '50');

// 修复后
const slider = page.locator('.el-slider__runway');
await slider.click({ position: { x: 0.5, y: 0.5 } });
```

1. **React + TypeScript 真的很香**

3. **ECharts 数据可视化强大**
   - 图表种类丰富，配置灵活
   - 性能优秀，支持大数据量
   - 交互性好，支持动画

### 5.2 产品层面

1. **MVP 思维很重要**
   - 优先完成核心功能（P0）
   - 快速迭代，不要追求完美
   - 从用户反馈中优化产品

2. **用户体验至上**
   - 界面简洁，操作流畅
   - 数据可视化直观易懂
   - 响应式设计，支持不同分辨率

3. **数据安全不可忽视**
   - 本地数据存储，不联网
   - 数据导出功能，防止数据丢失
   - 定期提醒用户备份数据

### 5.3 财务层面

1. **财务目标清晰化**
   - 通过软件明确了财务目标（316 万）
   - 知道了当前进度（32.3%）
   - 预计 6-8 年达到财务安全

2. **资产配置科学化**
   - 根据不同财务阶段调整资产配置

3. **复利效应可视化**
   - 增长曲线展示了复利的魔力
   - 坚持投资，时间是最好的朋友
   - 每月定投，加速财富积累

1. **产品化是关键**
