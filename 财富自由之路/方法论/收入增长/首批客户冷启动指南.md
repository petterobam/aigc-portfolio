# 首批客户冷启动指南

> 从 0 到 5 个付费客户的实操路径
> 创建时间：2026-04-21 | 心跳 #dc748734

---

## 一、为什么冷启动最难

企业家路径第三阶段的核心挑战：**内容有影响力 ≠ 有客户愿意付费**。

- 影响力是「知道你」，客户获取是「信任你到愿意付钱」
- 从内容消费者 → 咨询客户，转化率通常 0.5%-2%
- 前 5 个客户的价值远超收入本身：验证需求、积累案例、建立口碑

**核心公式**：
```
客户数 = 流量 × 信任转化率 × 行动召唤转化率
```
- 流量：文章发布后自然获取（19篇 × 平台曝光）
- 信任转化率：通过免费价值建立（关键杠杆）
- 行动召唤转化率：明确的下一步行动引导

## 二、冷启动四步法（30 天计划）

### 第 1 周：种子池建立（0 成本）

**目标**：收集 20-30 个潜在客户联系方式

| 动作 | 具体操作 | 预计时间 |
|------|----------|----------|
| 朋友圈/社群预告 | 发布「AIGC 技术咨询内测招募」，限 5 个免费诊断名额 | 30 分钟 |
| 文章末尾植入 CTA | 每篇已发布文章末尾加「免费技术咨询 → 加微信/留言」 | 每篇 2 分钟 |
| GitHub README 更新 | 仓库 README 添加咨询服务说明和联系方式 | 15 分钟 |
| 个人网站添加咨询页 | 在 aigc-portfolio 添加「技术服务」页面 | 1 小时 |

**关键心态**：先免费提供价值，再谈付费。免费诊断是最好的信任建立工具。

### 第 2 周：免费诊断执行（时间成本）

**目标**：完成 5 次免费技术诊断，建立信任

**诊断框架（每次 30 分钟）**：

```
1. 现状了解（5 分钟）
   - 当前技术栈？
   - 最大的痛点/瓶颈？
   - 已经尝试过什么？

2. 快速诊断（15 分钟）
   - 识别 3 个可优化的关键点
   - 每个点给出具体改进建议
   - 量化预期收益（时间/成本/收入）

3. 行动方案（10 分钟）
   - 1 个立即可做的动作（免费赠送）
   - 1 个短期优化方案（引出付费服务）
   - 跟进时间确认
```

**产出物**：每次诊断后发送一份「诊断报告 PDF」，包含：
- 现状分析
- 3 个优化建议
- 1 个免费行动项
- 付费服务介绍（非推销，仅列出可选方案）

### 第 3 周：付费转化（首批 5 单）

**定价策略**（低门槛启动）：

| 服务类型 | 定价 | 交付方式 | 目标客户 |
|----------|------|----------|----------|
| 技术选型咨询 | ¥299/次 | 1 小时视频 | 有想法但不确定技术栈的创业者 |
| 代码审查服务 | ¥499/次 | 文字 + 1 小时讨论 | 已有项目想优化的开发者 |
| AI 工具链搭建 | ¥1,999/套 | 远程安装 + 文档 | 想用 AI 提效但不知从何入手的团队 |
| 月度技术顾问 | ¥3,999/月 | 4 次视频 + 不限次消息 | 持续需要技术指导的小团队 |

**转化话术要点**：
- 不推销，让诊断报告的价值说话
- 「上次诊断发现的 XX 问题，如果需要我帮你系统解决...」
- 首批客户 8 折优惠 + 1 次免费跟进

### 第 4 周：案例积累 + 口碑裂变

**目标**：把首批 5 个客户变成你的销售团队

| 动作 | 具体操作 |
|------|----------|
| 案例撰写 | 每个客户写 1 篇「技术诊断案例」（脱敏处理） |
| 评价收集 | 请客户写 2-3 句评价，授权使用 |
| 转介绍激励 | 推荐新客户，双方各得 1 次免费咨询 |
| 社群分享 | 在技术社群分享诊断案例（非广告） |

## 三、流量转化漏斗设计

```
文章曝光（19 篇 × 多平台）
    ↓ 5%-10% 点击
个人网站访问
    ↓ 10%-20% 浏览服务页
技术咨询页
    ↓ 20%-30% 提交咨询
免费诊断预约
    ↓ 40%-60% 完成诊断
免费诊断完成
    ↓ 30%-50% 转化
付费客户 ✅
```

**保守估算**：
- 19 篇文章总曝光 5,000-10,000 次（首月）
- 网站访问 500-1,000 人
- 服务页浏览 50-200 人
- 免费诊断预约 10-30 人
- 完成诊断 5-15 人
- **付费转化 2-5 人**
- **首月收入 ¥600-¥10,000**

## 四、个人网站服务页设计要点

在 aigc-portfolio 添加 `/services` 页面：

### 页面结构
1. **服务概述**（一句话说明价值）
2. **服务项目**（3-4 个清晰的服务包）
3. **工作流程**（4 步：咨询 → 诊断 → 方案 → 执行）
4. **案例展示**（首批客户案例）
5. **客户评价**（社会证明）
6. **CTA**（免费预约按钮 → 微信/表单）

### 设计原则
- 简洁 > 华丽
- 具体 > 抽象（写明「30 分钟诊断 + PDF 报告」而非「专业服务」）
- 社会证明 > 自我吹嘘
- 行动召唤 > 被动等待

## 五、避坑清单

| 坑 | 解决方案 |
|----|----------|
| 等客户主动来找 | 主动出击：技术社群回答问题，建立专业形象 |
| 免费咨询被白嫖 | 设定明确边界：30 分钟 + 1 次免费，超出即付费 |
| 定价太高无人问 | 冷启动期低价是策略，不是贬低 |
| 定价太低不可持续 | 标明原价 + 首批折扣，为未来涨价铺路 |
| 一个客户耗太多时间 | 用标准化诊断框架控制每次 30 分钟 |
| 不敢收钱 | 记住：你在解决别人自己解决不了的问题，这有价格 |
| 案例不足说服力弱 | 用「假设性案例」或自己的项目作为素材 |

## 八、与当前进度的衔接

```
第二阶段（当前 95%）         第三阶段（目标）
├── 19 篇文章 ✅          ├── 首批 5 客户
├── 个人网站 ✅           ├── 月入 5K-10K
├── GitHub 品牌 ✅     └── 案例积累 3+
└── 内容待发布 ⏳ ← 用户操作
```

**本指南的核心逻辑**：第二阶段的产出（文章、网站、品牌）是第三阶段的流量入口。冷启动的关键是**把内容影响力转化为客户信任**，而免费诊断是最高效的转化工具。
