# 用户反馈收集模板

> 收集真实用户的声音，让产品变得更好。

---

## 为什么需要这个模板？

**产品不是做给自己看的，是做给用户用的。**

- 用户反馈是产品迭代的燃料
- 真实使用场景会暴露设计盲点
- 好的反馈收集机制能持续改进产品

---

## 反馈收集渠道

### 1. 主动反馈（用户主动提交）

| 渠道 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 产品内反馈按钮 | 低门槛，场景化 | 需要开发 | 日常使用问题 |
| 微信/邮件 | 直接沟通 | 效率低 | 深度反馈 |
| 问卷表单 | 结构化数据 | 用户体验差 | 大规模调研 |

### 2. 被动反馈（观察用户行为）

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 用户访谈 | 深度洞察 | 成本高 | 种子用户 |
| 行为数据分析 | 客观真实 | 隐私问题 | 功能优化 |
| 社交媒体监听 | 真实口碑 | 样本偏差 | 口碑管理 |

## 反馈分类

### A. Bug 反馈

**定义**：产品功能异常或错误

**收集内容**：
- 问题描述：发生什么了？
- 复现步骤：如何触发这个问题？
- 预期行为：应该发生什么？
- 实际行为：实际发生了什么？
- 环境信息：操作系统、版本、截图

**示例**：
```
问题描述：复利计算器在输入负数时崩溃
复现步骤：
1. 打开复利计算器
2. 在"年化收益率"输入 -5
3. 点击"计算"按钮
预期行为：显示错误提示"收益率不能为负"
实际行为：程序崩溃
环境信息：macOS 12.3，版本 1.0.0
```

### B. 功能建议

**定义**：用户希望增加的新功能

**收集内容**：
- 需求描述：想要什么功能？
- 使用场景：在什么情况下会用到？
- 期望价值：这个功能能解决什么问题？
- 优先级：有多想要这个功能？（1-5分）

**示例**：
```
需求描述：希望增加"账单提醒"功能
使用场景：每月固定支出（房租、保险）自动提醒
期望价值：避免忘记还款，减少逾期风险
优先级：4 分（很想要）
```

### C. 体验反馈

**定义**：对产品使用感受的评价

**收集内容**：
- 满意度：整体使用感受如何？（1-5分）
- 最喜欢的功能：哪个功能最有用？
- 最不喜欢的点：哪里最让人困扰？
- 改进建议：如何能让体验更好？

**示例**：
```
满意度：4 分
最喜欢的功能：财务自由进度条，很直观
最不喜欢的点：记账分类太少，希望自定义
改进建议：增加"导入银行流水"功能，减少手动录入
```

### D. 付费意愿

**定义**：用户愿意为产品付费的程度

**收集内容**：
- 付费意愿：是否愿意付费？
- 价格接受度：能接受的价格区间？
- 付费模式：偏好一次性买断还是订阅？
- 功能期望：付费后希望获得什么？

**示例**：
```
付费意愿：是
价格接受度：¥49-99
付费模式：一次性买断
功能期望：数据云同步 + 多设备使用
```

## 反馈处理流程

### 1. 收集阶段

**频率**：持续收集
**渠道**：产品内 + 微信 + 邮件
**工具**：Notion / 飞书多维表格 / Excel

### 2. 分类阶段

**频率**：每周一次
**操作**：
- 将反馈按类型分类（Bug / 建议 / 体验 / 付费）
- 按优先级排序（高 / 中 / 低）
- 标记状态（待处理 / 处理中 / 已完成 / 暂不处理）

### 3. 分析阶段

**频率**：每月一次
**操作**：
- 统计各类反馈数量和趋势
- 识别高频问题和需求
- 提炼产品优化方向

### 4. 响应阶段

**频率**：收到反馈后 24 小时内
**操作**：
- 感谢用户反馈
- 告知处理进度
- 收到反馈后跟进解决情况

## 反馈激励机制

### 为什么需要激励？

- 用户没有义务提供反馈
- 好的反馈需要时间和思考
- 激励能提高反馈质量和数量

### 激励方式

| 方式 | 成本 | 效果 | 适用场景 |
|------|------|------|---------|
| 感谢信 | 低 | 一般 | 所有反馈 |
| 延长免费期 | 低 | 好 | 早期用户 |
| 功能优先权 | 低 | 好 | 功能建议 |
| 现金/礼品卡 | 高 | 很好 | 深度访谈 |
| 产品内致谢 | 低 | 一般 | 重大贡献 |

## 反馈收集最佳实践

### 1. 降低反馈门槛

- ❌ 填写 20 个问题的问卷
- ✅ 产品内一键反馈按钮

### 2. 及时响应

- ❌ 收到反馈后石沉大海
- ✅ 24 小时内回复，告知处理进度

### 3. 闭环反馈

- ❌ 修复了问题但不告知用户
- ✅ 解决后通知用户，感谢贡献

### 4. 尊重用户

- ❌ 忽略负面反馈
- ✅ 认真对待每一条反馈，无论正负

### 5. 持续优化

- ❌ 收集了反馈但不分析
- ✅ 定期复盘，提炼产品优化方向

## 反馈收集工具推荐

### 轻量级工具
- **腾讯问卷**：免费，易用，适合简单调研
- **金数据**：表单收集，适合反馈收集
- **Notion / 飞书**：团队协作，适合内部整理

### 专业工具
- **UserVoice**：功能强大，适合 SaaS 产品
- **Canny**：用户反馈管理，适合产品迭代
- **Intercom**：用户沟通 + 反馈收集

### 自己开发
- 产品内嵌反馈按钮
- 反馈数据存本地或云端
- 定期导出分析
