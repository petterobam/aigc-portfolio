# 📋 任务管理目录（tasks/）

> 本目录集中管理项目的所有任务信息，包括当前活跃任务和历史归档任务。  
> HEARTBEAT.md 通过引用本目录下的文件来获取任务状态，避免心跳文件膨胀。

---

## 目录结构

```text
tasks/
├── README.md          # ← 你正在阅读的文件
├── current.md         # 当前活跃任务状态（唯一的"活"文件）
└── .archive/          # 已完成/过期的任务记录归档
    └── *.md           # 按日期或心跳编号命名的历史任务快照
```

---

## 文件说明

### `current.md`（当前任务状态）

这是任务系统的**唯一入口**，包含：

| 板块 | 内容 |
|------|------|
| 当前最高优先级任务 | P0/P1/P2 优先级分类的活跃任务列表 |
| 最新完成项 | 最近一次心跳完成的具体事项 |
| 系统状态仪表盘 | 关键指标的实时数据表 |
| 等待用户执行的事项 | 需要用户手动操作的任务（附操作指引） |
| 最新进展摘要 | 近期核心突破和指标变化 |
| Cron 执行记录 | 最近 5 次 cron 任务的执行摘要 |
| 里程碑记录 | 关键里程碑时间线 |
| 下一步行动 | 短期/中期/长期规划 |

### `.archive/`（历史任务归档）

当 `current.md` 中的任务完成或过期时，将相关内容快照归档到此目录。

**命名规范**：
- 按日期归档：`YYYY-MM-DD-任务快照.md`
- 按心跳归档：`heartbeat-NNN-任务快照.md`
- 按里程碑归档：`milestone-简述.md`

## 使用规范

### AI 读取
1. 每次心跳 → 读取 `tasks/current.md` 获取当前任务上下文
2. 无需读取 `.archive/` 中的历史文件（除非需要回溯）

### AI 写入
1. **更新任务状态** → 直接编辑 `tasks/current.md`
2. **任务完成归档** → 将完成的任务块剪切到 `.archive/` 中的新文件
3. **新增任务** → 添加到 `current.md` 对应优先级区域
4. **Cron 报告** → 更新 `current.md` 中的 Cron 执行记录表格（保留最近 5 条），旧记录自动滚动到 `.archive/`

### 与 HEARTBEAT.md 的关系
- `HEARTBEAT.md` 是 AI 的**身份和信念**文件（不变的东西）
- `tasks/current.md` 是 AI 的**任务和状态**文件（频繁变化的东西）
- HEARTBEAT.md 中只保留一行引用：`> 📋 当前任务详见：[tasks/current.md](./tasks/current.md)`

---

## 归档触发条件

以下情况应将内容从 `current.md` 归档到 `.archive/`：

1. **任务全部完成** → 归档整个任务批次
2. **里程碑达成** → 归档里程碑详情，`current.md` 中只保留一行摘要
3. **Cron 记录超过 5 条** → 最旧的记录移入归档
4. **月度切换** → 上月任务快照归档，开启新月任务
