## 📋 使用步骤

### 1. 准备工作
- 确保已安装Git
- 确保在personal-website目录中
- 确保代码已准备就绪

### 2. 执行脚本
运行脚本后，按照提示选择GitHub仓库创建方式：
- **选项1**：使用GitHub CLI（最简单，需要已安装gh）
- **选项2**：手动创建网页版（适合没有GitHub CLI的用户）
- **选项3**：使用Personal Access Token（适合有token的用户）

### 3. 配置GitHub Pages
脚本会指导你完成GitHub Pages配置：
- 访问GitHub仓库设置
- 选择"GitHub Actions"作为源
- 保存配置

### 4. 验证部署
脚本会自动验证网站是否部署成功：
- 检查HTTP状态码
- 提供访问链接
- 确认网站正常运行

## 📝 注意事项

### 必备条件
- GitHub账户已创建
- 已安装Git
- 网络连接正常

### 推荐配置
- 优先使用GitHub CLI（最简单）
- 确保Personal Access Token有足够权限（repo权限）

### 常见问题
1. **认证失败**：检查GitHub CLI状态或Personal Access Token
2. **网站不显示**：等待1-5分钟让GitHub Actions完成构建
3. **配置错误**：检查config.toml文件格式
