## 技术实现方案

### 方案选择

**推荐方案**：使用 Hugo + GitHub Pages

**理由**：
- 静态网站生成器，速度快
- 无服务器成本（GitHub Pages 免费托管）
- 支持自定义主题
- 易于部署和维护
- SEO 友好

**备选方案**：
- Next.js + Vercel（更灵活，但需要 React 开发）
- WordPress（功能强大，但需要服务器成本）

### 实现步骤

**步骤 1：安装 Hugo**
```bash
brew install hugo
```

**步骤 2：创建站点**
```bash
hugo new-site personal-website
cd personal-website
```

**步骤 3：添加主题**
```bash
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
echo "theme = 'ananke'" >> config.toml
```

**步骤 4：创建页面**
```bash
hugo new about.md
hugo new services.md
hugo new case-studies.md
hugo new contact.md
```

**步骤 5：编写内容**
- 编辑 `content/about.md`
- 编辑 `content/services.md`
- 编辑 `content/case-studies.md`
- 编辑 `content/contact.md`

**步骤 6：预览网站**
```bash
hugo server -D
# 访问：http://localhost:1313
```

**步骤 7：部署到 GitHub Pages**
```bash
# 1. 创建 GitHub 仓库
# 2. 推送代码
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/your-website.git
git push -u origin main

# 3. 配置 GitHub Pages
# Settings → Pages → Source: gh-pages branch
```

**步骤 8：配置域名（可选）**
- 在域名提供商购买域名
- 在 GitHub 仓库配置 CNAME 文件
- 配置 DNS 解析

### GitHub Actions 自动部署

**创建 `.github/workflows/deploy.yml`**：

```yaml
name: Deploy Hugo site to Pages

on:
  push:
    branches: ["main"]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: 'latest'

      - name: Build
        run: hugo --minify

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```
