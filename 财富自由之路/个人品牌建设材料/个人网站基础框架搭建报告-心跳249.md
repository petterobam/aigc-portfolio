## 二、执行过程

### 2.1 项目初始化

#### 1. 清理旧项目

**问题**：发现已有的 `personal-website` 目录是基于 Hugo 的，与上次心跳（#248）选择的技术方案（VuePress 2）不符。

**解决方案**：删除 Hugo 项目，按照技术方案创建 VuePress 2 项目。

```bash
cd ~/.openclaw/workspace/财富自由之路
rm -rf personal-website
```

#### 2. 创建项目目录

```bash
mkdir -p personal-website
cd personal-website
npm init -y
```

#### 3. 安装 VuePress 2

```bash
npm install -D vuepress@next @vuepress/bundler-vite@next
```

**安装结果**：
- 添加了 173 个依赖包
- VuePress 版本：^2.0.0-rc.28
- Vite bundler 版本：^2.0.0-rc.28

#### 4. 安装默认主题

```bash
npm install -D @vuepress/theme-default@next --legacy-peer-deps
```

**问题**：安装时出现 peer dependency 冲突。

**错误信息**：
```
npm error ERESOLVE unable to resolve dependency tree
npm error Could not resolve dependency:
npm error peer vuepress@"2.0.0-rc.9" from @vuepress/theme-default@2.0.0-rc.28
```

**解决方案**：使用 `--legacy-peer-deps` 选项忽略 peer dependency 检查。

**安装结果**：
- 添加了 103 个依赖包
- 主题版本：^2.0.0-rc.127
- 警告：whatwg-encoding@3.1.1 已废弃（不影响功能）

## 四、遇到的问题与解决方案

### 4.1 Hugo 项目冲突

**问题描述**：发现已有的 Hugo 项目与上次技术方案（VuePress 2）不符。

**影响**：如果继续使用 Hugo 项目，会偏离上次心跳制定的技术方案。

**解决方案**：删除 Hugo 项目，按照技术方案创建 VuePress 2 项目。

**教训**：在开始新项目前，先确认是否有已有的项目需要清理或整合。

---

### 4.2 主题依赖冲突

**问题描述**：安装 @vuepress/theme-default 时出现 peer dependency 冲突。

**错误信息**：
```
npm error ERESOLVE unable to resolve dependency tree
npm error Could not resolve dependency:
npm error peer vuepress@"2.0.0-rc.9" from @vuepress/theme-default@2.0.0-rc.28
```

**影响**：无法安装默认主题，项目无法正常启动。

**解决方案**：使用 `--legacy-peer-deps` 选项忽略 peer dependency 检查。

**命令**：
```bash
npm install -D @vuepress/theme-default@next --legacy-peer-deps
```

**结果**：成功安装，主题正常工作。

**教训**：VuePress 2 还在 RC 阶段，依赖关系可能不稳定，必要时可以使用 `--legacy-peer-deps`。

---

### 4.3 脚本找不到问题

**问题描述**：在错误目录下运行 npm run docs:dev，导致脚本找不到。

**错误信息**：
```
npm error Missing script: "docs:dev"
```

**影响**：无法启动开发服务器。

**解决方案**：切换到正确的目录（personal-website）再运行。

**命令**：
```bash
cd personal-website
npm run docs:dev
```

**结果**：成功启动开发服务器。

**教训**：运行 npm 脚本前，先确认当前目录是否正确。
