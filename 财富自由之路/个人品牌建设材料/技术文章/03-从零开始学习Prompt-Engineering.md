## Prompt Engineering 核心概念

### 什么是提示词（Prompt）？

**定义**：
提示词（Prompt）是人类与 AI 模型交互的输入，包含意图、上下文、约束等信息。

**构成要素**：
```
提示词 = 意图 + 上下文 + 约束 + 示例
```

**示例**：
```
意图：帮我写一个 Python 函数
上下文：这个函数用于计算斐波那契数列
约束：使用递归实现，添加注释
示例：fib(5) = 5
```

**完整提示词**：
```
请帮我写一个 Python 函数，用于计算斐波那契数列。

要求：
1. 使用递归实现
2. 添加详细的注释
3. 函数名为 fib

示例：
- fib(0) = 0
- fib(1) = 1
- fib(5) = 5

请直接返回代码，不要解释。
```

### 提示词工程（Prompt Engineering）

**定义**：
提示词工程是设计和优化提示词，以从大模型中获得高质量输出的艺术和科学。

**核心目标**：
- 提高输出准确性
- 减少幻觉（Hallucination）
- 控制输出格式
- 提升效率

**关键技能**：
- 理解大模型的工作原理
- 掌握提示词设计模式
- 能够调试和优化提示词
- 理解上下文和约束的重要性

## 提示词设计模式

### 模式 1：角色扮演（Role-Playing）

**定义**：
让 AI 扮演特定角色，以获得更专业的回答。

**示例**：
```
你现在是一位资深的 Python 后端工程师，有 10 年的架构设计经验。

请帮我设计一个高并发的订单系统。

要求：
1. 能够处理 10 万 QPS
2. 保证数据一致性
3. 支持水平扩展

请提供：
1. 系统架构图（描述即可）
2. 技术栈选型
3. 关键模块设计
4. 可能的瓶颈和解决方案
```

### 模式 2：思维链（Chain-of-Thought）

**定义**：
引导 AI 展示推理过程，提高复杂任务的准确性。

**示例**：
```
请回答以下问题，并一步步展示推理过程。

问题：如果一个农场有鸡和兔子，共有 10 个头和 28 条腿，那么鸡和兔子各有多少只？

请按照以下格式回答：
步骤 1：设鸡的数量为 x，兔子的数量为 y
步骤 2：根据头的数量列出方程
步骤 3：根据腿的数量列出方程
步骤 4：解方程组
步骤 5：验证答案

请完整展示推理过程。
```

### 模式 3：自纠（Self-Consistency）

**定义**：
让 AI 多次思考，检查答案的一致性。

**示例**：
```
请计算以下数学题：

123 × 456 = ?

请：
1. 先计算一次，得到答案
2. 再用不同的方法验证一次
3. 如果两次结果一致，输出答案
4. 如果不一致，分析原因并重新计算

请展示完整的推理过程。
```

### 模式 4：提示词模板（Prompt Template）

**定义**：
创建可复用的提示词模板，提高效率。

**示例**：
```python

## 实战案例

### 案例 1：代码生成

**任务**：生成一个 Python 函数，计算两个日期之间的工作日天数（排除周末和节假日）。

**提示词**：
```
请用 Python 写一个函数，计算两个日期之间的工作日天数。

要求：
1. 函数名为 calculate_workdays
2. 参数：start_date, end_date（datetime.date 类型）
3. 排除周末（周六、周日）
4. 支持自定义节假日列表
5. 添加类型提示和注释

示例：
- calculate_workdays(2026-03-30, 2026-04-05) = 5
  （3/30, 3/31, 4/1, 4/2, 4/4，排除 4/3 周五？不对，让我重新算）
  - 3/30 周一：工作日
  - 3/31 周二：工作日
  - 4/1 周三：工作日
  - 4/2 周四：工作日
  - 4/3 周五：工作日
  - 4/4 周六：周末
  - 4/5 周日：周末
  - 总共 5 个工作日

请直接返回代码，不要解释。
```

**输出**：
```python
from datetime import date, timedelta
from typing import List

def calculate_workdays(start_date: date, end_date: date, holidays: List[date] = None) -> int:
    """
    计算两个日期之间的工作日天数（排除周末和节假日）

    Args:
        start_date: 开始日期
        end_date: 结束日期
        holidays: 节假日列表（可选）

    Returns:
        工作日天数
    """
    if holidays is None:
        holidays = []

    workdays = 0
    current_date = start_date

    while current_date <= end_date:
        # 检查是否为周末
        if current_date.weekday() < 5:  # 0-4 为周一到周五
            # 检查是否为节假日
            if current_date not in holidays:
                workdays += 1
        current_date += timedelta(days=1)

    return workdays

# 示例使用
if __name__ == "__main__":
    start = date(2026, 3, 30)
    end = date(2026, 4, 5)
    holidays = [date(2026, 4, 4), date(2026, 4, 5)]  # 清明节
    print(calculate_workdays(start, end, holidays))  # 输出: 4
```

```python
from typing import List, Dict, Union

def process_data(data: List[Dict[str, Union[str, int]]]) -> List[int]:
    """
    处理数据，根据类型计算值

    Args:
        data: 数据列表，每个元素包含 'type' 和 'value' 键

    Returns:
        处理后的结果列表

    Raises:
        ValueError: 如果数据格式不正确
    """
    # 类型映射：A -> 2, B -> 3, 其他 -> 1
    type_multiplier = {
        'A': 2,
        'B': 3,
    }

    result = []
    for item in data:
        # 获取类型和值，如果不存在则使用默认值
        item_type = item.get('type', '')
        value = item.get('value', 0)

        # 验证数据格式
        if not isinstance(item_type, str) or not isinstance(value, (int, float)):
            raise ValueError(f"Invalid data format: {item}")

        # 根据类型计算值
        multiplier = type_multiplier.get(item_type, 1)
        result.append(int(value * multiplier))

    return result

# 列表推导式版本（更简洁）
def process_data_v2(data: List[Dict[str, Union[str, int]]]) -> List[int]:
    """列表推导式版本"""
    type_multiplier = {'A': 2, 'B': 3}

    return [
        int(item.get('value', 0) * type_multiplier.get(item.get('type', ''), 1))
        for item in data
    ]
```

## 常见错误与调试技巧

### 错误 1：提示词不清晰

**问题**：
```
帮我优化这段代码
```

**结果**：
- AI 不知道优化什么
- 输出质量差

**修复**：
```
请帮我优化下面这段 Python 代码。

背景：这是一个用于处理用户上传图片的后端 API
目标：将处理时间从 5 秒降低到 1 秒以内

要求：
1. 使用多进程加速
2. 保持功能不变
3. 添加错误处理

代码：[插入代码]
```

### 错误 2：上下文不足

**问题**：
```
计算斐波那契数列
```

**结果**：
- AI 不知道计算哪一项
- 不知道用什么方法

**修复**：
```
请计算斐波那契数列的第 10 项。

要求：
1. 使用递归实现
2. 展示计算过程

请一步步思考。
```

### 错误 3：没有输出格式约束

**问题**：
```
分析这段代码
```

**结果**：
- AI 可能输出长篇大论
- 没有结构化信息

**修复**：
```
请分析下面这段代码。

请按照以下格式返回：
1. 功能说明
2. 时间复杂度
3. 空间复杂度
4. 优化建议

代码：[插入代码]
```

### 错误 4：没有错误处理

**问题**：
```
计算：123 × 456
```

**结果**：
- AI 可能计算错误
- 没有验证机制

**修复**：
```
请计算：123 × 456

请：
1. 先计算一次
2. 再验证一次
3. 如果结果一致，输出答案
4. 如果不一致，重新计算

请展示完整的推理过程。
```

## 进阶技巧

### 技巧 1：使用变量和模板

```python
# 提示词模板
CODE_GENERATION_TEMPLATE = """
请用 {language} 写一个 {function_type} 函数。

功能：{function_description}

要求：
{requirements}

请直接返回代码，不要解释。
"""

# 使用模板
prompt = CODE_GENERATION_TEMPLATE.format(
    language="Python",
    function_type="高阶函数",
    function_description="用于过滤列表中的偶数",
    requirements=[
        "函数名为 filter_even",
        "使用列表推导式",
        "添加类型提示",
        "添加注释",
    ]
)
```

### 技巧 2：多轮对话优化

```python
# 第一轮
response1 = ask_llm("写一个 Python 函数计算斐波那契数列")

# 第二轮（优化）
response2 = ask_llm(f"""
请优化下面这个函数：

{response1}

要求：
1. 添加缓存机制
2. 添加类型提示
3. 添加错误处理
""")

# 第三轮（测试）
response3 = ask_llm(f"""
为下面这个函数编写单元测试：

{response2}

测试用例：
- fib(0) = 0
- fib(1) = 1
- fib(5) = 5
- fib(10) = 55
""")
```

import requests

prompt = f"""
根据以下天气信息，给出穿衣建议：

地点：北京
天气：{weather['description']}
温度：{weather['temperature']}°C

请给出具体的建议（上衣、裤子、鞋子等）。
