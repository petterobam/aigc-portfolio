## 二、技术方案选择

### 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| VuePress | 文档型网站，易于部署，支持 Markdown | 主题定制限制较多 | 技术博客、文档站 |
| Astro | 性能优异，支持多种 UI 框架 | 生态相对较新 | 快速构建、多页面站 |
| Hugo | 构建速度快，Go 语言 | 模板语法较复杂 | 大型内容站 |
| Next.js | React 生态，功能强大 | 学习曲线较陡 | 复杂交互站 |
| WordPress | 插件丰富，上手简单 | 性能较差，安全性问题 | 快速上线 |

### 最终选择：VuePress 2

**选择理由**：

1. ✅ **技术栈匹配**：已经熟悉 Vue 3 + TypeScript
2. ✅ **文档型网站**：适合展示技术文章、方法论、项目案例
3. ✅ **易于部署**：支持 GitHub Pages 静态托管
4. ✅ **快速上手**：配置简单，内容维护方便
5. ✅ **SEO 友好**：静态生成，易于搜索引擎收录
6. ✅ **主题扩展**：有丰富的第三方主题可选
7. ✅ **成本为零**：开源免费，无服务器成本

## 六、部署方案

### GitHub Pages 部署

1. **创建 GitHub 仓库**
   - 仓库名：`personal-website`
   - 初始化为公开仓库

2. **配置 GitHub Actions**
   ```yaml
   # .github/workflows/deploy.yml
   name: Deploy to GitHub Pages

   on:
     push:
       branches: [main]

   jobs:
     build-and-deploy:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v3

         - name: Setup Node.js
           uses: actions/setup-node@v3
           with:
             node-version: '18'

         - name: Install pnpm
           uses: pnpm/action-setup@v2
           with:
             version: 8

         - name: Install dependencies
           run: pnpm install

         - name: Build VuePress site
           run: pnpm docs:build

         - name: Deploy to GitHub Pages
           uses: peaceiris/actions-gh-pages@v3
           with:
             github_token: ${{ secrets.GITHUB_TOKEN }}
             publish_dir: docs/.vuepress/dist
   ```

3. **启用 GitHub Pages**
   - 设置 → Pages → Source 选择 GitHub Actions
   - 部署完成后访问：`https://oyjie.github.io/personal-website/`

### 自定义域名（可选）

- 购买域名（如：`aigc-consultant.com`）
- 在域名 DNS 设置中添加 CNAME 记录
- 在 GitHub Pages 设置中添加自定义域名
