## 二、API 接口

### 2.1 统计数据

```
GET /api/kg/stats
```

返回实体和关系的汇总统计，无需参数。

### 2.2 实体搜索

```
GET /api/kg/entities?q={关键词}&limit={数量}
```

**参数**：
- `q`（必填）：搜索关键词，支持中文、梵文、英文
- `limit`（可选）：返回数量限制

**返回字段**：
- `id`：实体唯一ID
- `entity_type`：实体类型（person/text/school/monastery/place/dynasty/concept）
- `name_zh`：中文名
- `name_sa`：梵文名（Sanskrit）
- `name_pi`：巴利文名（Pāli）
- `name_bo`：藏文名（Tibetan）
- `name_en`：英文名
- `description`：简要描述
- `properties`：结构化属性（如朝代、坐标、角色等）
- `external_ids`：外部数据库ID（如 DILA、Wikidata、CBETA）

### 2.3 实体详情（含关系）

```
GET /api/kg/entities/{id}
```

返回单个实体的完整信息，**包含所有关系**（`relations` 数组）。

**关系对象字段**：
- `predicate`：关系类型
- `direction`：`outgoing`（主动关系）/ `incoming`（被动关系）
- `target_id`：目标实体ID
- `target_name`：目标实体名称
- `target_type`：目标实体类型
- `confidence`：置信度（0-1）
- `source`：数据来源标注（如 `auto:cbeta_metadata`、`seed:lineage`、`dila_catalog`）

## 三、实测案例分析：鸠摩罗什

以 ID=15（鸠摩罗什 Kumārajīva）为例，探索知识图谱的深度。

### 基本信息

| 字段 | 值 |
|------|-----|
| 中文名 | 鳩摩羅什 |
| 梵文名 | Kumārajīva |
| 时代 | 十六國 |
| 角色 | 译者（translator） |
| 活动坐标 | 34.33°N, 108.91°E（长安） |
| 外部ID | DILA: A001583 |

### 关系网络

**师徒关系（teacher_of）**：
- 🔴 **弟子**：僧肇（ID:48）、僧叡、道生、道融、道恒 — 即"什门四圣"加道恒
- 🔵 **老师**：須利耶蘇摩（incoming）

**活动地点（active_in）**：
- 十六國（朝代）、长安、草堂寺、石澗寺、龟兹（置信度0.9）

**翻译作品（translated）**（部分，数据被截断但已非常丰富）：
- 中论、十二門論、大智度論 — **三论宗核心论典**
- 妙法蓮華經（法华经）— **天台宗根本经典**
- 佛說阿彌陀經 — **净土宗核心经典**
- 小品般若波羅蜜經 — 般若经系
- 坐禪三昧經 — 禅观经典
- 十誦律 — 戒律经典

### 数据来源分析

关系数据标注了来源类型，体现了数据的可靠性层次：
- `auto:cbeta_metadata`：从 CBETA 元数据自动提取（最权威）
- `seed:lineage`：人工种子数据，法脉传承
- `seed:person_place`：人工种子数据，人物地点
- `dila`：来自 DILA（法鼓文理学院）人名权威数据库
- `dila_catalog`：来自 DILA 目录

## 四、知识图谱的研究价值

### 4.1 法脉传承可视化

22,687 条师徒关系构成了庞大的佛教传承网络。可用于：
- 追溯某位僧人的师承渊源
- 绘制宗派的法脉传承图
- 发现跨宗派的师承联系

### 4.2 译经史研究

2,993 条翻译关系 + 102 条译经地关系，可用于：
- 分析某位译师的翻译全貌（如鸠摩罗什翻译了哪些经典）
- 追溯某部经典的翻译史（谁译、何时、何地）
- 比较不同译师的翻译风格和选经偏好

### 4.3 典籍互引网络

1,477 条引用关系（cites）+ 63 条注释关系（commentary_on），可用于：
- 构建典籍影响图谱
- 发现核心枢纽经典（被引用最多的典籍）
- 追踪思想传播路径

### 4.4 地理与寺院研究

59,836 个寺院实体 + 338 个地点实体，是研究佛教地理的宝贵资源。

## 五、API 使用建议

1. **搜索优先**：先用 `/api/kg/entities?q=...` 搜索获取实体ID
2. **详情深入**：再用 `/api/kg/entities/{id}` 获取完整关系网络
3. **注意截断**：详情接口返回的关系可能较多，需注意响应体可能被截断
4. **数据质量**：关注 `confidence` 和 `source` 字段评估数据可靠性
5. **多语言支持**：搜索支持中/梵/英多种语言
