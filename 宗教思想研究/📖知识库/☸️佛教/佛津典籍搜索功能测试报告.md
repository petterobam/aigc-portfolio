> **测试日期**：2026-04-15
> **测试范围**：佛津 API（fojin.app/api/search）典籍检索功能

## 一、API 基本信息

- **接口地址**：`https://fojin.app/api/search?q={关键词}&limit={数量}`
- **返回格式**：JSON
- **分页参数**：`page`, `size`（默认 size=20）
- **数据源**：以 CBETA（中华电子佛典）为主，兼有 kanseki-repo 等日藏数据源

## 三、API 功能评估

### ✅ 优势

1. **结构化数据完整**：每条结果包含大正藏编号、CBETA链接、译者、朝代、分类
2. **关联译本自动标注**：`related_translations` 字段自动关联同经异译，极大便利版本比较
3. **评分排序合理**：核心经文排在最前，注疏论释随后
4. **多数据源融合**：不仅有CBETA，还收录 kanseki-repo（日本汉籍）等
5. **CBETA直链**：每条结果提供 `cbeta_url`，可直接跳转阅读原文

### ⚠️ 限制

1. **全文内容不返回**：`has_content` 字段均为 false，API 仅提供元数据
2. **平行阅读需前端**：多版本对照功能依赖 JS 渲染，API 层面暂不支持逐段对照
3. **辞典搜索 API 路径未明**：/api/dictionary 返回 404，辞典查询可能使用不同接口
4. **无内容分段**：无法通过 API 获取经文具体章节/品的信息

### 📋 可复用的检索路径

| 研究需求 | API 调用方式 | 说明 |
|---------|-------------|------|
| 查找某经所有译本 | `search?q={经名}` + 查看 related_translations | 关联字段自动聚合异译 |
| 查找某经注疏 | `search?q={经名}` + 筛选标题含"疏/赞/注/论" | 注疏通常排在经文之后 |
| 查找某译者作品 | `search?q={译者名}` | 可按译者检索 |
| 获取CBETA阅读链接 | 结果中的 `cbeta_url` 字段 | 直接跳转在线阅读 |

## 四、下一步建议

1. **平行阅读功能**需通过浏览器访问 fojin.app 前端体验，API层面暂不支撑
2. **辞典功能**需进一步探索正确的 API 端点或通过前端使用
3. **知识图谱**（/kg）功能需通过前端访问
4. 基于 API 的元数据检索已经非常有价值，可支持版本比较研究的初步筛选
