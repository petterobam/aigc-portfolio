# 📋 任务管理中心

> **定位**：研究任务的计划、追踪与归档中心，确保每项研究工作有始有终、可追溯。

---

## 🏗️ 三层任务机制

本工作区采用三层任务管理机制，从宏观到微观，确保研究工作有序推进：

```
┌─────────────────────────────────────────────────────┐
│  第一层：HEARTBEAT.md                                │
│  ↳ 引用「任务总览.md」，获取当前活跃任务全貌          │
├─────────────────────────────────────────────────────┤
│  第二层：任务总览.md（任务清单的清单）                 │
│  ↳ 索引所有任务清单文件，记录完成状态和优先级          │
├─────────────────────────────────────────────────────┤
│  第三层：TASK-XXX-标题.md（具体任务清单）              │
│  ↳ 每个文件是一批相关任务的集合，含具体待办和进度      │
├─────────────────────────────────────────────────────┤
│  归档层：归档/                                       │
│  ↳ 已完成的任务清单移入归档，保留历史记录              │
└─────────────────────────────────────────────────────┘
```

### 层级说明

| 层级 | 文件 | 职责 | 更新频率 |
|------|------|------|---------|
| **入口层** | `HEARTBEAT.md` | 引用任务总览，每次心跳检查待办 | 每次心跳 |
| **索引层** | `任务总览.md` | 所有任务清单的目录，含状态和优先级 | 新增/完成任务清单时 |
| **执行层** | `TASK-XXX-标题.md` | 具体任务条目，含检查框和进度记录 | 执行任务时 |
| **归档层** | `归档/` | 已完成任务清单的存档 | 任务清单全部完成时 |

---

## 📝 任务清单命名规范

- 格式：`TASK-{三位编号}-{简洁可理解的标题}.md`
- 示例：
  - `TASK-001-佛津平台资源调研.md`
  - `TASK-002-般若经系研读计划.md`
  - `TASK-003-原始佛教经典入门.md`
- 编号递增，不跳号，不重用

---

## 📋 任务清单模板

每个 TASK 文件应遵循以下结构：

```markdown
# TASK-XXX: 标题

> **状态**：🟡 进行中 / ✅ 已完成 / 📌 待开始 / ⏸️ 暂停
> **优先级**：🔴 高 / 🟡 中 / 🟢 低
> **创建日期**：YYYY-MM-DD
> **预计完成**：YYYY-MM-DD
> **关联目录**：相关知识库/专题目录路径

## 🎯 目标

简述这批任务要达成的目标。

- [ ] 任务 1 描述
- [ ] 任务 2 描述
- [ ] 任务 3 描述
- [x] 已完成的任务（示例）

|------|---------|---------|------|
| | | | |

（执行过程中的发现、思考、调整记录）
```

## 🔄 任务生命周期

```
📌 待开始 → 🟡 进行中 → ✅ 已完成 → 📦 归档
                ↕
             ⏸️ 暂停
```

1. **创建**：新建 TASK 文件，在「任务总览.md」中注册
2. **执行**：逐条完成任务，更新检查框和执行记录
3. **完成**：所有任务条目完成，状态改为 ✅
4. **归档**：将完成的 TASK 文件移入 `归档/` 目录，更新任务总览

---

## ⚠️ 使用规范

1. **每次心跳先看任务总览**：确认当前活跃任务和优先级
2. **一个 TASK 文件聚焦一个主题**：不要把不相关的任务塞进同一个文件
3. **分批合理**：每个 TASK 文件包含 5-15 个具体任务条目，太少太碎，太多难以追踪
4. **及时更新状态**：完成一项勾一项，不要积攒
5. **归档不删除**：完成的任务清单移入归档而非删除，保留完整研究轨迹
