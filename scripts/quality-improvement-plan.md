# 写作质量优化方案 - 避免重复段落

> 创建时间：2026-03-26 14:30
> 优先级：P0（最高优先级任务3）

---

## 🎯 目标

优化写作技巧和自查技巧，避免重复段落的出现，提升番茄短篇小说的质量。

---

## 🔍 现状分析

### 现有检测脚本

| 脚本 | 功能 | 问题 |
|------|------|------|
| check-duplicates-strict.py | 检测重复段落（30字以上） | 事后检测 |
| check-duplicate-paragraphs.py | 检测重复段落（50字以上） | 事后检测 |

### 问题总结

1. **事后检测**：检测时内容已经生成，需要修改
2. **检测方法简单**：只检查完全相同的段落，不检测相似段落
3. **缺少创作指导**：没有写作技巧指导和预防措施
4. **没有实时反馈**：创作过程中没有实时检测和提醒

## 💡 优化方案

### 方案1：写作技巧优化（预防性）

#### 1.1 表达方式多样化

**目标**：使用多样的表达方式，避免重复。

**具体方法**：
- **同义词替换**：使用同义词、近义词替换重复词汇
  - 例如："愤怒" → "暴怒"、"怒火中烧"、"怒不可遏"
  - 例如："害怕" → "恐惧"、"惊慌"、"战战兢兢"

- **句式结构变化**：
  - 主动句 ↔ 被动句
  - 长句 ↔ 短句
  - 简单句 ↔ 复合句
  - 陈述句 ↔ 疑问句 ↔ 感叹句

- **修辞手法**：
  - 比喻：明喻、暗喻、借喻
  - 夸张：扩大夸张、缩小夸张
  - 排比：增强气势
  - 对比：突出反差

#### 1.2 内容结构优化

**目标**：避免内容重复，保持段落之间的独立性。

**具体方法**：
- **每段聚焦一个主题**：一个段落只讲一件事
- **避免回溯**：不要反复讲述同一个情节
- **信息推进**：每段都要提供新信息，不要重复
- **逻辑递进**：段落之间要有逻辑关系，不是简单的罗列

#### 1.3 情绪表达技巧

**目标**：用不同的方式表达相同或相似的情绪。

**具体方法**：
- **正面描写**：直接描述情绪状态
  - 例如："他感到非常愤怒。"

- **侧面描写**：通过动作、神态、语言表现情绪
  - 例如："他的脸涨得通红，拳头紧握。"

- **环境烘托**：通过环境描写表现情绪
  - 例如："窗外雷声轰鸣，仿佛在诉说着他的愤怒。"

- **心理活动**：通过内心独白表现情绪
  - 例如："愤怒像火焰一样在他心中燃烧。"

### 方案2：自查技巧优化（预防性 + 事后）

#### 2.1 创作时的实时自查

**目标**：在创作过程中实时检测重复，及时修改。

**具体方法**：

**阶段1：段落级别自查**
- 每写完一段，问自己：
  - [ ] 这段话在前面出现过吗？
  - [ ] 这段话和前面哪段话最相似？
  - [ ] 这段话提供了新信息吗？
  - [ ] 这段话可以和其他段落合并吗？

**阶段2：章节级别自查**
- 每写完一章，问自己：
  - [ ] 这一章有重复段落吗？
  - [ ] 这一章和前面哪章最相似？
  - [ ] 这一章有推进情节吗？
  - [ ] 这一章可以精简吗？

**阶段3：全文级别自查**
- 每写完全文，问自己：
  - [ ] 全文有重复段落吗？
  - [ ] 全文的段落之间有逻辑递进吗？
  - [ ] 全文的信息推进清晰吗？
  - [ ] 全文的情绪推进合理吗？

#### 2.2 自动化检测脚本

**目标**：使用脚本自动检测重复，提高检测效率。

**具体方法**：

**脚本1：实时相似度检测**
- 功能：检测相似段落（相似度 > 80%）
- 算法：使用编辑距离（Levenshtein Distance）或余弦相似度
- 阈值：相似度 > 80% 时警告
- 输出：列出相似段落及其位置

**脚本2：同义词库生成**
- 功能：生成同义词库，辅助替换
- 数据源：网络词典、同义词库
- 输出：JSON 格式的同义词库

**脚本3：自动替换建议**
- 功能：自动生成替换建议
- 算法：基于同义词库和上下文
- 输出：替换建议列表

#### 2.3 自查清单

**创作前检查**：
- [ ] 明确这一段的写作目的
- [ ] 确认这一段的情绪基调
- [ ] 想好表达方式（正面/侧面/环境/心理）

**创作中检查**：
- [ ] 检查是否有重复词汇
- [ ] 检查是否有重复句式
- [ ] 检查是否有重复内容
- [ ] 检查是否有回溯情节

**创作后检查**：
- [ ] 运行重复检测脚本
- [ ] 运行相似度检测脚本
- [ ] 检查段落之间的逻辑递进
- [ ] 检查信息的推进是否清晰
