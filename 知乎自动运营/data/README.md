# 💾 数据存储

> 知乎运营项目的数据文件存储目录

---

## 🎯 用途

- 存储自动化系统采集的原始数据
- 存储处理后的结构化数据
- 存储数据分析和报告文件
- 提供数据的持久化存储

## 📋 数据类型

### 1. 采集数据

由自动化脚本采集的原始数据：

**answer-data/**: 知乎回答数据
- 回答内容
- 赞同数、收藏数、评论数
- 发布时间
- 作者信息

**article-data/**: 知乎专栏数据
- 专栏文章内容
- 阅读数、点赞数、收藏数
- 发布时间
- 关联信息

**fan-data/**: 粉丝数据
- 粉丝列表
- 粉丝画像数据
- 粉丝增长趋势

**hot-topics/**: 热点数据
- 知乎热榜
- Hacker News
- GitHub Trending
- arXiv 论文

### 2. 分析数据

经过处理和分析后的数据：

**reports/**: 分析报告
- 数据分析报告
- 趋势分析报告
- 效果评估报告

### 3. 缓存数据

临时缓存的数据文件：

**cache/**: 缓存文件
- API 响应缓存
- 页面内容缓存
- 临时处理结果

## 📁 数据文件命名规范

### 按日期组织

```
answer-data/
├── 2026-03-28/
│   ├── answer-001.json
│   ├── answer-002.json
│   └── ...
├── 2026-03-29/
│   └── ...
```

### 文件命名格式

- 原始数据：`{类型}-{序号}.{ext}`
  - 示例：`answer-001.json`, `article-005.json`
- 报告文件：`{类型}-{日期}.{ext}`
  - 示例：`daily-report-2026-03-28.md`, `weekly-report-2026-03-28.md`
- 缓存文件：`{类型}-{时间戳}.{ext}`
  - 示例：`page-cache-1774650313008.html`

## 📊 数据文件格式

### JSON 格式

用于结构化数据存储：

```json
{
  "id": "answer_id",
  "title": "回答标题",
  "content": "回答内容",
  "author": {
    "name": "作者名",
    "id": "author_id"
  },
  "stats": {
    "upvotes": 100,
    "comments": 20,
    "collections": 50
  },
  "created_at": "2026-03-28T12:00:00Z",
  "updated_at": "2026-03-28T12:00:00Z",
  "url": "https://www.zhihu.com/question/xxx/answer/xxx"
}
```

### CSV 格式

用于表格数据存储：

```csv
id,title,upvotes,comments,collections,created_at
001,回答标题,100,20,50,2026-03-28T12:00:00Z
002,回答标题2,200,30,80,2026-03-28T13:00:00Z
```

### Markdown 格式

用于报告和文档：

```markdown
# 数据分析报告

## 概述

{报告概述}

## 数据详情

{数据详情}

## 结论

{结论和发现}
```

## 🔍 数据查找

### 按日期查找

```bash
ls -lt data/{类型}/{日期}/
```

### 按关键词搜索

```bash
grep -r "关键词" data/{类型}/
```

### 按文件类型查找

```bash
find data/ -name "*.json"
find data/ -name "*.csv"
```
