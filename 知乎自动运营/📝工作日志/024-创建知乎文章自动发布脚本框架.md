## 💡 关键学习

### 1. Playwright 自动化的最佳实践

**Cookie 管理**：
- Cookie 是自动化的关键，必须定期维护和刷新
- 使用 `context.addCookies()` 加载 Cookie
- 检查关键 Cookie（知乎：d_c0、z_c0）

**人类化行为**：
- 随机延迟（避免被识别为机器人）
- 逐字输入（模拟真实打字）
- 操作后短暂停顿（人类行为特征）

**错误处理**：
- 元素查找失败时，尝试多个选择器
- 操作超时时，添加重试机制
- 保存错误截图（便于调试）

### 2. 内容发布流程设计

**发布流程**：

**数据格式**：

## 📝 备注

### 技术要点

**Playwright API 使用**：
- `chromium.launch()` - 启动浏览器
- `browser.newContext()` - 创建上下文
- `context.newPage()` - 创建页面
- `context.addCookies()` - 加载 Cookie
- `page.goto()` - 导航到页面
- `page.$()` - 查找元素
- `element.click()` - 点击元素
- `page.keyboard.type()` - 模拟键盘输入
- `page.evaluate()` - 在页面上下文中执行代码
- `page.screenshot()` - 截图

**选择器策略**：
- 使用多个备选选择器（提高稳定性）
- 优先使用稳定的 CSS 选择器（class、data-testid）
- 避免使用动态生成的选择器（id、随机字符串）

**Cookie 文件路径**：
- 知乎 Cookie：`auth/zhihu-cookies-latest.json`
- 番茄 Cookie：`cookies/fanqie-latest.json`
- 其他 Cookie：`cookies/<platform>-latest.json`

### 发布脚本使用注意事项

**使用前**：
1. 确保 Cookie 已配置
2. 准备好文章文件（JSON 格式）
