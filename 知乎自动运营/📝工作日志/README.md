## 📊 日志命名规范

### 按序号命名

- 格式：`{序号}-{简短描述}.md`
- 序号：001, 002, 003, ...
- 简短描述：10-15 字，概括本次工作核心内容

### 按日期命名

- 格式：`YYYY-MM-DD.md`
- 用途：汇总当日所有心跳工作记录
- 内容：当日各次心跳的总结和关键发现

---

## 📈 日志使用场景

### 日常使用

1. **查看进展**: 翻看最新日志，了解最近做了什么
2. **查找经验**: 搜索关键词，快速定位相关经验教训
3. **复盘总结**: 每周/每月汇总日志，做深度复盘

### 问题排查

1. **定位问题**: 通过日志时间线，追溯问题发生的时间点
2. **了解背景**: 查看当时的工作背景，理解决策原因
3. **参考方案**: 查看类似问题的解决过程，借鉴经验
