### 2. 批量发布脚本完善

#### 问题诊断
- `publish-zhihu-article.js` 已有完整的实现
- `batch-publish-articles.js` 的核心发布逻辑（`publishColumnArticle` 和 `publishAnswer`）还没有完全实现，只有 TODO 注释

#### 解决方案
创建 `batch-publish-articles-v2.js`，将 `publish-zhihu-article.js` 的完整实现移植过去

#### 核心功能
1. **Cookie 管理**：加载知乎 Cookie，检查关键 Cookie（d_c0、z_c0）
2. **登录状态检查**：访问创作中心验证登录状态
3. **人类化输入**：模拟真实用户行为，逐字输入，随机延迟
4. **文章发布**：
   - 导航到发布页面
   - 输入标题
   - 输入正文（使用 evaluate 直接设置 innerHTML，模拟粘贴）
   - 添加话题标签（最多 5 个）
   - 点击发布按钮
5. **发布验证**：检查是否跳转到文章详情页，获取文章 URL

### 3. 文件匹配问题修复

#### 问题诊断
- 有 11 篇文章找不到对应的 Markdown 文件
- 原因：标准化元数据文件名和实际 Markdown 文件名不匹配
- 示例：
  - `MoE架构-GPT4秘密武器-standardized.json` 找不到对应的 Markdown 文件
  - 实际有 `MoE架构-GPT4的秘密武器-稀疏激活与路由设计.md`

#### 解决方案
创建 `fix-md-file-mapping.js` 脚本，通过以下方式解决：
1. **明确映射**：为已知的文件不匹配创建明确映射
2. **自动查找**：对于没有明确映射的文件，自动在多个目录中查找
3. **文件复制**：将找到的文件复制到高优先级目录

### 4. 内容类型问题修复

#### 问题诊断
- 有些文章的 `type` 字段不是"专栏文章"或"回答"
- 示例：
  - `type: 实战教程`
  - `type: OpenClaw 技巧系列 - 储备选题`
  - `type: article`
  - `type: 深度教程`
  - `type: undefined`

#### 解决方案
修改 `publishArticle` 函数，让它能够处理所有内容类型：
1. **默认处理**：所有非回答类型都发布为专栏文章
2. **回答类型**：回答类型暂不支持自动发布（需要搜索问题）

### 5. 字段安全检查

#### 问题诊断
- 有些元数据文件缺少 `publishPlan.relatedColumn` 字段
- 导致脚本报错：`Cannot read properties of undefined (reading 'relatedColumn')`

#### 解决方案
添加字段安全检查：
```javascript
const relatedColumn = metadata.publishPlan?.relatedColumn || '未指定';
const likes = metadata.estimatedMetrics?.likes || '未知';
const favorites = metadata.estimatedMetrics?.favorites || '未知';
```
