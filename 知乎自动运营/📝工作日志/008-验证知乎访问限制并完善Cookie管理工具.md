### 1. 创建知乎专用 Cookie 提取脚本（Playwright 版本）

**文件**: `🛠️自动化系统/scripts/extract-zhihu-cookies-playwright.js`

**功能**:
- 通过 Playwright 的 `connectOverCDP` 连接到已运行的 Chrome

**特点**:
- 参考番茄 Cookie 提取脚本的设计思路
- 针对知乎的域名和 session cookie 进行适配

- 详细的调试输出和 Cookie 过期检查

**使用方法**:
```bash
# 1. 以调试模式启动 Chrome
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222

# 2. 在 Chrome 中登录知乎

# 3. 运行提取脚本
node scripts/extract-zhihu-cookies-playwright.js
```

## 🔍 学习与发现

### 1. 知乎访问限制已验证

**发现**:
- 知乎现在对未登录用户有严格的访问限制
- 未登录访问热榜页面会被重定向到登录页面
- 这意味着**数据采集系统必须先解决登录问题**

**解决方案**:
1. **使用有效的知乎 Cookie**
   - 通过 `extract-zhihu-cookies-playwright.js` 提取 Cookie
   - 通过 `login-zhihu-save-cookies.js` 手动登录获取 Cookie
   - Cookie 有效期约 30 天

2. **定期刷新 Cookie**
   - 实现 Cookie 失效检测
   - 失效时自动重新提取或登录
   - 建议每月刷新一次

3. **持久化 Session**
   - 使用 Playwright 的持久化上下文
   - 保存 Session 到文件
   - 下次使用时加载 Session

---

### 2. Cookie 提取两种方式

**方式1: CDP 连接（推荐）**
- 通过 `connectOverCDP` 连接到已运行的 Chrome
- 优点：无需手动登录，直接提取
- 缺点：需要 Chrome 以调试模式启动
- 适用：已有登录状态的浏览器

**方式2: 手动登录**
- 启动新的浏览器实例
- 用户手动登录
- 优点：不需要调试模式
- 缺点：需要用户交互
- 适用：CDP 端口不可用

---

### 3. 知乎页面结构分析

**发现**:
- 知乎使用 React 框架
- 页面数据通过 JavaScript 动态加载
- 选择器可能随页面版本变化而失效
- 需要等待 `networkidle` 或 `domcontentloaded` 事件

**最佳实践**:
- 使用多个备选选择器
- 添加适当的等待时间
- 保存页面 HTML 用于调试
- 定期检查选择器是否有效
