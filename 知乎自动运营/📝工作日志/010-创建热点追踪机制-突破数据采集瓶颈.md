## 🔍 学习与发现

### 1. 数据采集的多源策略

**发现**:
- 单一数据源风险高（如知乎需要Cookie）
- 多数据源可以提高数据获取的稳定性
- 公开数据源可以作为知乎数据的补充

**应用**:
- Hacker News（技术社区热点）
- GitHub Trending（开发者关注点）
- arXiv（最新研究成果）
- 知乎热榜（国内热点，需要Cookie）

### 2. Cookie管理的必要性

**发现**:
- 知乎对未登录用户有严格限制
- Cookie 提取需要图形界面操作
- Cookie 有效期有限（需要定期刷新）

**解决方案**:
1. 手动提取一次 Cookie
2. 保存到 `auth/zhihu-cookies.json`
3. 脚本自动加载 Cookie
4. 定期手动刷新 Cookie（约30天）

### 3. 网络超时的处理

**发现**:
- Playwright 默认超时是 30 秒
- 网络不稳定时容易超时
- 需要增加重试机制和更长的超时时间

**优化方向**:
- 增加重试机制（失败后重试2-3次）
- 增加超时时间配置（根据数据源调整）
- 添加代理支持（可选）
