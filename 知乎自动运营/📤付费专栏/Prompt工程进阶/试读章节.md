## 前言

Chain-of-Thought（思维链，简称 CoT）是 2022 年 Google 提出的 Prompt 技术，通过引导模型"逐步思考"，显著提升了复杂任务的准确性。

从数学推理到逻辑推导，从代码生成到文本分析，CoT 已经成为 Prompt 工程中最核心的技术之一。

本篇文章将系统讲解 CoT 的核心原理、实现方法和优化技巧，帮助你掌握这一革命性技术。

## 1. CoT 的核心原理

### 1.1 为什么 CoT 有效？

CoT 的核心思想是：**引导模型逐步思考，分解复杂任务，提升推理能力**。

**传统 Prompt 的问题**：
```
Prompt：一个水果店有 100 个苹果，第一天卖出 30 个，第二天卖出剩下的 40%，还剩多少个？

模型回答：还剩 42 个。

问题：模型可能直接给出答案，没有展示推理过程，
如果计算错误，难以发现和纠正。
```

**CoT Prompt 的改进**：
```
Prompt：让我们一步步思考。

问题：一个水果店有 100 个苹果，第一天卖出 30 个，第二天卖出剩下的 40%，还剩多少个？

模型回答：
1. 第一天卖出 30 个，还剩 100 - 30 = 70 个
2. 第二天卖出 70 的 40%，即 70 × 0.4 = 28 个
3. 第二天还剩 70 - 28 = 42 个

答案：还剩 42 个。

优势：模型展示了完整的推理过程，可以验证每一步的正确性。
```

### 1.2 CoT 的数学原理

CoT 的有效性可以从以下角度解释：

**1️⃣ 注意力机制（Attention Mechanism）**
- CoT 通过引导词（如"让我们一步步思考"）激活模型的注意力机制
- 模型会更加关注推理路径，而不是直接跳到答案

**2️⃣ 推理路径（Reasoning Path）**
- CoT 将复杂任务分解为多个子任务
- 每个子任务的推理过程更加清晰，减少了错误累积

**3️⃣ 中间验证（Intermediate Verification）**
- CoT 的每一步都可以独立验证
- 如果某一步出错，可以及时发现并纠正

### 1.3 CoT 的适用场景

CoT 适用于以下场景：

**✅ 适用场景**：
- **数学推理**：需要逐步计算的任务
- **逻辑推理**：需要逐步推导的任务
- **代码生成**：需要逐步设计的任务
- **文本分析**：需要逐步分析的任务

**❌ 不适用场景**：
- **简单任务**：可以直接回答的问题（如"巴黎是哪个国家的首都？"）
- **创造性任务**：不需要逻辑推理的任务（如"写一首诗"）
- **主观任务**：需要主观判断的任务（如"这个设计好看吗？"）

## 2. CoT 的实现方法

### 2.1 Zero-shot CoT

**核心思想**：不提供示例，直接使用引导词引导模型思考。

**引导词示例**：
- "让我们一步步思考。"
- "请详细解释你的推理过程。"
- "请分解这个问题，逐步解决。"

**Prompt 模板**：
```python
def zero_shot_cot_prompt(problem):
    return f"""让我们一步步思考。

问题：{problem}

推理过程：1. """

# 使用示例
print(zero_shot_cot_prompt("一个水果店有 100 个苹果，第一天卖出 30 个，第二天卖出剩下的 40%，还剩多少个？"))
```

**优点**：
- 简单易用，不需要准备示例
- 适用于各种任务
- 不占用大量 Token

**缺点**：
- 效果依赖于模型能力
- 不如 Few-shot CoT 稳定

### 2.2 Few-shot CoT

**核心思想**：提供示例，展示完整的思考过程。

**Prompt 模板**：
```python
def few_shot_cot_prompt(problem):
    return f"""# 任务：解决数学推理问题

# 示例 1
问题：一本书有 200 页，第一天读了 20%，第二天读了 50 页，还剩多少页？
推理过程：
1. 第一天读了 200 × 20% = 40 页
2. 第一天还剩 200 - 40 = 160 页
3. 第二天读了 50 页
4. 第二天还剩 160 - 50 = 110 页
答案：还剩 110 页

# 示例 2
问题：一个商店有 500 元，上午卖了 30%，下午卖了 200 元，还剩多少元？
推理过程：
1. 上午卖了 500 × 30% = 150 元
2. 上午还剩 500 - 150 = 350 元
3. 下午卖了 200 元
4. 下午还剩 350 - 200 = 150 元
答案：还剩 150 元

# 当前任务
问题：{problem}
推理过程：1. """

# 使用示例
print(few_shot_cot_prompt("一个水果店有 100 个苹果，第一天卖出 30 个，第二天卖出剩下的 40%，还剩多少个？"))
```

**优点**：
- 效果稳定，准确率高
- 适合复杂任务
- 可以控制推理风格

**缺点**：
- 需要准备示例
- 占用较多 Token

### 2.3 Auto CoT

**核心思想**：自动生成示例，减少人工准备示例的成本。

**实现方法**：
```python
from sentence_transformers import SentenceTransformer
from sklearn.cluster import KMeans

class AutoCoTGenerator:
    def __init__(self, embedding_model="all-MiniLM-L6-v2"):
        self.embedding_model = SentenceTransformer(embedding_model)

    def generate_examples(self, questions, num_examples=3):
        """自动生成 CoT 示例"""

        # 1. 向量化所有问题
        embeddings = self.embedding_model.encode(questions)

        # 2. 聚类：选择代表性问题
        kmeans = KMeans(n_clusters=num_examples)
        cluster_labels = kmeans.fit_predict(embeddings)

        # 3. 从每个聚类中选择最接近中心的问题
        selected_questions = []
        for cluster_id in range(num_examples):
            cluster_indices = np.where(cluster_labels == cluster_id)[0]
            center = kmeans.cluster_centers_[cluster_id]

            # 找到最接近中心的示例
            best_index = cluster_indices[
                np.argmin([
                    self._cosine_similarity(center, embeddings[i])
                    for i in cluster_indices
                ])
            ]
            selected_questions.append(questions[best_index])

        # 4. 为每个问题生成 CoT（使用 LLM）
        examples = []
        for question in selected_questions:
            cot = self._generate_cot_with_llm(question)
            examples.append({
                "question": question,
                "cot": cot
            })

        return examples

    def _generate_cot_with_llm(self, question):
        """使用 LLM 生成 CoT"""
        # 这里调用 LLM 生成 CoT
        prompt = f"让我们一步步思考。\n\n问题：{question}\n\n推理过程：1. "
        # 调用 LLM API
        # response = llm.generate(prompt)
        # return response
        return "（模拟生成的 CoT）"

    def _cosine_similarity(self, vec1, vec2):
        return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))

# 使用示例
generator = AutoCoTGenerator()

questions = [
    "一个水果店有 100 个苹果，第一天卖出 30 个，第二天卖出剩下的 40%，还剩多少个？",
    "一本书有 200 页，第一天读了 20%，第二天读了 50 页，还剩多少页？",
    "一个商店有 500 元，上午卖了 30%，下午卖了 200 元，还剩多少元？",
    "一个图书馆有 1000 本书，第一天借出 15%，第二天借出 200 本，还剩多少本？"
]

examples = generator.generate_examples(questions, num_examples=3)
print("自动生成的 CoT 示例：", examples)
```

**优点**：
- 减少人工准备示例的成本
- 可以自动适应不同任务
- 示例质量较高

**缺点**：
- 需要额外的计算资源
- 依赖于 LLM 的生成质量

## 3. CoT 的应用场景

### 3.1 数学推理

**任务类型**：逐步计算，避免跳步错误

**示例**：
```
问题：一个三角形的三边分别是 3、4、5，计算它的面积。

推理过程：
1. 判断这是一个直角三角形（3² + 4² = 5²）
2. 直角三角形的面积公式：面积 = (直角边1 × 直角边2) / 2
3. 面积 = (3 × 4) / 2 = 6
4. 答案：面积是 6

对比（无 CoT）：
问题：一个三角形的三边分别是 3、4、5，计算它的面积。
答案：面积是 6。

问题：直接给出答案，没有展示推理过程，
如果计算错误，难以发现和纠正。
```

### 3.2 逻辑推理

**任务类型**：逐步推导，确保逻辑连贯

**示例**：
```
问题：如果今天是星期三，那么 100 天后是星期几？

推理过程：
1. 100 天 = 14 周零 2 天
2. 14 周后还是星期三
3. 再过 2 天，就是星期三 + 2 天 = 星期五
4. 答案：100 天后是星期五

对比（无 CoT）：
问题：如果今天是星期三，那么 100 天后是星期几？
答案：星期五。

问题：直接给出答案，没有展示推理过程，
如果推理错误，难以发现和纠正。
```

### 3.3 代码生成

**任务类型**：逐步设计，确保代码正确

**示例**：
```
问题：用 Python 实现一个计算斐波那契数列的函数。

推理过程：
1. 斐波那契数列的定义：F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2)
2. 可以使用递归或迭代实现
3. 递归实现：基例是 F(0) = 0 和 F(1) = 1，递归关系是 F(n) = F(n-1) + F(n-2)
4. 递归代码：
```python
def fibonacci(n):
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n-1) + fibonacci(n-2)
```
5. 迭代实现：更高效，避免重复计算
```python
def fibonacci_iterative(n):
    if n <= 0:
        return 0
    elif n == 1:
        return 1

    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b

    return b
```

对比（无 CoT）：
问题：用 Python 实现一个计算斐波那契数列的函数。
代码：
```python
def fibonacci(n):
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n-1) + fibonacci(n-2)
```

问题：直接给出代码，没有展示设计思路，
如果代码有 bug，难以发现和纠正。
```

## 4. CoT 的优化技巧

### 4.1 引导词优化

**最佳引导词**：
- ✅ "让我们一步步思考。"（最常用）
- ✅ "请详细解释你的推理过程。"
- ✅ "请分解这个问题，逐步解决。"
- ✅ "请先分析问题，再给出答案。"

**避免的引导词**：
- ❌ "快速思考。"（可能导致推理不充分）
- ❌ "直接给出答案。"（违背 CoT 的初衷）

**引导词位置**：
- ✅ 放在 Prompt 的开头（效果最好）
- ⚠️ 放在 Prompt 的中间（效果一般）
- ❌ 放在 Prompt 的结尾（效果最差）

**示例**：
```python
# 最佳实践
prompt = """让我们一步步思考。

问题：{question}

推理过程：1. """

# 避免使用
prompt = """问题：{question}

请快速思考。推理过程：1. """
```

### 4.2 中间步骤优化

**优化原则**：
1. **明确每一步的目标**：每一步都应该清晰表达"做什么"和"为什么做"
2. **使用编号或列表**：使用"1. 2. 3."或"- - -"标记步骤
3. **每一步要具体**：避免模糊的描述，如"计算出结果"，应该写"计算出结果为 X"

**示例对比**：

**❌ 优化前**：
```
推理过程：
1. 计算出结果
2. 得出答案
```

**✅ 优化后**：
```
推理过程：
1. 计算出第一天卖出后剩余的数量：100 - 30 = 70 个
2. 计算出第二天卖出的数量：70 × 40% = 28 个
3. 计算出第二天剩余的数量：70 - 28 = 42 个
4. 答案：还剩 42 个
```

### 4.3 验证机制

**自我验证**：在推理过程中加入验证步骤，确保每一步的正确性。

**示例**：
```
推理过程：
1. 计算出第一天卖出后剩余的数量：100 - 30 = 70 个
   验证：100 - 30 = 70 ✅

2. 计算出第二天卖出的数量：70 × 40% = 28 个
   验证：70 × 0.4 = 28 ✅

3. 计算出第二天剩余的数量：70 - 28 = 42 个
   验证：70 - 28 = 42 ✅

4. 答案：还剩 42 个
   最终验证：第一天卖出 30 + 第二天卖出 28 = 58 个
                剩余 = 100 - 58 = 42 个 ✅
```

**人类验证**：在推理过程中，邀请人类审核关键步骤。

**示例**：
```
推理过程：
1. 计算出第一天卖出后剩余的数量：100 - 30 = 70 个
   🧑‍💻 人类审核：正确 ✅

2. 计算出第二天卖出的数量：70 × 40% = 28 个
   🧑‍💻 人类审核：正确 ✅

3. 计算出第二天剩余的数量：70 - 28 = 42 个
   🧑‍💻 人类审核：正确 ✅

4. 答案：还剩 42 个
```

2. 第二天卖出 70 的 40%，即 70 × 0.4 = 28 个

```

1. 判断这是一个直角三角形：5² + 12² = 25 + 144 = 169 = 13² ✅
2. 直角三角形的面积公式：面积 = (直角边1 × 直角边2) / 2
3. 直角边是 5 和 12
4. 面积 = (5 × 12) / 2 = 30

答案：面积是 30
```

**题目3（困难）**：
```
一个人从 A 地出发，先向东走 10 公里，再向北走 20 公里，再向西走 30 公里，最后向南走 40 公里，
问这个人最终距离 A 地多远？

推理过程：
1. 建立坐标系：A 地为原点 (0, 0)
2. 向东走 10 公里：坐标变为 (10, 0)
3. 向北走 20 公里：坐标变为 (10, 20)
4. 向西走 30 公里：坐标变为 (-20, 20)
5. 向南走 40 公里：坐标变为 (-20, -20)
6. 最终距离 A 地的距离 = √(x² + y²) = √((-20)² + (-20)²) = √(400 + 400) = √800 ≈ 28.28 公里

答案：最终距离 A 地约 28.28 公里
```

### 5.2 任务2：代码生成

**题目：用 Python 实现一个快速排序算法**

**推理过程**：
1. 快速排序的核心思想：
   - 选择一个基准元素（pivot）
   - 将小于 pivot 的元素放到左边
   - 将大于 pivot 的元素放到右边
   - 递归地对左右两边进行排序

2. 递归实现：
```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr

    pivot = arr[0]
    left = [x for x in arr[1:] if x <= pivot]
    right = [x for x in arr[1:] if x > pivot]

    return quick_sort(left) + [pivot] + quick_sort(right)
```

3. 迭代实现（更高效，避免栈溢出）：
```python
def quick_sort_iterative(arr):
    stack = [(0, len(arr) - 1)]

    while stack:
        low, high = stack.pop()

        if low < high:
            pivot_index = partition(arr, low, high)
            stack.append((low, pivot_index - 1))
            stack.append((pivot_index + 1, high))

    return arr

def partition(arr, low, high):
    pivot = arr[high]
    i = low - 1

    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]

    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1
```
