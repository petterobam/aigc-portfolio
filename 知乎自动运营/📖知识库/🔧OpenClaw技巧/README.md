# OpenClow 常见错误和解决方案

## 错误类型分类

### 1. 配置错误
#### 错误1: JSON 格式错误
**现象**：Agent 无法启动
**原因**：JSON 格式不正确（少逗号、引号等）
**排查步骤**：
1. 用 JSON Validator 检查格式
2. 检查语法错误
**解决方案**：修正 JSON 格式
**预防措施**：使用编辑器自动格式化

#### 错误2: 路径错误
**现象**：找不到文件或目录
**原因**：路径配置错误
**排查步骤**：
1. 检查路径是否正确
2. 检查文件是否存在
**解决方案**：修正路径
**预防措施**：使用绝对路径

### 2. 运行错误
#### 错误3: Agent 执行超时
**现象**：Agent 运行超时
**原因**：任务复杂度高、网络问题
**排查步骤**：
1. 检查任务复杂度
2. 检查网络连接
**解决方案**：
- 增加超时时间
- 简化任务
- 优化网络
**预防措施**：设置合理的超时时间

### 3. 性能问题
#### 错误4: 响应速度慢
**现象**：Agent 响应速度慢
**原因**：任务复杂度高、并发过多
**排查步骤**：
1. 检查任务复杂度
2. 检查并发数量
**解决方案**：
- 优化 Prompt
- 增加缓存
- 减少并发
**预防措施**：监控性能指标

## 快速排查流程
1. 检查日志（logs/ 目录）

**示例文章结构**：
```markdown
# OpenClow 批处理和自动化技巧

## 1. 批处理技巧

### 场景1: 批量处理多个文件
**问题**：需要处理 100 个 Markdown 文件
**方案**：使用 Shell 脚本 + OpenClow Agent

**实现**：
```bash
for file in *.md; do
  openclaw ask "优化这个文件的文章结构" $file
done
```

**优化**：并发处理（GNU parallel）

### 场景2: 批量生成 Prompt
**问题**：需要生成多个相似 Prompt
**方案**：使用模板 + 变量替换

**实现**：
```javascript
const prompts = files.map(file => ({
  role: "writer",
  task: "优化文章结构",
  input: file
}));
```

## 2. 并发控制

### 限制并发数量
**问题**：并发过多导致资源耗尽
**方案**：使用队列控制并发

**实现**：
```javascript
const concurrency = 5;
const queue = new PQueue({ concurrency });
```

**效果**：CPU 使用率从 100% 降到 50%

## 3. 缓存机制

### 场景1: 缓存 LLM 响应
**问题**：相同 Prompt 重复调用 LLM
**方案**：使用 Redis 缓存

**实现**：
```javascript
const cached = await redis.get(key);
if (cached) return cached;
const result = await callLLM(prompt);
await redis.set(key, result);
```

**效果**：响应时间从 2s 降到 0.1s

## 4. 性能优化

### 优化1: 减少不必要的 Prompt
**前**：每次都重新生成完整 Prompt
**后**：缓存常用 Prompt 模板

**效果**：API 调用次数减少 30%

### 优化2: 使用更快的模型
**前**：所有任务都用 GPT-4
**后**：简单任务用 GPT-3.5，复杂任务用 GPT-4

**效果**：成本降低 50%，速度提升 2 倍

## 5. 自动化技巧

### 定时任务
**场景**：每天早上 9 点发送日报
**方案**：使用 Cron 调度器

**配置**：
```json
{
  "schedule": "0 9 * * *",
  "task": "daily-report"
}
```

### 事件触发
**场景**：收到邮件后自动回复
**方案**：使用 Webhook

**实现**：
```javascript
app.post('/webhook/email', async (req, res) => {
  await openclaw.run('auto-reply', req.body);
  res.status(200).send();
});
```

- 定期优化（Review 代码、Prompt）
- 文档记录（记录优化经验）
