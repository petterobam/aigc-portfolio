# 📊 周复盘机制

> 每周对知乎技术内容运营进行全面复盘，总结经验教训，持续优化策略

---

## 🎯 周复盘目标

### 核心目标

1. **数据复盘**：总结本周发布的文章数据表现
2. **策略优化**：根据数据调整内容策略
3. **经验沉淀**：总结成功经验和失败教训
4. **下周规划**：制定下周的发布计划和优化方向

### 复盘周期

- **复盘时间**：每周日 18:00-20:00
- **复盘范围**：上周一至本周日
- **复盘形式**：生成周复盘报告文档

---

### 第一步：数据汇总

| 客单价 | 平均客单价 | 自动计算 |

### 第二步：数据分析

#### 文章数据分析

**高表现文章识别**
- 定义：赞同数 500+ 或收藏数 250+
- 分析：标题、内容结构、标签、发布时间、字数、代码片段数
- 总结：高表现文章的共同特征

**低表现文章分析**
- 定义：赞同数 < 100 或收藏数 < 50
- 分析：标题吸引力、内容深度、标签选择、发布时间
- 总结：低表现文章的问题和改进方向

#### 内容类型效果分析

| 内容类型 | 发布数 | 平均赞同 | 平均收藏 | 平均评论 | 评价 |

| 实战教程 | - | - | - | - | - |
| 原理深度 | - | - | - | - | - |
| 热点解读 | - | - | - | - | - |
| 复盘总结 | - | - | - | - | - |

#### 赛道效果分析

| 赛道 | 发布数 | 平均赞同 | 平均收藏 | 平均评论 | 评价 |
|------|--------|---------|---------|---------|------|
| OpenClaw 技巧 | - | - | - | - | - |
| AIGC 原理 | - | - | - | - | - |
