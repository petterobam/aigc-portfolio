# 📊 临时统计分析

> 临时统计分析目录，用于存放临时性的数据分析报告和统计结果

---

## 🎯 目录用途

### 主要用途

1. **临时分析**：存放临时性的数据分析报告
2. **实验性分析**：存放实验性的分析结果
3. **一次性分析**：存放一次性分析报告
4. **草稿分析**：存放分析过程中的草稿

### 与归档记录的区别

| 维度 | 临时统计分析 | 历史数据快照 | 数据导出 |
|------|-------------|-------------|---------|
| 用途 | 临时性分析 | 定期快照保存 | 导出数据 |
| 保存周期 | 短期 | 长期 | 长期 |
| 清理规则 | 定期清理 | 永久保存 | 永久保存 |
| 归档规则 | 不归档 | 自动归档 | 自动归档 |

---

## 📋 文件规范

### 文件命名规范

```
{分析主题}-{日期}-{状态}.{扩展名}

状态标识：
- draft      - 草稿
- inprogress - 进行中
- review     - 审核中
- final      - 最终版

示例：
- 爆款文章分析-20260331-draft.md
- 标签热度分析-20260331-final.md
```

### 文件清理规则

1. **定期清理**：每月 1 日清理上月草稿
2. **过期清理**：超过 3 个月的临时分析自动删除
3. **手动清理**：完成分析后手动删除草稿文件
4. **归档规则**：有价值的分析报告转移到 `归档记录/深度分析归档/`
