# scripts/eval

> 自动化脚本评估和测试目录，用于测试自动化脚本的功能和性能

---

## 🎯 目录用途

### 主要用途

1. **脚本测试**：测试自动化脚本的功能和性能
2. **单元测试**：测试脚本的单个功能模块
3. **集成测试**：测试脚本的完整功能流程
4. **性能测试**：测试脚本的性能指标

---

## 📋 测试类型

### 测试分类

| 测试类型 | 用途 | 命名规范 |
|---------|------|---------|
| 单元测试 | 测试单个功能模块 | `test-{module}.js` |
| 集成测试 | 测试完整功能流程 | `test-{workflow}.js` |
| 性能测试 | 测试性能指标 | `perf-{workflow}.js` |
| 压力测试 | 测试高负载性能 | `stress-{workflow}.js` |

---

## 📋 测试规范

### 测试脚本规范

1. **测试文件命名**
   - 使用 `test-` 或 `perf-` 或 `stress-` 前缀
   - 使用描述性的文件名

2. **测试用例编写**
   - 每个测试用例应该独立
   - 测试用例应该有清晰的描述
   - 测试用例应该有明确的预期结果

3. **测试结果记录**
   - 记录测试时间
   - 记录测试结果（通过/失败）
   - 记录测试数据

## 📋 测试清理规则

### 清理规则

1. **定期清理**：每月 1 日清理上月测试报告
2. **保留规则**：保留最近 3 个月的测试报告
3. **归档规则**：有价值的测试报告转移到 `🗂️归档记录/`
