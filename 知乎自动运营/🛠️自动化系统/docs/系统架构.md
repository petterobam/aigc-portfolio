## 系统概述

### 目标

实现知乎平台的完全自动化运营，包括：
- 内容发布（回答、专栏文章）
- 数据采集（赞同、收藏、评论、浏览量）
- 互动运营（自动回复、点赞、关注）
- 内容分析（热门话题、竞品分析、标题优化）

### 核心特性

- ✅ **自动化**: 全流程自动化，无需人工干预
- ✅ **稳定**: 基于真实浏览器，Cookie 自动管理
- ✅ **可扩展**: 模块化设计，易于添加新功能
- ✅ **可维护**: 清晰的代码结构和文档
- ✅ **安全**: Cookie 和会话信息安全管理

## 模块设计

### 1. Cookie 管理器（cookie-manager.js）

**职责**:
- 保存 Cookie 到 JSON 文件
- 加载 Cookie 到 Playwright 上下文
- 检测 Cookie 是否有效
- 刷新过期 Cookie

**API**:
```javascript
class CookieManager {
  // 保存 Cookie
  async saveCookies(cookies, path)
  
  // 加载 Cookie
  async loadCookies(path)
  
  // 检测 Cookie 是否有效
  async validateCookies(cookies, testUrl)
  
  // 刷新过期 Cookie
  async refreshCookies(context, testUrl)
}
```

---

### 2. Session 管理器（session-manager.js）

**职责**:
- 创建持久化浏览器上下文
- 保存上下文状态
- 恢复上下文状态
- 管理浏览器生命周期

**API**:
```javascript
class SessionManager {
  // 创建持久化上下文
  async createContext(path, userDataDir)
  
  // 保存上下文状态
  async saveState(context, path)
  
  // 恢复上下文状态
  async restoreState(browser, path)
  
  // 关闭浏览器
  async closeBrowser(browser)
}
```

## 数据流

### 发布流程

```
1. 用户浏览器提取 Cookie
   ↓
2. Cookie 管理器保存 Cookie
   ↓
3. Playwright 客户端加载 Cookie
   ↓
4. 导航到发布页面
   ↓
5. 填充内容（标题、正文、标签）
   ↓
6. 提交表单
   ↓
7. 验证发布成功
   ↓
8. 保存发布记录
```

---

### 数据采集流程

```
1. Playwright 客户端加载 Cookie
   ↓
2. 导航到数据页面
   ↓
3. 提取数据（赞同、收藏、评论、浏览量）
   ↓
4. 处理分页和懒加载
   ↓
5. 保存数据到 JSON 文件
   ↓
6. 生成数据报告
```

---

### 互动运营流程

```
1. Playwright 客户端加载 Cookie
   ↓
2. 导航到评论页面
   ↓
3. 提取评论列表
   ↓
4. 根据规则筛选评论
   ↓
5. 执行互动操作（回复、点赞、关注）
   ↓
6. 保存互动记录
```

---

### 分析流程

```
1. 加载历史数据
   ↓
2. 导航到分析页面（如热门话题）
   ↓
3. 提取分析数据
   ↓
4. 对比历史数据
   ↓
5. 生成分析报告
```

## 错误处理

### 1. Cookie 失效

**检测**:
- 访问需要登录的页面时，检测是否跳转到登录页
- Cookie 中的 `expires` 字段检查

**处理**:
- 提示用户重新登录
- 自动重新登录（如果可用）
- 刷新 Cookie

---

### 2. 网络错误

**检测**:
- `page.goto()` 超时
- 网络请求失败

**处理**:
- 重试机制（最多 3 次）
- 延迟重试（指数退避）
- 记录错误日志
- 通知用户

---

### 3. 元素找不到

**检测**:
- `page.waitForSelector()` 超时
- `page.click()` 失败

**处理**:
- 检查页面结构是否变化
- 更新选择器
- 记录错误日志
- 通知用户

---

### 4. 反爬机制

**检测**:
- 账号被封
- 访问受限
- 验证码

**处理**:
- 停止自动化操作
- 通知用户
- 等待人工干预

## 性能优化

### 1. 并发操作

**策略**:
- 多个数据采集任务并发执行
- 使用 `Promise.all()` 或 `Promise.allSettled()`

**示例**:
```javascript
// 并发采集多个回答的数据
const answerIds = ['123', '456', '789'];
const data = await Promise.all(
  answerIds.map(id => collectAnswerData({ answerId: id }))
);
```

---

### 2. 缓存机制

**策略**:
- 缓存 Cookie 和会话状态
- 缓存页面数据

**示例**:
```javascript
// 缓存 Cookie
const cachePath = 'auth/cookies-cache.json';
if (fs.existsSync(cachePath)) {
  const cookies = JSON.parse(fs.readFileSync(cachePath, 'utf8'));
  await context.addCookies(cookies);
}
```

---

### 3. 延迟和节流

**策略**:
- 添加随机延迟，避免高频操作
- 使用节流控制请求频率

**示例**:
```javascript
// 随机延迟
const randomDelay = Math.random() * 2000 + 1000; // 1-3 秒
await new Promise(resolve => setTimeout(resolve, randomDelay));
```

---

### 4. 资源清理

**策略**:
- 及时关闭浏览器和上下文
- 清理临时文件

**示例**:
```javascript
// 关闭浏览器
await browser.close();

// 清理临时文件
fs.unlinkSync('temp/data.json');
```
