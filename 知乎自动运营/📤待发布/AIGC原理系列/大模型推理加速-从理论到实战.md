## 二、推理加速的核心技术

### 技术 1：KV Cache

**问题**：自注意力机制的重复计算

在 Transformer 解码器中，生成每个 token 时：
1. 需要计算当前 token 与之前所有 token 的注意力
2. 之前 token 的 Key 和 Value 矩阵会被重复计算
3. 计算复杂度：O(n²)，n 是序列长度

**解决方案**：缓存 Key 和 Value 矩阵

第一次计算时，保存每个 token 的 Key 和 Value：
```python
# 第一次计算
class KVCache:
    def __init__(self, dtype=torch.float16, device='cuda'):
        self.dtype = dtype
        self.device = device
        self.keys = []
        self.values = []

    def update(self, key, value):
        """更新缓存：追加新的 key 和 value"""
        self.keys.append(key.to(self.dtype).to(self.device))
        self.values.append(value.to(self.dtype).to(self.device))
        return torch.cat(self.keys, dim=1), torch.cat(self.values, dim=1)

    def get(self):
        """获取缓存的所有 key 和 value"""
        return torch.cat(self.keys, dim=1), torch.cat(self.values, dim=1)
```

后续生成时，直接使用缓存的 Key 和 Value：
```python
def generate_with_kv_cache(model, prompt, max_tokens=100):
    """使用 KV Cache 的生成函数"""
    tokens = model.tokenizer.encode(prompt)
    kv_cache = KVCache()

    # 前向传播，建立初始缓存
    with torch.no_grad():
        outputs = model(
            input_ids=torch.tensor([tokens]).cuda(),
            use_cache=True
        )
        kv_cache.update(outputs.past_key_values[0][0], outputs.past_key_values[0][1])

    generated_tokens = tokens.copy()

    # 逐个生成 token
    for _ in range(max_tokens):
        with torch.no_grad():
            # 只计算当前 token 的 attention
            outputs = model(
                input_ids=torch.tensor([[generated_tokens[-1]]]).cuda(),
                past_key_values=outputs.past_key_values,
                use_cache=True
            )

            # 获取下一个 token
            next_token = torch.argmax(outputs.logits[:, -1, :], dim=-1).item()
            generated_tokens.append(next_token)

            # 更新 KV Cache
            kv_cache.update(outputs.past_key_values[0][0], outputs.past_key_values[0][1])

            if next_token == model.tokenizer.eos_token_id:
                break

    return model.tokenizer.decode(generated_tokens)
```

**效果**：
- 计算量从 O(n²) 降低到 O(n)
- 实际测试：生成 1000 个 token，速度提升 5-10 倍
- 内存占用：每个 token 需要存储 2 × n_heads × d_head × size_of(dtype)

### 技术 2：量化（Quantization）

**问题**：FP16/FP32 的计算和存储开销

- FP16：每个参数占用 2 字节，175B 参数需要 350GB
- FP32：每个参数占用 4 字节，175B 参数需要 700GB
- 计算速度：FP32 > FP16 > INT8 > INT4

**解决方案**：降低精度到 INT8/INT4

**原理**：将浮点数映射到整数范围

```python
def quantize_to_int8(tensor, scale, zero_point):
    """将 FP16 张量量化到 INT8"""
    # 量化公式：int8 = round(fp16 / scale) + zero_point
    quantized = torch.round(tensor / scale) + zero_point
    return torch.clamp(quantized, -128, 127).to(torch.int8)

def dequantize_from_int8(int8_tensor, scale, zero_point):
    """将 INT8 张量反量化到 FP16"""
    # 反量化公式：fp16 = (int8 - zero_point) * scale
    return (int8_tensor.to(torch.float16) - zero_point) * scale

def calibrate_quantization_scale(tensor, quantization_range=(-128, 127)):
    """计算量化参数"""
    # 计算 scale 和 zero_point
    min_val = tensor.min().item()
    max_val = tensor.max().item()

    # 确保 min_val 和 max_val 不相等
    if min_val == max_val:
        max_val = min_val + 1.0

    scale = (max_val - min_val) / (quantization_range[1] - quantization_range[0])
    zero_point = quantization_range[1] - max_val / scale

    return scale, zero_point
```

**完整量化流程**：

```python
class QuantizedLinear(nn.Module):
    """量化线性层"""
    def __init__(self, in_features, out_features, quantization_bits=8):
        super().__init__()
        self.in_features = in_features
        self.out_features = out_features
        self.quantization_bits = quantization_bits

        # 原始权重（FP16）
        self.weight = nn.Parameter(torch.randn(out_features, in_features, dtype=torch.float16))

        # 量化参数
        self.register_buffer('scale', torch.ones(1, dtype=torch.float16))
        self.register_buffer('zero_point', torch.zeros(1, dtype=torch.float16))

    def quantize_weight(self):
        """量化权重"""
        self.scale, self.zero_point = calibrate_quantization_scale(self.weight)
        self.quantized_weight = quantize_to_int8(self.weight, self.scale, self.zero_point)

    def forward(self, x):
        """使用量化权重计算"""
        # 反量化权重（计算时使用 FP16）
        weight_fp16 = dequantize_from_int8(self.quantized_weight, self.scale, self.zero_point)
        return F.linear(x, weight_fp16)

def quantize_model(model, quantization_bits=8):
    """量化整个模型"""
    quantized_model = copy.deepcopy(model)

    for name, module in quantized_model.named_modules():
        if isinstance(module, nn.Linear):
            # 替换为量化线性层
            quantized_linear = QuantizedLinear(
                module.in_features,
                module.out_features,
                quantization_bits
            )
            quantized_linear.weight.data = module.weight.data
            quantized_linear.bias.data = module.bias.data if module.bias is not None else None
            quantized_linear.quantize_weight()

            # 替换模块
            set_module(quantized_model, name, quantized_linear)

    return quantized_model
```

**量化策略对比**：

| 量化策略 | 精度损失 | 推理速度 | 内存占用 | 推荐场景 |
|---------|---------|---------|---------|---------|
| FP32 | 0% | 基准 | 100% | 高精度任务 |
| FP16 | <1% | 2x | 50% | 大多数场景 |
| INT8 | 1-3% | 4x | 25% | 生产环境 |
| INT4 | 3-5% | 6x | 12.5% | 边缘设备 |

**实际效果**：
- INT8 量化：推理速度提升 3-4 倍，内存占用降低 75%
- INT4 量化：推理速度提升 5-6 倍，内存占用降低 87.5%
- 精度损失：INT8 通常 <3%，INT4 通常 <5%

### 技术 3：投机采样（Speculative Decoding）

**问题**：大模型逐个生成 token，速度慢

- GPT-4 生成 1 个 token 需要 30-50ms
- 生成 1000 个 token 需要 30-50 秒
- 大部分时间是"串行"等待，无法并行

**解决方案**：小模型先行，大模型验证

**原理**：
1. 使用小模型（如 GPT-2）快速生成候选 tokens
2. 使用大模型（如 GPT-4）验证这些 tokens
3. 接受验证通过的 tokens，拒绝不通过的

```python
def speculative_decoding(small_model, large_model, prompt, max_tokens=100, draft_tokens=10):
    """投机采样算法"""
    tokens = small_model.tokenizer.encode(prompt)
    generated_tokens = tokens.copy()

    while len(generated_tokens) < max_tokens:
        # 步骤1：小模型快速生成候选 tokens
        with torch.no_grad():
            draft_outputs = small_model.generate(
                input_ids=torch.tensor([generated_tokens]).cuda(),
                max_new_tokens=draft_tokens,
                do_sample=False,
                temperature=0.0
            )
            draft_tokens_list = draft_outputs[0][len(generated_tokens):].tolist()

        # 步骤2：大模型验证候选 tokens
        accepted_tokens = []
        for i, candidate_token in enumerate(draft_tokens_list):
            with torch.no_grad():
                # 计算大模型的概率分布
                large_outputs = large_model(
                    input_ids=torch.tensor([generated_tokens + draft_tokens_list[:i]]).cuda()
                )
                large_probs = F.softmax(large_outputs.logits[0, -1, :], dim=-1)

                # 计算小模型的概率分布
                small_outputs = small_model(
                    input_ids=torch.tensor([generated_tokens + draft_tokens_list[:i]]).cuda()
                )
                small_probs = F.softmax(small_outputs.logits[0, -1, :], dim=-1)

                # 验证候选 token
                if large_probs[candidate_token] >= small_probs[candidate_token]:
                    accepted_tokens.append(candidate_token)
                else:
                    # 拒绝：重新采样
                    new_token = torch.multinomial(large_probs, num_samples=1).item()
                    accepted_tokens.append(new_token)
                    break

        # 步骤3：添加接受的 tokens
        generated_tokens.extend(accepted_tokens)

        # 如果遇到 EOS，停止生成
        if generated_tokens[-1] == large_model.tokenizer.eos_token_id:
            break

    return small_model.tokenizer.decode(generated_tokens)
```

**效果**：
- 小模型：GPT-2（1.5B）生成速度 1000 tokens/s
- 大模型：GPT-4（175B）生成速度 20 tokens/s
- 投机采样：综合速度提升 5-10 倍
- 精度：几乎无损失（因为最终由大模型验证）

### 技术 4：批处理优化（Batching）

**问题**：单请求处理效率低

- GPU 大部分时间空闲（等待用户输入）
- 单请求无法充分利用 GPU 并行能力
- 内存利用率低

**解决方案**：批量处理多个请求

**动态批处理（Dynamic Batching）**：

```python
class DynamicBatchScheduler:
    """动态批处理调度器"""
    def __init__(self, max_batch_size=8, max_wait_time_ms=50):
        self.max_batch_size = max_batch_size
        self.max_wait_time_ms = max_wait_time_ms
        self.pending_requests = []
        self.results = {}

    async def add_request(self, request_id, prompt, max_tokens=100):
        """添加请求到批处理队列"""
        self.pending_requests.append({
            'request_id': request_id,
            'prompt': prompt,
            'max_tokens': max_tokens,
            'timestamp': time.time()
        })

        # 等待结果
        while request_id not in self.results:
            await asyncio.sleep(0.01)

        return self.results.pop(request_id)

    async def process_batch(self, model):
        """处理一批请求"""
        while True:
            # 收集一批请求
            batch = []
            start_time = time.time()

            while len(batch) < self.max_batch_size:
                if self.pending_requests:
                    batch.append(self.pending_requests.pop(0))
                else:
                    await asyncio.sleep(0.001)

                # 超时：即使未满也处理
                if (time.time() - start_time) * 1000 >= self.max_wait_time_ms:
                    break

            if not batch:
                continue

            # 批量生成
            prompts = [req['prompt'] for req in batch]
            max_tokens_list = [req['max_tokens'] for req in batch]

            # 拼接输入（使用 padding）
            tokenized = model.tokenizer(
                prompts,
                padding=True,
                return_tensors='pt'
            )

            with torch.no_grad():
                outputs = model.generate(
                    input_ids=tokenized['input_ids'].cuda(),
                    attention_mask=tokenized['attention_mask'].cuda(),
                    max_length=max_tokens_list[0] + len(tokenized['input_ids'][0])
                )

            # 分离结果
            for i, req in enumerate(batch):
                generated = model.tokenizer.decode(
                    outputs[i][len(tokenized['input_ids'][i]):],
                    skip_special_tokens=True
                )
                self.results[req['request_id']] = generated

async def demo_dynamic_batching():
    """演示动态批处理"""
    scheduler = DynamicBatchScheduler(max_batch_size=8, max_wait_time_ms=50)

    # 创建模型（模拟）
    model = create_mock_model()

    # 启动批处理线程

        scheduler.add_request(i, f"问题{i}，请回答。", max_tokens=50)

    ]

    # 等待所有请求完成
    results = await asyncio.gather(*requests)

    print(f"完成 {len(results)} 个请求")
```

**连续批处理（Continuous Batching）**：

```python
class ContinuousBatchScheduler:
    """连续批处理调度器（更高效）"""
    def __init__(self, max_batch_size=8):
        self.max_batch_size = max_batch_size
        self.active_requests = {}  # {request_id: {'tokens': [...], 'state': ...}}
        self.request_queue = []
        self.results = {}

    async def add_request(self, request_id, prompt, max_tokens=100):
        """添加请求"""
        self.request_queue.append({
            'request_id': request_id,
            'prompt': prompt,
            'max_tokens': max_tokens
        })

        while request_id not in self.results:
            await asyncio.sleep(0.01)

        return self.results.pop(request_id)

    async def step(self, model):
        """处理一步（一个 token）"""
        # 步骤1：从队列中添加新请求
        while len(self.active_requests) < self.max_batch_size and self.request_queue:
            new_req = self.request_queue.pop(0)
            tokens = model.tokenizer.encode(new_req['prompt'])
            self.active_requests[new_req['request_id']] = {
                'tokens': tokens,
                'generated_tokens': [],
                'max_tokens': new_req['max_tokens'],
                'finished': False
            }

        # 步骤2：准备批次
        request_ids = list(self.active_requests.keys())
        batch_tokens = [
            self.active_requests[req_id]['tokens'] +
            self.active_requests[req_id]['generated_tokens']
            for req_id in request_ids
        ]

        # Padding
        max_len = max(len(tokens) for tokens in batch_tokens)
        padded_tokens = [
            tokens + [model.tokenizer.pad_token_id] * (max_len - len(tokens))
            for tokens in batch_tokens
        ]

        # 步骤3：批量生成一个 token
        input_ids = torch.tensor(padded_tokens).cuda()

        with torch.no_grad():
            outputs = model(input_ids)
            next_tokens = torch.argmax(outputs.logits[:, -1, :], dim=-1).tolist()

        # 步骤4：更新状态
        for req_id, next_token in zip(request_ids, next_tokens):
            req = self.active_requests[req_id]

            if next_token == model.tokenizer.eos_token_id or \
               len(req['generated_tokens']) >= req['max_tokens']:
                # 完成
                generated = model.tokenizer.decode(
                    req['tokens'] + req['generated_tokens'],
                    skip_special_tokens=True
                )
                self.results[req_id] = generated
                req['finished'] = True
            else:
                # 继续生成
                req['generated_tokens'].append(next_token)

        # 步骤5：移除已完成的请求
        self.active_requests = {
            req_id: req
            for req_id, req in self.active_requests.items()
            if not req['finished']
        }
```

**效果**：
- 动态批处理：吞吐量提升 3-5 倍
- 连续批处理：吞吐量提升 5-8 倍
- 内存利用率：从 <20% 提升到 >80%

## 四、常见陷阱与解决方案

### 陷阱 1：量化后精度损失严重

**问题**：INT4 量化后，模型性能明显下降

**原因**：
- 量化参数（scale、zero_point）不准确
- 对异常值（outliers）敏感
- 关键层（如最后一层）量化损失大

**解决方案**：
```python
def smart_quantization(model, quantization_bits=8):
    """智能量化：保护关键层"""
    quantized_model = copy.deepcopy(model)

    for name, module in quantized_model.named_modules():
        if isinstance(module, nn.Linear):
            # 判断是否是关键层
            is_critical = 'lm_head' in name or 'final' in name

            # 关键层使用 FP16，其他层量化
            if is_critical:
                quantization_bits = 16  # FP16
            else:
                quantization_bits = 8  # INT8

            quantized_linear = QuantizedLinear(
                module.in_features,
                module.out_features,
                quantization_bits
            )
            quantized_linear.weight.data = module.weight.data
            quantized_linear.bias.data = module.bias.data if module.bias is not None else None
            quantized_linear.quantize_weight()

            set_module(quantized_model, name, quantized_linear)

    return quantized_model
```

### 陷阱 2：批处理导致延迟增加

**问题**：批处理提高了吞吐量，但单个请求的延迟增加

**原因**：
- 等待批处理队列填满
- 固定批大小导致延迟不可控

**解决方案**：
- 使用动态批处理（设置 max_wait_time_ms）
- 使用连续批处理（更灵活）
- 优先级队列（紧急请求优先处理）

### 陷阱 3：投机采样的小模型选择不当

**问题**：投机采样速度提升不明显

**原因**：
- 小模型和大模型的分布差异太大
- 小模型生成的候选 tokens 质量差，大部分被拒绝

**解决方案**：

### 陷阱 4：KV Cache 内存泄漏

**问题**：长时间运行后，显存占用不断增加

**原因**：
- KV Cache 没有及时清理
- 缓存的 tokens 过长

**解决方案**：
```python
class KVCacheWithLimit:
    """带限制的 KV Cache"""
    def __init__(self, max_cache_length=2048):
        self.max_cache_length = max_cache_length
        self.keys = []
        self.values = []

    def update(self, key, value):
        """更新缓存，限制长度"""
        self.keys.append(key)
        self.values.append(value)

        # 如果超过限制，删除最早的
        if len(self.keys) > self.max_cache_length:
            self.keys.pop(0)
            self.values.pop(0)

        return torch.cat(self.keys, dim=1), torch.cat(self.values, dim=1)
```

## 五、总结与展望

### 推理加速的核心原则

1. **减少重复计算**：KV Cache
2. **降低精度**：量化（FP16 → INT8 → INT4）
3. **并行处理**：批处理、投机采样
4. **算子融合**：BetterTransformer、Flash Attention
5. **硬件优化**：专用芯片（TPU、ASIC）

### 技术选择建议

| 场景 | 推荐技术组合 | 预期加速比 |
|-----|------------|-----------|
| 单卡部署 | FP16 + KV Cache + BetterTransformer | 5-8x |
| 多卡部署 | FP16 + KV Cache + 批处理 | 8-15x |
| 边缘设备 | INT4 + 量化 + 模型蒸馏 | 10-20x |
| 生产环境 | 全部优化 | 15-30x |

### 未来趋势

1. **更高效的量化**：INT2、INT1 量化
2. **模型蒸馏**：大模型 → 小模型（无损）
3. **专用芯片**：ASIC、FPGA 加速
4. **动态推理**：根据输入复杂度动态调整
5. **分布式推理**：多机多卡协同
