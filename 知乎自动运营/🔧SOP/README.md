## 📅 每日运营清单（快速参考）

> 每天花 10 分钟完成以下动作，保持账号活跃度。

### ☀️ 每日必做（10 分钟）

- [ ] 查看知乎热榜，识别可介入的技术话题
- [ ] 检查昨日发布内容的数据（赞同/收藏/评论）
- [ ] 回复评论区留言（重点回复高质量问题）
- [ ] 检查待发布队列，确认今日是否有发布计划

### 📅 每周必做（30 分钟）

- [ ] 整理选题池，补充 3-5 个新选题
- [ ] 完成 1-2 篇草稿的创作或修改
- [ ] 更新数据看板，记录本周核心数据
- [ ] 复盘本周发布内容效果，提炼规律
- [ ] 更新工作日志

### 📆 每月必做（2 小时）

- [ ] 月度数据复盘，分析增长趋势
- [ ] 竞品分析，研究同赛道头部创作者
- [ ] 知识库更新，补充 AIGC 技术/OpenClaw 最新进展
- [ ] 方法论沉淀，将验证有效的经验写入方法论目录
- [ ] 规划下月内容计划

## 📝 内容发布流程（快速参考）

### Step 1：选题确认
- 从选题池中选取优先级最高的选题
- 确认选题角度、目标人群、核心价值点

### Step 2：草稿创作
- 参考 `📚方法论/📝内容创作/` 中的写作规范
- 结构：结论先行 → 原理展开 → 实操演示 → 避坑总结
- 必须包含：代码/截图/流程图至少一项

### Step 3：发布前检查
- [ ] 标题符合爆款公式，精准触达搜索意图
- [ ] 正文结构清晰，有标题层级
- [ ] 配图/代码完整，无缺失
- [ ] 字数达标（回答 800+，专栏文章 1500+）
- [ ] 末尾有引导关注或付费专栏的 CTA

### Step 4：发布与记录
- 通过知乎创作中心发布
- 发布后在 `📤已发布/` 目录归档内容
- 同步更新 `📝发布计划/` 中的计划状态

### Step 5：发布后运营
- 发布后 1 小时内回复前 3 条评论（提升互动率）
- 24 小时后记录初始数据（赞同/收藏/评论）
- 72 小时后判断内容是否需要优化补充

## 🤖 自动化操作规范

> 使用 playwright-browser 进行自动化操作时，需遵守以下规范：

1. **登录方式**：使用 MCP 方式操控浏览器，不要打开全新浏览器（会丢失登录态）
2. **操作频率**：避免高频操作触发平台风控，批量操作之间需间隔随机时间
3. **数据采集**：采集完成后立即写入数据看板，避免数据丢失
4. **发布操作**：自动发布前必须人工确认内容质量，不要全自动无人值守发布
5. **异常处理**：遇到验证码/封号提示，立即停止操作并记录到工作日志
