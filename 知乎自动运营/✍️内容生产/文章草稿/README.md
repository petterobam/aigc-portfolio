---

## 📋 目录用途

本目录存储写作中的文章草稿，文章完成后移动到 `../发布包/` 目录。草稿可能包含：
- **正在写作的文章**：已开始写作，但未完成
- **待优化的文章**：已完成初稿，但需要优化
- **待评估的文章**：已完成写作，但需要量化评估

---

## 🔄 工作流程

### 写作流程

```
1. 选题（从选题池选择）
   ↓
2. 构思（设计内容和结构）
   ↓
3. 写作（撰写内容）
   ↓
4. 优化（优化标题和互动性）
   ↓
5. 评估（量化评估和优化）
   ↓
6. 完成（移动到发布包目录）
```

### 状态定义

| 状态 | 说明 | 下一步 |
|------|------|--------|
| 正在写作 | 已开始写作，但未完成 | 继续写作 |
| 待优化 | 已完成初稿，但需要优化 | 优化标题和互动性 |
| 待评估 | 已完成写作，但需要量化评估 | 使用评估工具评估 |
| 已完成 | 已完成所有步骤 | 移动到发布包目录 |

## 🎯 质量评估

### 评估维度

| 维度 | 分值 | 评估标准 |
|------|------|---------|
| 标题质量 | 20分 | 长度（10-30字）、包含数字、价值承诺、吸引力 |
| 内容结构 | 30分 | 章节层级、开头结尾、段落长度、图片数量 |
| 技术深度 | 25分 | 原理讲解、代码示例、对比分析、案例研究 |
| 实用性 | 15分 | 可复制性、避坑指南、实用价值 |
| 互动性 | 10分 | 引导评论、引导关注、引导点赞收藏 |

### 质量等级

| 等级 | 分数 | 预期表现 |
|------|------|---------|
| 爆文 | 90-100 | 预期赞同500+，预期收藏150+ |
| 优秀 | 80-89 | 预期赞同300+，预期收藏100+ |
| 良好 | 70-79 | 预期赞同200+，预期收藏50+ |
| 及格 | 60-69 | 预期赞同100+，预期收藏30+ |

## 🚀 完成后的操作

### 1. 量化评估

- 使用评估工具评估文章质量
- 查看各维度得分
- 识别优缺点

### 2. 优化调整

- 根据评估结果优化文章
- 重点优化标题和互动性
- 再次评估优化效果

### 3. 生成发布包

- 将 Markdown 转换为 JSON
- 添加标题、内容、标签等字段
- 保存到 `../发布包/` 目录

### 4. 移动草稿

- 将草稿移动到 `🗂️归档记录/` 目录
- 或删除草稿文件

## 💡 使用指南

### 开始写作

1. 从选题池选择一个选题
2. 使用文章结构模板构思内容
3. 开始写作

### 优化文章

1. 完成初稿后，量化评估文章质量
2. 根据评估结果优化文章
3. 重点优化标题和互动性
4. 再次评估优化效果

### 完成文章

1. 确认文章质量达到目标等级
2. 将 Markdown 转换为 JSON（发布包格式）
3. 保存到 `../发布包/` 目录
4. 将草稿移动到 `🗂️归档记录/` 目录或删除
