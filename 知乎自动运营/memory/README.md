## 📋 记忆类型

### 1. 经验教训

记录在工作中学到的重要经验和教训：
- 技术问题的解决方法
- 流程优化的经验
- 避坑指南

### 2. 方法论沉淀

记录经过验证的有效方法：
- 内容创作的有效技巧
- 数据分析的方法
- 自动化脚本的使用经验

### 3. 重要决策

记录项目中的重要决策和背景：
- 为什么选择这个方案
- 当时的考虑因素
- 后续的验证结果

### 4. 关键信息

记录项目中的关键信息：
- 知乎平台的规则变化
- 账号的重要状态
- 外部资源的位置

## 💡 记忆使用指南

### 什么时候创建记忆

1. **遇到重要问题时**: 记录问题和解决方法
2. **验证有效方法时**: 记录可复用的经验
3. **做出重要决策时**: 记录决策背景和原因
4. **发现关键信息时**: 记录重要的配置和规则

### 记忆的粒度

- 不要太细：琐碎的细节不值得记录
- 不要太泛：空泛的记录没有实用价值
- 刚刚好：记录核心要点和关键步骤

### 记忆的可执行性

好的记忆应该：
- 清晰：有明确的步骤或方法
- 实用：可以直接应用到工作中
- 可验证：可以验证有效性
