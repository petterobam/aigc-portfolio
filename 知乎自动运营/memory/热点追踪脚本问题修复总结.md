# 知乎热点追踪脚本 - 问题修复总结

## 修复时间
2026-03-28

## 问题描述
热点追踪脚本在测试时遇到 3 个关键问题：
1. arXiv 选择器错误（采集到 0 篇论文）
2. Hacker News 和 GitHub Trending 超时（60秒）
3. 知乎热榜需要 Cookie（Cookie 文件不存在）

## 根因分析

### arXiv 选择器问题
- **根本原因**: 使用了错误的选择器 `.list-identifier`
- **实际结构**: arXiv 使用定义列表（`dl`, `dt`, `dd`），论文信息分布在 `dt` 和 `dd` 元素中
- **正确选择器**: 使用 `dd` 元素作为主容器，在 `dd` 内部查找 `.list-title` 和 `.list-authors`

### 超时问题
- **根本原因**: 超时时间设置过短（30秒），等待策略过于严格（`networkidle`）
- **实际需求**: 网站加载时间超过 30 秒，需要更宽松的等待条件
- **正确配置**: 增加超时时间到 60 秒，改用 `domcontentloaded` + 手动等待

## 解决方案

### 1. 修复 arXiv 选择器
```javascript
// 修改前
const paperElements = document.querySelectorAll('.list-identifier');

// 修改后
const paperElements = document.querySelectorAll('dd');

// 查找对应的 dt 元素获取链接
const dtElement = element.previousElementSibling;
let link = '';
if (dtElement) {
  const linkEl = dtElement.querySelector('a[href*="abs"]');
  if (linkEl) {
    link = linkEl.href;
  }
}
```

### 2. 修复超时问题
```javascript
// 所有数据源统一修改
await page.goto(url, {
  waitUntil: 'domcontentloaded',  // 修改前: networkidle
  timeout: 60000  // 修改前: 30000
});
await page.waitForTimeout(3000);  // 增加手动等待
```

## 技术要点

### Playwright 网页抓取
- `$$eval` 是测试选择器的好方法
- `previousElementSibling` 可以获取兄弟元素
- `textContent` 需要手动清理（替换前缀、trim）

### arXiv 页面结构
- 使用定义列表（`dl`, `dt`, `dd`）组织内容
- 论文信息分布在 `dt`（ID、链接）和 `dd`（标题、作者）
- 选择器需要匹配实际 DOM 结构

### 自动化系统建设
- 先解决核心问题，再逐步完善
- 即使部分功能失败，系统仍然有价值
- 调试和验证是系统建设的关键环节
