# 标题：Transformer 原理深度解析（附源码推导）

## 开篇
Transformer 是 2017 年由 Google 提出的序列到序列模型，彻底改变了 NLP 领域。它抛弃了传统的 RNN/CNN 结构，通过自注意力机制实现了并行计算，大大提高了训练效率。

本文将深入解析 Transformer 的核心原理，包括：
- 自注意力机制的数学推导
- 位置编码的作用
- 完整的代码实现

## 核心内容

### 1. 核心思想

**自注意力机制**
Transformer 的核心是自注意力机制，它允许模型在处理序列时，同时考虑所有位置的信息，而不是像 RNN 那样逐个处理。

**优势**
- 并行计算（训练速度快）
- 长距离依赖（能看到整个序列）
- 灵活性强（适用于各种任务）

### 2. 数学推导

**2.1 Attention 公式**

$$Attention(Q, K, V) = softmax(\frac{QK^T}{\sqrt{d_k}})V$$

**符号解释**
- Q (Query): 查询向量
- K (Key): 键向量
- V (Value): 值向量
- $d_k$: 向量维度

**推导过程**
1. 计算 Query 和 Key 的相似度：$QK^T$
2. 缩放除以 $\sqrt{d_k}$：防止梯度消失
3. Softmax 归一化：得到注意力权重
4. 加权求和：输出注意力结果

**直观理解**
可以理解为：对于每个位置，我都要问"整个序列中哪些部分与我相关？"，然后根据相关性加权求和。

### 3. 代码实现

```python
import torch
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, query, key, value):
        batch_size = query.size(0)

        # Linear projections
        Q = self.W_q(query)  # (batch_size, seq_len, d_model)
        K = self.W_k(key)    # (batch_size, seq_len, d_model)
        V = self.W_v(value)  # (batch_size, seq_len, d_model)

        # Split into multiple heads
        Q = Q.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = K.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = V.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)

        # Scaled dot-product attention
        scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(torch.tensor(self.d_k, dtype=torch.float32))
        attention_weights = torch.nn.functional.softmax(scores, dim=-1)
        output = torch.matmul(attention_weights, V)

        # Concatenate heads
        output = output.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)

        # Final linear projection
        output = self.W_o(output)

        return output, attention_weights
```

**代码解释**
1. 输入：Q, K, V（形状：batch_size × seq_len × d_model）
2. 线性变换：得到多头表示
3. 注意力计算：计算注意力权重并加权求和
4. 拼接多头：将多个头的输出拼接
5. 线性投影：最终输出

### 4. 应用案例

**机器翻译**
- 输入：中文句子
- 输出：英文句子
- 效果：BLEU 分数提升 5+

**文本生成**
- 输入：前缀文本
- 输出：续写内容
- 效果：生成质量提升 20%

## 总结升华

Transformer 的核心创新是自注意力机制，它解决了 RNN 无法并行计算的问题，同时能够捕捉长距离依赖。它成为了 NLP 领域的基石，被广泛用于各种任务（翻译、生成、分类等）。

**局限性**
- 计算复杂度高：$O(n^2)$
- 长序列处理困难
- 训练数据需求大

**发展趋势**
- 线性注意力（Linformer）
- 稀疏注意力（Sparse Transformer）
- 混合模型（RNN + Attention）

**学习建议**
1. 从简单的例子入手（单层 Attention）
