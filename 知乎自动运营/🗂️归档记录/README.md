# 🗂️ 归档记录

> 历史工作成果的归档存储，保持主工作区的整洁

---

## 🎯 用途

- 归档已完成的阶段性工作成果
- 存储历史版本的文件和文档
- 保持主工作区的简洁和高效
- 提供历史记录的可追溯性

---

## 📂 目录结构

```
🗂️ 归档记录/
├── 📄 README.md                   ⭐ 本文件（归档索引）
│
├── 📦 {项目名}-{日期}/            按项目归档
│   ├── 文档
│   ├── 脚本
│   └── 数据
│
└── 📋 {日期}-归档记录.md           归档清单和说明
```

---

## 📋 归档原则

### 何时归档

1. **阶段性工作完成**: 一个阶段的所有任务完成后
2. **版本迭代**: 文档或脚本有重大版本更新后
3. **项目结项**: 某个独立项目完成后
4. **清理整理**: 定期清理不常用文件时

### 归档内容

1. **文档**: 已完成的项目文档、分析报告
2. **脚本**: 已废弃或不再使用的脚本
3. **数据**: 历史数据文件（可选择性归档）
4. **笔记**: 临时的工作笔记和草稿

### 不归档内容

1. **活跃文件**: 仍在使用和更新的文件
2. **核心文档**: README、HEARTBEAT.md 等核心文件
3. **配置文件**: 正在使用的配置文件
4. **工作日志**: 心跳工作日志（保持在工作日志目录）

## 📦 归档操作流程

### 1. 创建归档目录

```bash
mkdir -p "~/.openclaw/workspace/知乎自动运营/🗂️归档记录/{项目名}-{日期}"
```

### 2. 移动文件

```bash
mv {文件路径} "~/.openclaw/workspace/知乎自动运营/🗂️归档记录/{项目名}-{日期}/"
```

### 3. 创建归档记录

在 `归档记录/{日期}-归档记录.md` 中记录：
- 归档日期
- 归档内容清单
- 归档原因
- 归档文件的位置

### 4. 更新主工作区

删除或更新相关的引用和链接

## 🔍 查找归档内容

### 方法 1：通过归档记录

查看 `归档记录/{日期}-归档记录.md` 文件，了解归档内容的位置

### 方法 2：通过目录结构

根据项目名称和日期，直接浏览归档目录

### 方法 3：搜索文件

```bash
cd ~/.openclaw/workspace/知乎自动运营/🗂️归档记录/
find . -name "*关键词*"
```
